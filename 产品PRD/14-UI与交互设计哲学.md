# 14 · UI 与交互设计哲学（**强制律 · loop 必遵**）

> 这是与「**开发铁律**」（13 §0 / 00 §0，管引擎兼容）并列的**第二条律**（管页面与交互）。**开发/loop 每写一页都要过本文红线**；违反即返工。
> 栈：**React + Vite + Ant Design Pro**（ProTable / ProForm / EditableProTable）——现项目即用此栈，本律可直接落地。
> 依据：Linear / Pencil&Paper《Enterprise Data Tables》/ AntD Pro / Airtable / SAP Fiori / Dynamics Saved Views / Optimistic UI(Simon Hearne) / LogRocket Modal / UXmatters《Wizards vs Forms》。状态：定稿。日期：2026-06-16。

## 0. 第一性原理
**这是给高频文员用的记录系统**。工具应"消失"（交互 <100ms 才有瞬时感）。**天天录的是熟悉任务 → 用密集表单/网格，绝不用分步向导或流程图**（向导只给低频陌生任务）。一切以"每次操作点击最少、信息一屏看全、像 Excel 一样顺手"为准。

## 1. 骨架四件套（全系统一套，学一次会全部）
**工作台/待办 → 台账列表 → 详情抽屉 → 动作按钮**。
- 登录直落工作台/待办收件箱，**从待办进单**。
- 每类单据 = 一张密集**台账表**；点行**右侧滑抽屉**看/改，**列表不消失、不跳页**。
- 详情页顶部给**"当前能做的动作按钮"**（提交/审核/发货），点即推进；**状态机后台跑、前端只显状态药丸**。
- **节点图（FlowEditor）只给管理员配流程，终端用户永不从节点图推进。**

## 2. Saved Views 当导航
常用筛选存**命名视图**（今日待录入/欠票未到/在途未收/待推金蝶/我的待审）当标签，可设默认、按岗位预置。替代"每天重设筛选"。

## 3. 录单效率（对"爱表格"群体最关键）
- **可编辑网格 inline edit**：Tab/Enter/Esc、数字右对齐+等宽字体、**Excel 粘贴整块批量建行**、向下填充、**扫码枪直接入格**。
- **键盘流**：纯键盘走完"建单→填→存→建下一张"。
- **乐观 UI**：改状态/保存立即反馈，后台异步落库；**失败就地标红、绝不静默吞**（接金蝶推单"静默缺行"坑）。

## 4. 批量操作（标配）
多选框 + 选中后顶部 alert 条挂**批量改状态 / 批量推金蝶 / 批量导出**。

## 5. 密度可控 + 不丢关键列
紧凑/默认/宽松行高切换并记忆；**冻结单号/客户列 + 右侧操作列**；表头吸顶；点行**就地展开**看明细。

## 6. 进阶（v2，非首期硬需）
`Cmd/K` command palette 把"建单/查单/切视图/切公司"集中；快捷键边用边学。

## 7. 角色裁剪 + 一套壳
每角色的列/卡/动作按角色裁（字段防火墙 + 动作权限）；全模块同骨架。

## 8. 未做功能 = 可见占位
留口子的都做"**功能已就绪·待开通**"。

## ⛔ 红线（违反即返工，loop 自检必过）
1. **禁止**用流程图/节点图让用户推进业务。
2. **禁止**用分步向导跑高频录单任务。
3. **禁止**深层页面跳转丢上下文（用列表-详情抽屉 + 就地展开）。
4. **禁止** modal 套 modal 承载主录单流（提速就不惜代价避免 modal；用抽屉/整页）。
5. **禁止**斑马纹（用细分割线/hover）。
6. **禁止**强行大留白挤掉可见行数（密度交给用户）。
7. **禁止** "magic pushbutton"（点完干等、无中间态/无反馈）。

## 组件落地（Ant Design Pro，强制复用同一积木）
- **台账列表 = `ProTable`**（查询表单 + 分页/排序/筛选 + 列配置 + 密度 + `rowSelection` 批量 + `tableAlertRender`）。
- **详情/录单 = `DrawerForm`/`ProForm`（抽屉优先）+ `EditableProTable`（明细网格）**。
- **提交 = 预览 → ChangeCard → commit**（引擎现成）。
- **引擎映射（零对抗）**：`DataExplorer` ≈ ProTable 台账、`DocEditor` ≈ ProForm 详情 + EditableProTable 子表、`FlowEditor` 节点图**只给 admin**。

## 每类页面的标准布局（loop 照此生成）
| 页类 | 标准布局 |
|---|---|
| **单据类**(采购/销售/出入库/报关…) | 顶部 saved-views 标签 + 查询条 → **ProTable 台账**(冻结 单号/客户/操作 列) → 点行**右抽屉**(主表 ProForm + 明细 EditableProTable + 上下游关联 + 历史 + 顶部动作按钮) |
| **工作台/审批中心** | 角色化待办收件箱(list_user_todos) + **一键批** + 我的 KPI 卡;审批=preview→ChangeCard→commit |
| **主数据/配置** | ProTable + 抽屉编辑;字典/枚举内联管理 |
| **报表/看板** | 手写聚合路由 + 看板卡(按角色裁成本/毛利列) + 下钻明细 ProTable |
| **留口子功能** | 可见入口 + "功能已就绪·待开通"状态 + 后端预留接口 |

> **loop 自检清单(每页过)**：① 没让用户碰节点图？② 没用向导跑高频任务？③ 详情走抽屉不跳页？④ 多行录入是可编辑网格(支持键盘/粘贴/扫码)？⑤ 有 saved views + 批量 + 密度？⑥ 乐观 UI 且失败就地标红？⑦ 按角色裁了列/动作？任一否 → 返工。
