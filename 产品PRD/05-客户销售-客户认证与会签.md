## 1. 模块定位

**客户认证（供应商准入，薄版）**：大客户把富泰列为"合格供应商"前的准入审核——富泰提交资料、双方签采购/质量协议（协议里常埋 违约金/索赔/质保期冲突 等对我方不利条款，须审出来）。
**薄**：系统只管 **认证状态 + 资料清单勾选 + 协议风险审查项打勾留痕 + 附件**，**不**在系统里写协议正文。
**审核 = 并行会签**：PA + 财务 + BOSS **三方并行（任意顺序）各签一票，全"同意"才通过，任一"驳回"打回**。本模块同时定义一个**可复用的「并行会签标准件」**（§3），备货 ≥20 万会审等任何"多人全签"关卡复用它。

### 2.4 引擎映射 & 兼容性

- **认证单 = doc_type `CUSTOMER_QUALIFICATION`** + `WorkflowDefinition.states`（上方状态机）—— ✅ 引擎现成单据机制。
- **资料清单 / 风险审查 = 子表**（`qualification_doc_line` / `qualification_risk_line`，名含 `_line` → 前端 SubTableEditor 网格）—— ✅。
- **附件** —— ➕（引擎无原生附件域，加上传端点 + 附件字段，与 03b/05b/07a 共用）。
- **并行会签** —— ➕业务层标准件（§3），**不动引擎核心**。
- **通过回写客户认证码** —— ➕ `@register_transition_effect`（APPROVED→写 customer.qualified_code）。
- 权限：销售端可发起/看本人客户；签票方只可编辑自己那行会签（`editable_fields` 限定）。

### 2.5 验收 + 待甲方

- 验收：必备资料未齐/风险项未判 → 不能提交；三方未集齐同意 → 不能通过；任一驳回 → 打回并保留各方意见留痕；到期自动置失效。
- 待甲方：① 资料清单与风险审查项的**标准模板**（首批由业务签字）；② 认证类型字典；③ 是否需"PM/FAE 必填技术质保判断"才能提交（默认否，可指派）。

## 3. 并行会签标准件（可复用组件，**不动引擎核心**）

> 用途：任何"**多人全签才过**"的关卡（客户认证审核、备货 ≥20 万会审，及将来任何会签）都复用本件，**避免每处各写一套、也避免改引擎**。

### 3.1 机制（三块现成积木拼）

1. **会签签字子表 `cosign_line`**（挂在要会签的单据上）：`required_role`（应签角色）、`signed_by`（签字人）、`decision`（同意/驳回）、`comment`、`signed_at`。提交进入会签态时，按本关卡配置（如 `[PA, FINANCE, BOSS]`）**预生成三行待签**。
2. **"我签字" = 一个自循环编辑动作**（`UNDER_COSIGN` 态的 `editable_fields` 只含 `cosign_line`，`allowed_roles=[PA,FINANCE,BOSS]`）：每个签票方往**自己那行**填 同意/驳回 + 意见。**并行、任意顺序、随时签**。
3. **"通过" = 一个推进动作挂校验器**（`@register_transition_validator` / `hard_rule`）：放行条件 = **所有 `required_role` 行 `decision=同意`**；只要有一行 `decision=驳回` → 走 `REJECTED`；未集齐 → 不可推进（留在会签态）。

### 3.2 引擎映射 & 兼容性

- 子表 = ✅（SubTableEditor 现成）；自循环编辑 = ✅（execute_transition 编辑模式）；集齐校验 = ✅（hard_rules DSL 的 `all/count` + 跨行判断，或 validator）。
- **整体 = ➕业务层"会签标准件"**：用现有积木拼出"全签才过"，**不改 `execute_transition`、不给引擎加会签原生语义**（符合开发铁律：引擎不认业务术语）。
- 对照引擎原生：引擎节点 `allowed_roles` 语义是"任一角色可推进"；本件用"子表攒票 + 集齐校验"把它升级成"全签才过"，是**纯业务层组合**。

### 3.3 配置项（每个会签关卡各配一份，不改代码）

- `required_roles`：哪些角色必须签（认证=PA+财务+BOSS；备货≥20万会审=同款，具体角色待甲方）。
- `pass_rule`：默认"全同意"；如需"多数通过"或"任一关键角色一票否决"可在校验器配置。
- `parallel`：默认并行（任意顺序）；如某关卡要依次签，改成串行子状态即可（连校验器都省）。

### 3.4 待甲方

- 备货 ≥20 万会审是否并行三方会签（默认：可复用本件，角色待定）。
- 各会签关卡的 `required_roles` 全集与"一票否决"角色。
