# PHOTONTECK（富泰）CRM+WMS · 产品 PRD 索引

> 这是富泰 6 家公司 **CRM+WMS 商业交付级业务 PRD** 的总索引。本系统 = 6 家公司的「业务 system-of-record」，把散落在 Excel / 飞书多维表 / 邮件里的 CRM、采购供应链、仓储 WMS、报关、单据/标签全流程收口到一个有权限、有留痕、有状态机的系统；**财务做账继续在金蝶云星空**，本系统推业务单据给金蝶 + 拉金蝶已确定的报表/视图，不实时回填、不算成本。

## 与引擎 PRD 的关系

- **本目录 = 业务 PRD**（产品视角：角色、JTBD、页面、字段、状态机、关卡、金蝶推送清单、验收标准）。
- **引擎 PRD = `技术文档/00~09`**（技术底座：元数据/注册引擎、流程状态机、命令、校验副作用、权限多租户、Agent、审计、前端壳、配置）。**核心机制以代码为准**，docstring 过期处已在引擎文档标注。
- **业务设计必须落在引擎上**：每个页面逐条标 ✅compatible / ➕extension / ❌incompatible（兼容性）。引擎已知缺/弱处（报表非元数据化 D7、金蝶无原生推送、标签模板仅 custom_html 逃生舱、无采集层/定时器/邮件、多公司切换死代码 D-05c、边级角色推进路径欠账 D-02e、会话安全 D-05e/f）均以 ➕扩展或标注 gap 处理，**汇总见 `_引擎扩展点汇总.md`**。
- **唯一写入路径** `Command → Workflow → Domain`；**写必留痕**（四本审计）；**元数据优先于代码**——业务 = 状态机 + 字段变更 + 副作用，三者各有归属互不耦合。

## 决策 A / B + 12 条结构决策速查

> 全部以 `00-导航与参与者总览.md` 为权威；下游统一采纳。

### 决策 A（2026-06-16）· 财务行级范围
- 普通 `FINANCE`（应收/应付/出纳）= **严格本公司账套，绝不跨公司**；引擎须将 `FINANCE` **移出**行级 privileged 集。
- 新增 `FINANCE_DIRECTOR`（财务总监）= **跨公司只读汇总**（read-privileged）：行级可跨公司只读，但**不**绕过流程角色闸、**不**写单据。
- 行级只读 privileged 集 = **{BOSS, FINANCE_DIRECTOR}**；流程+表级全权 privileged 集 = {ADMIN, BOSS}（两集成员不同，引擎 05 D-05g）。

### 决策 B（2026-06-16）· 多公司单用户授权
- 一个用户可被授权操作**多家公司，且在各家角色相同**——用户 `role` 单一、跨所开通公司一致，**不需要 per-company 角色**。
- "开通哪些公司"由**管理员按用户可配置**（授权 = 用户 × {公司集}）。
- 会话载 `active_company_id` + 顶部**公司切换器**，仅可在已开通公司间切换。
- `_company_filter` 改为：read-privileged（BOSS/财务总监）→看全部只读；普通用户→限 `active_company_id`。
