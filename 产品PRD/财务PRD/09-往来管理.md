## 0. ★录音事实定位（本模块为什么独立、v1 错在哪）

录音原话（L241-259）：

> "往来管理就是…你的辅助账上面可以挂客户或者供应商…有一部分涉及到内部单位，比如同一个集团内部 A 公司跟 B 公司之间有往来…通过往来管理去管理他的**账期、什么时候发生的，以及他们俩之间能不能勾对**…一笔业务在集团内部只需要一家单位做，另外一家单位复核就可以了。"
> "主流的都会把这个单**拎出来做一个类似于功能小模块，像凭证处理这种级别的一个模块**。"
> "一般都是**应收、应付、预收、预付、其他应收、其他应付这几类**。"
> "往来核对这一块比较关键的就是你的**业务发生的时间**。"

含税铁律（L538-558）：

> "如果是往来的话，**往来肯定是含税的，是一致的，不可能一家不含税，一家含税**…你往来如果是一致的话，那就能核对上…只要内部往来是一致的，那合并总账就没有问题。"
> "只有你算成本/收入的时候，税进不进成本、进不进收入这部分有区别；但往来肯定含税。"

**本模块定位（v2 正）**：往来管理是一个**独立功能小模块**（与"凭证处理"同级），围绕"往来款 6 类台账 + 账期 + 业务发生时间 + 往来对账 + 内部单位勾对（一家做一家复核）+ 支撑合并报表抵消"。它**不挂凭证**（凭证只属于总账，见模块拆分铁律），它对接的是各业务模块（应收 / 应付 / 出纳 / 税务）生成的**单据/台账**，做的是**对账与勾对视角**，不是新的过账入口。

## ① 模块定位

把"6 类往来款（应收/应付/预收/预付/其他应收/其他应付）的**统一台账 + 账期 + 业务发生时间 + 往来对账单 + 内部单位勾对 + 合并抵消支撑**"做成一个**独立功能小模块**：

- **数据来源**：各业务模块的单据 / 台账（应收=销售出库→应收单；应付=采购入库→应付单；预收预付=出纳；其他应收应付=费用/借款手录）。本模块**不新建过账入口、不挂凭证**。
- **核心动作**：①往来挂账（辅助核算挂客户/供应商/内部单位）；②账龄/账期监控（按业务发生时间、到期日）；③往来对账单（与外部往来单位对账）；④**内部单位勾对**（集团内 A 应收=B 应付，自动化匹配 → 一家做一家复核）；⑤为合并报表提供内部往来抵消数据。
- **铁律**：往来款必含税、双方一致；只有内部往来一致才能勾对、才能合并；本模块只做对账视角，过账动作回到各源模块 + 总账。

**边界（录音模块拆分铁律 L334-526）**：

| 邻接模块 | 边界划分 |
|---|---|
| 总账 | 唯一挂凭证；往来核对完不进凭证生命周期。本模块**只读引用**已过账凭证/已生成单据 |
| 应收管理 | 应收单/收款单/结算单 + 收入确认（在应收）；本模块从应收**台账**取"应收/预收"往来余额，做对账视角，不重复核销 |
| 应付管理 | 应付单/付款单/结算单 + 暂估应付；本模块取"应付/预付"往来余额 |
| 出纳与资金 | 真实收付（收款单/付款单/银行日记账/银企直连）；本模块不管收付到哪个账户，只关心往来单位、业务实质、摘要 |
| 税务 | 销售发票/采购发票（含税信息）；往来必含税，发票含税额=往来金额，本模块对账以含税额为准 |
| 期末/合并报表 | 合并报表的"内部往来抵消"取数来自本模块的**勾对结果**；勾对干净→抵消，不干净→挂出差异（见 §④ §10） |

## ② 财务对象清单（统一对象范式 8 列；类型 A/B/C/D/E）

本模块 4 个对象：**统一往来台账（C 派生）**、**往来对账单（D 报表）**、**内部往来勾对单（B 单据，唯一可录的业务单）**、**往来辅助核算配置（A 主数据，挂科目维度）**。
注意：本模块**无 B 类凭证对象**——往来不挂凭证（v2 铁律）。唯一的 B 类单据是"内部勾对单"，且它也不直接过账（只标记勾对状态 + 触发对方复核）。

> ★对账状态（台账行 recon_status）不是状态机，是源单聚合派生属性：`UNRECONCILED → RECONCILING → MATCHED / DIFF`。MATCHED 仅当被一张 `MATCHED` 的勾对单覆盖。

### 3-2 往来台账/对账单：无状态机（C/D 派生只读）

台账与对账单是派生视图，无独立状态流；其"对账状态"随源单据 + 勾对单变化。源单据的状态机在**各源模块**（应收/应付/出纳），本模块不重复定义。

## ④ ★往来对账与合并抵消取数（非凭证·对账逻辑）

> 本模块不出借贷分录（不挂凭证）。本节定义"对账匹配 + 内部抵消"的取数公式，每条标「财务确认」。

### 4-1 往来余额取数（统一台账 by 类别）

| 往来类别 | 余额公式（含税）| 余额方向 | 对应科目 | 财务确认 |
|---|---|---|---|---|
| 应收 AR | Σ(应收单含税 amount − paid_amount) | 借方（资产）| 1122 | ✓ |
| 应付 AP | Σ(应付单含税 amount − paid_amount) | 贷方（负债）| 2202 | ✓ |
| 预收 ADV_RECV | Σ(预收单 − 已转应收抵扣) | 贷方（负债）| 2203 | ✓ |
| 预付 ADV_PAY | Σ(预付单 − 已转应付抵扣) | 借方（资产）| 1123 | ✓ |
| 其他应收 OTHER_AR | Σ(借款单/其他应收 − 收回)| 借方 | 1221 | ✓ 科目码待财务确认 |
| 其他应付 OTHER_AP | Σ(其他应付 − 支付)| 贷方 | 2241 | ✓ 科目码待财务确认 |

> ★含税铁律：所有往来余额均以**含税额**计（录音 L538）。本位币余额 = Σ(原币余额 × 入账汇率)，期末重估在调汇模块。

### 4-2 内部往来勾对匹配规则（一家做一家复核）

| 匹配维度 | 条件 | 财务确认 |
|---|---|---|
| 类别互补 | A 侧应收 ↔ B 侧应付（或 A 预付 ↔ B 预收）| ✓ |
| 金额一致 | A.含税额 = B.含税额（往来必一致，差额=0 才可 MATCHED）| ✓ |
| 币种一致 | A.currency = B.currency | ✓ |
| 单位互指 | A 的往来单位=B 公司 且 B 的往来单位=A 公司（内部单位 is_internal=True）| ✓ |
| 业务发生时间 | biz_date 在容差窗口内（录音"往来核对最关键就是业务发生时间"）；窗口阈值待财务确认 | ✓ 容差待定 |

> 匹配成功 → 两侧 recon_status=MATCHED → 写"内部往来抵消对照"。一笔业务集团内**只需一家做账、另一家复核**（B 侧不重复录，只确认）。

### 4-3 合并报表内部抵消取数

| 场景 | 处理 | 财务确认 |
|---|---|---|
| 内部往来已 MATCHED（干净）| 合并时**直接抵消**：A 应收 − B 应付 = 0，从合并资产负债表剔除 | ✓ |
| 内部往来有差异（DIFF）| **不自动抵消**，挂出差异清单待人工核对（往来不一致→不能合并）| ✓ |
| 跨币种内部往来 | 按统一汇率折本位币后比对（差额=汇率口径差，非业务差）| ✓ 汇率口径待定 |

> 录音 L538-558：往来一致 → 合并总账没问题。"6 公司行级隔离不混账"≠"不出合并报表"——合并是**报表层手工/半自动汇总**，依赖本模块勾对结果（MEMORY 合并报表口径）。

### 4-4 ★税口径澄清（含税铁律落地）

| 场景 | 往来金额口径 | 成本/收入口径 | 财务确认 |
|---|---|---|---|
| 内地主体（CAS，增值税）| 往来记**含税**总额（如 113=100+13 税）| 成本=不含税（进项税可抵→不进成本）；收入=不含税（税不算收入）| ✓ |
| 香港主体（HKFRS，无 VAT 抵扣）| 往来记**含税**总额（发票含税额）| 成本/收入按发票口径 | ✓ |
| 内地 A ↔ 香港 B 内部往来 | **双方均含税且金额一致**（不可一家含税一家不含税）| 各自成本/收入口径可不同，**不影响往来勾对** | ✓ |

> ★彭玉洁原误解"内地含税、香港不含税、合并怎么处理"——录音已澄清：往来一律含税一致，税差只在成本/收入侧，**往来层无此问题**。

## ⑥ 校验规则 + 边界 + 错误处理（失败精确提示，绝不静默）

| # | 规则 | hard_rule 条件 | 失败提示 |
|---|---|---|---|
| V1 | 往来必含税一致 | 勾对时 A.含税额 = B.含税额 | 「内部往来不一致：A 含税 {a} ≠ B 含税 {b}，往来款必须双方含税一致，无法勾对」|
| V2 | 类别互补 | A 应收 ↔ B 应付 / A 预付 ↔ B 预收 | 「往来类别不互补：A={ca} B={cb}，应收须对应付」|
| V3 | 跨公司复核 | 复核人 company_id = company_b_id 且 A≠B | 「内部勾对须由对方公司（{B}）复核，当前用户不属该公司」|
| V4 | 币种一致 | A.currency = B.currency | 「往来币种不一致 {ca}/{cb}，请先统一币种或折算」|
| V5 | 业务发生时间 | biz_date 在容差窗口内（默认同期，可配）| 「业务发生时间相差超容差，请核对是否同一笔业务」|
| V6 | 原币本位币双金额 | 外币往来 exchange_rate>0 且 amount_base=amount×rate | 「外币往来缺本位币金额/汇率，请补 {currency} 入账汇率」|
| V7 | 源单存在 | 勾对明细的 a_doc/b_doc 均为有效未作废单据 | 「源单 {doc} 不存在或已作废，无法勾对」|
| V8 | 勾对幂等 | 同一源单不可被两张 MATCHED 勾对单重复覆盖 | 「该往来单已被勾对单 {no} 勾对，重复操作被拦截」|
| V9 | 撤回留痕 | 撤回勾对必填原因，写审计 | 「撤回勾对须填原因」|
| V10 | 合并锁后禁改 | 合并报表已生成期间，勾对不可撤回 | 「{period} 合并已生成，勾对锁定，如需调整请先反合并」|

**边界**：
- ①**源单红冲后**：本模块台账自动反映新余额，已 MATCHED 的勾对自动失配回 DRAFT（红冲是认可原业务但金额更新，往来随之变）。
- ②**外币内部往来**：A 内地（CNY）↔ B 香港（USD/HKD）——金额一致指**原币业务实质一致**，比对时折本位币；汇率口径差挂"汇率差异"非业务差异。
- ③**预收/预付转抵**：预收转应收、预付转应付后，往来类别在台账上从 ADV_RECV→AR / ADV_PAY→AP 迁移（源模块动作触发，本模块视图随动）。
- ④**其他应收/应付（员工借款等特殊业务）**：录音 L400"不排除财务有一些除了常规业务之外的特殊业务，比如说是什么员工的借款呀，或者什么其他乱七八糟的费用"——这类无上游业务单，在源模块（费用/出纳）手录后进入本模块台账，party_type=EMPLOYEE。注：录音的"往来必含税"铁律（L553）针对的是与外部/内部单位的应收应付往来款；员工借款等其他应收应付是否同样含税口径，**录音未明示，待财务确认**（不可硬断言"借款无税"）。

## ⑩ 验收用例（Given-When-Then，含边界）

- **AC1 6 类统一台账**：Given 存在应收/应付/预收/预付/其他应收/其他应付各若干；When 查 `/api/current-account/ledger`；Then 6 类按往来单位汇总，每行**含税额 + 原币/本位币双列 + 业务发生时间 + 到期日 + 未清额**，应收合计/应付合计/净往来正确。
- **AC2 往来必含税**：Given 应收单含税 ¥113,000（含 13% 税 ¥13,000）；When 进往来台账；Then amount=113,000（含税），无"不含税往来"列，对账以含税额为准。
- **AC3 内部勾对一家做一家复核（核心）**：Given 内地 A 对香港 B 有应收 ¥56,500、B 对 A 有应付 ¥56,500（含税一致）；When A 侧 ACCOUNTANT 发起勾对并提交 → B 侧 ACCOUNTANT（属 B 公司）复核确认；Then 勾对单 MATCHED，两侧 recon_status=MATCHED，B 侧无需重复录单（只复核）。
- **AC4 勾对含税不一致拦截（边界）**：Given A 应收含税 ¥56,500、B 应付含税 ¥50,000；When 勾对；Then V1 拒绝「往来款必须双方含税一致」，无法 MATCHED。
- **AC5 跨公司复核硬校验（边界）**：Given A 侧用户试图自己确认（非 B 公司）；When 复核；Then V3 拒绝「须由对方公司复核」（一家做一家复核落地）。
