# 05 · 凭证模板与业财一体化映射（业财翻译表 / 自动凭证发动机）

> 重构基线：会计/ERP 专家答疑录音（`ERP产品|会计师|IT.md`）为最高事实源，优先级高于任何 v1 猜测稿。本文凡与录音冲突一律以录音为准；关键纠错以「v1错→v2正」小注标出。
> 本文件只写 PRD，不动代码。所有引擎接入纯扩展（`@register_command` / `@register_transition_effect` + `WorkflowDefinition.states` JSONB + `__doc_types__`），`core/registry.py` / `services/workflow.py` / `services/commands.py` 字节级零 diff。

## ① 模块定位（一句话业务价值）

**“业财翻译表”= 一份运维可自由配置、不动开发的【两映射】规则：业务单据 ↔ 凭证。** 系统把【需要生成凭证的业务单据】（采购入库单 / 销售出库单 / 收款单 / 付款单 / 进项发票 / 销项发票 / 退货单 / 折旧 / 费用 / 工资计提…）按这套规则翻译成借贷平衡的【凭证草稿】，草稿流到【总账模块】统一审核 → 过账。运维想改“某单据生成什么凭证”只需在规则表里配（借贷科目 + 金额取数 + 摘要 + 辅助核算 + 现金流量项 + 取哪个日期），不需要每次动开发。

> ★录音铁律（录音 L508-526 业财翻译表本体 + L340 模块拆分）：
> 1. **只有“总账”模块挂凭证；其他所有模块只挂业务单据。**（录音 L340：“这里面只有总账是挂的凭证，其他模块都是单据”）业务单据经本翻译表生成凭证草稿 → 流到总账统一审核过账。凭证的生命周期（审核 / 复核 / 过账 / 反过账 / 红冲）**只属于总账**，不下放到各业务单。
> 2. **不是所有单据都进凭证。**（录音 L526：“它不是所有的单据都会涉及到凭证，像订单类的根本它不会到凭证那一步，你只需要把跟凭证关联的，我采购入库单、销售出库单这种是需要生成凭证的单据，去做这样的映射”）只对【采购入库单 / 销售出库单】这种需要生成凭证的单据做映射。订单类（SALES_ORDER / PURCHASE_ORDER）不进凭证。
> 3. 翻译表 = 两个映射（录音 L514：“你这个规则就是对应了两个映射，一个就是我的前面的业务单据，一个就是我的凭证”）。凭证上每个要填的字段都可配：①借贷方科目（录音 L514“跟科目相关系的，可以写死的那种”）②金额（录音 L514：灵活，取采购入库单上的哪个成本？取发票的含税 / 不含税 / 税额？）③摘要（录音 L526：可取入库单备注，也可固化为“供应商名字加订单日期”等）④辅助核算（录音 L520）⑤**日期（录音 L520：单据有“单据创建的日期”和“业务发生的日期”两个，凭证取哪个由运维自由选）**。

> 【v1错→v2正】v1 把本模块定名为“自动凭证发动机”并将“收入确认凭证拆分粒度/CAS14 五步法/合同资产合同负债”作为本模块核心交付——录音里专家从未谈“五步法 / 合同资产 / 合同负债”，那是 v1 自行加戏。本 v2 收敛回录音口径：本模块只负责【业务单→凭证】的翻译机制 + 全量映射字典；收入确认的会计判断归【应收管理模块】（录音 L406：“收入确认，按理说应该是在放在应收模块里面”），本模块只承载其凭证规则。

本模块是全部业务模块的“公共翻译层 + 公共过账层”：采购入库 / 销售出库 / 应收 / 应付 / 出纳 / 存货成本 / 暂估 / 固资 / 薪酬 / 费用 / 税务各模块的“⑥效果（生成凭证）”统一调用本模块 `map_to_voucher_entries()` 生成草稿；草稿统一在总账走 `finance.post_voucher`。

## ② 本模块财务对象清单（对象范式）

本模块自有 2 个对象（A 主数据 + 1 套规则数据载体）+ 复用总账的 Voucher（B 交易单据，凭证生命周期归总账，本文只描述“翻译产出草稿”这一段）。退货红冲不再作为独立“业务广播对象”（见 §④.4 纠错）。

### 对象 2.1 — VoucherTemplate 凭证模板（业财翻译表头·❌新建·A 主数据）

| 范式列 | 内容 |
|--------|------|
| ① 对象 | `VoucherTemplate`（model，❌新建表）；`__queryable__=True`；无 doc_type（纯配置主数据，不走业务状态机，走专用“凭证模板维护屏” + DataExplorer 通用壳） |
| ② 类型 | **A 主数据**（公司级账套化配置；运维维护，不动开发） |
| ③ 数据 | 一行模板头=「某公司 · 某业务单类型 · 某触发动作」→ 生成一张凭证；模板行（`VoucherTemplateLine`）=该凭证的每条分录的取数规则。字段见 §3.1 |
| ④ 动作×角色×时机 | 新增 / 改 / 停用 / 启用 / 测试取数：`FINANCE_MANAGER`（建账期维护）+ 运维角色（`OPS_CONFIG`，录音“让运维可以自己去自由配置”）；查看：全财务。改模板 = 高危 → 必 `WorkflowDefAuditLog` 留痕（变更前后 JSON 快照）；生效走 `effective_date`，**不追溯改历史凭证** |
| ⑤ 逆操作 | 不冲销（配置类）。误配 → 停用（`is_active=False`）+ 新增正确版本（新 `effective_date`）。历史已过账凭证不受影响（凭证生成时即把分录快照写死，与模板解耦） |
| ⑥ 效果 | 被 `map_to_voucher_entries()` 读取，把业务单字段按 `amount_formula` / `tax_handling` / `account_source` 解析为 `VoucherEntry[]` 草稿；本身不直接产生余额变动（过账才动余额，过账归总账） |

| ⑧ 准则/账簿 | 内地 3 家 CAS / 香港 3 家 HKFRS 双套：同一业务单类型不同准则可配不同 `account_code`；按公司 `region`（CN→CAS / HK→HKFRS）取模板集。本位币随公司（香港可 USD） |
| +拓 | **方向不写死**：模板行只标 `dr_cr_role`（DR/CR 增减语义），最终借贷由引擎 `f(account.balance_direction, dr_cr_role)` 推导（录音 L25/L64：余额方向是科目最关键属性，借贷增减由“科目余额方向+增减”推导；子科目须沿用父科目余额方向，不可冲突）。每条模板发布前须 `confirmed_by_id`（财务确认）才能被发动机引用，防误配自动入账 |

### 对象 2.3 — Voucher 凭证草稿（✅复用·B 交易单据·生命周期归总账，本文只覆盖“翻译产出”段）

| 范式列 | 内容 |

### 3.2 Voucher / VoucherEntry 字段字典（✅已有，需➕双金额/辅助核算/现金流量列）

**VoucherEntry（行，✅已有 `models.py:1287`）**：`id` / `voucher_id`(FK) / `line_number`(SmallInt,UNIQUE(voucher_id,line_number)) / `account_id`(FK→account) / `description`(摘要) / `debit`(Numeric16,2) / `credit`(Numeric16,2) / `currency`(String3,默认CNY) / `exchange_rate`(Numeric12,6,默认1)。

> ★➕扩展（录音强制，非可选）：
> - **双金额**：现表 `debit`/`credit` 只有一对金额 + `exchange_rate`。录音 L223 明确“借贷方要给两个金额：原币 + 本位币”，**不是只存原币靠汇率推算**。故需➕ `debit_orig`/`credit_orig`（原币）与 `debit`/`credit`（本位币，过账累加 AccountBalance 用本位币）。【v1错→v2正】v1 §3.2 只列单对金额 + exchange_rate，违反录音“两个金额都要给”，本 v2 补齐。
> - **辅助核算列**：➕ `aux_party_type`/`aux_party_id`（往来对象/客户/供应商）、`aux_dept_id`、`aux_project_id`、`aux_bank_account_id`（银行账户辅助核算）。供往来报表 / 多栏账 / 现金/银行日记账取数。
> - **现金流量项目列**：➕ `cash_flow_item_id`（FK→现金流量项目主数据），过账时落，现金流量表据此取数。
> - **往来 / 现金流量标识** 在科目主数据侧（录音 L43：科目要标识“现金流量项目”“哪些科目是往来类的科目”），不在凭证行；凭证行只落具体值。

> **字段防火墙**：成本/买价字段（6401 成本行、暂估估价、`InventoryTransaction.unit_cost`）对 `SALES`/`SALES_ASSISTANT` 隐藏；`is_auto_generated=True` 的分录行对所有角色只读（要改先反过账改源单）。**职责分离开关** `segregation_of_duties`（公司级，默认 ON）：校验 制单≠审核≠过账；小公司可关。

## ⑥ 校验规则 + 边界 + 错误处理（失败精确提示，绝不静默）

| 校验点 | hard_rule | 失败提示（精确） |
|---|---|---|
| 借贷平衡 | 过账前 Σ`debit`=Σ`credit`（本位币列） | 「凭证不平衡：借方合计 {x}，贷方合计 {y}，差额 {x-y}，禁止过账」 |
| 双金额完整 | 外币行 `debit_orig`/`credit_orig` 与本位币列**都须有值** | 「外币分录须同时给原币金额与本位币金额（{cur} 行第 {n} 行缺本位币/原币）」 |
| 期间锁（自然月） | `AccountingPeriod.status` 须 OPEN；**期间一律自然月**，无 period_cutoff_day | 「会计期间 {period} 已结账/已关账，不可过账/反过账；请用红冲或调整期凭证」 |
| 明细科目 | 每行 `account.is_leaf=True` | 「科目 {code} 为汇总科目，不能挂分录，请选下级明细」 |
| 反过账逐月 | 只能反当前最新已结账月 | 「反过账只能逐月返回：当前已结至 {m}，不能跳反 {target}」 |
| 资金类复核 | 资金类凭证过账前须经“复核”节点 | 「资金类凭证须先复核再过账」 |
| 职责分离 | 开关 ON 时 制单≠审核≠过账 | 「制单/审核/过账不可为同一人（职责分离已启用）」 |
| 模板未确认 | 引用模板须 `confirmed_by_id` 非空 | 「凭证模板 {source_doc_type}/{action} 未经财务确认，无法自动生成凭证」 |
| 订单类禁配 | `source_doc_type` ∉ {SALES_ORDER, PURCHASE_ORDER, QUOTATION…} | 「订单类单据不生成凭证，禁止为其配置模板」 |
| 现金流量缺项 | 资金类科目分录建议带 `cash_flow_item` | 「分录 {n} 为资金类科目但未指定现金流量项目，现金流量表将取不到此数（确认放行？）」 |
| 暂估幂等 | 进项发票冲回须找到 `provisional_flag=True` 原暂估凭证 | 「未找到暂估应付凭证或已冲回，跳过（幂等）」 |
| 业财同源原子 | 业务单 effect 与凭证草稿同事务 | 任一失败全回滚，CommandLog=FAILED，无脏数据 |
| 外币汇率 | 外币行须有当日 `exchange_rate`（取 ExchangeRate；资金类两种汇率可选） | 「币种 {cur} 缺 {date} 汇率，请先维护 ExchangeRate（月初入账汇率/结算汇率）」 |

边界：①跨期凭证→走调整期 `VOUCHER_ADJUSTMENT`；②暂估转实差异为负→1405 红字调减、移动加权回调；③同一业务单重复触发→`source_doc_type+source_doc_id` 幂等查重，已有凭证返回提示不重建；④红冲已被再红冲→检测原凭证已有红字配对，阻断重复红冲。
