# 03 · 币种 / 本位币 / 汇率

> 本模块 = 整套财务 ERP 的「计量基准层」。它不挂凭证、不挂业务单据，只提供三类主数据 + 取数口径，供所有交易模块、凭证内核、期末过程、报表层调用：
> ① **本位币**（公司级基准币，财报必以本位币出具）；
> ② **汇率维护表**（月度一汇率即可，理论一月两个：月初入账汇率 + 月末汇兑汇率）；
> ③ **汇率取数口径**（不同科目类别取不同汇率：固资按入账当天锁定、资金类两种汇率可选、月末货币性项目调汇）。
>
> 本模块只定义「主数据 + 取数口径」。**汇兑损益的 effect 实现、调汇凭证生成**属于「期末结账」模块（顺序：汇兑损益在前、结转损益在后），本文只给契约。
>
> 引擎现状已 grep 核实（2026-06-17，`backend/models.py`）：
> - `Company.currency`（本位币）**✅已有**（`String(3)`, 默认 USD）。
> - `ExchangeRate` 模型 **✅已有**，但 **❌缺 `company_id`**（当前全局表）、**❌缺 `rate_type`**（不区分入账/月末汇率）、**❌缺 `effective_period`**（按月维护口径未落地）。
> - `VoucherEntry` **✅有** `debit`/`credit`/`currency`/`exchange_rate`，但 **❌只有一对金额列** —— 录音明确要求「借贷方要给原币金额 + 本位币金额两个」，故需 **➕扩展原币列**（见 §4）。
>
> 标注：✅引擎已有 / ➕扩展现有 / ❌需新建。
>
> 录音对应段落：本位币 L169-172；汇率不必逐日 L175-178；固资/资金类取汇率 L178；月末汇兑损益 L189-199；汇兑损益在前结转损益在后 L207-208；理论一月两汇率 L231-232；凭证借贷原币+本位币双金额 L223-226。

## 2 · 本位币（Functional Currency · 公司级主数据）

### 2.1 对象范式表

| 列 | 内容 |
|----|------|
| ① 对象 | `Company.currency`（本位币，✅已有 `String(3)` 默认 USD）+ 币种枚举集（常量层） |
| ② 类型 | **A 主数据**（公司属性，不走状态机） |
| ③ 数据 | 见 §2.2 |
| ④ 动作×角色×时机 | **设本位币**：建账期（公司创建/首期初建账时）由 `FINANCE_DIRECTOR`+`BOSS` 设定；**启用后不可改**（强约束） |
| ⑤ 逆操作 | 本位币启用后**不可改**（无逆操作）。如必须变更=重新建账（迁数据），属重大动作需 `FINANCE_DIRECTOR`+`BOSS` 会签 + 审计留痕 |
| ⑥ 效果 | 所有外币业务按汇率折本位币入账；**财务报表（三大表）必以本位币出具**（L172）；账簿/科目余额表本位币列以此为基准 |
| ⑦ 回链 | 被 `VoucherEntry`（本位币金额）、`AccountBalance`（本位币余额）、报表层引用 |
| ⑧ 准则/账簿 | 内地 3 家 CAS → 本位币 CNY；香港 3 家 HKFRS → 本位币可 HKD 或 USD（公司级可配，MEMORY）。一企一本位币（L172） |

### 2.2 字段级数据字典

| 字段 | 类型 | 必填 | 默认 | 枚举/说明 | 约束 |
|------|------|------|------|-----------|------|
| Company.currency | String(3) ✅ | ✓ | "USD" | **本位币** ISO 三字码 {CNY, HKD, USD, EUR, JPY, …} | 启用后不可改（应用层强约束，非 DB） |
| Company.region | String(10) ✅ | ✓ | "HK" | HK / CN（决定默认本位币与准则） | 内地→建议 CNY，HK→可 HKD/USD |
| （币种枚举集） | 常量层 | — | — | 系统支持币种白名单 = {CNY, HKD, USD, EUR, JPY, GBP, …}（运维可扩） | 非表，常量 |

> ➕扩展（建议，非阻断）：`Company.base_currency_locked Boolean`（本位币是否已锁定，建账完成后置 True 触发「不可改」校验）。若不加列，可用「公司已有任一已过账凭证 ⇒ 视为锁定」推导。**封板时与财务确认是否加此列**。

### 2.3 本位币规则（录音口径）
- **一企一本位币**（L172「企业只能选择一种币别作为本位币……这就是个基准」）。
- 内地公司本位币 = CNY（CAS）；香港公司本位币 = HKD 或 USD（HKFRS，公司级可配，MEMORY）。
- 外币业务（非本位币）→ 按汇率折算成本位币入账（L172「要用汇率去折算成你的本位币去入账」）。
- 财务报告**必须以本位币出具**（L172「你的财务报告都必须要求本位币出具」）。
- **不可按公司随意切换**本位币（启用后锁定）；6 公司各自独立设本位币、互不影响（账套隔离）。

| 列 | 内容 |
|----|------|

| ③ 数据 | 见 §3.2 |

| +拓 | **月度一汇率即财务可接受**（L178）；理论一月只需两个汇率：月初入账 + 月末汇兑（L231-232）；逐日实时为可选（仅对接官方源时） |

### 3.2 字段级数据字典

| 字段 | 类型 | 必填 | 默认 | 枚举/说明 | 约束 |
|------|------|------|------|-----------|------|
| id | Integer PK ✅ | — | — | | |
| from_currency | String(3) ✅ | ✓ | — | 原币（外币）ISO 码 | indexed（联合唯一） |
| to_currency | String(3) ✅ | ✓ | — | 目标币（一般 = 本位币） | 联合唯一 |

| **rate_type** | String(12) | ✓ | "ENTRY" | ➕**新建**：`ENTRY`(入账/月初) / `CLOSING`(月末汇兑) / `SETTLEMENT`(结算时，资金类可选) / `SPOT`(逐日实时,可选) | |
| **effective_period_id** | Integer FK→accounting_period | — | NULL | ➕**建议**：直接挂会计期间（月度维护语义更清晰）；可由 effective_date 推导，二选一 | |

> ➕扩展（N6）：现表 `UNIQUE(from_currency, to_currency, effective_date)` → 改为 `UNIQUE(company_id, from_currency, to_currency, rate_type, effective_date)`，支持「同公司同货币对同月可分别维护入账汇率与月末汇率」。**封板时与财务确认 `rate_type` 枚举集是否够用**。

### 3.3 月度维护口径（录音核心，N2）
- **主口径 = 月度维护**：每月维护一个（入账）汇率即可，该月内发生的外币业务统一取这个汇率入账（L178「你每个月维护一个汇率，那你只要在这个一月内发生的数据，我就取这一个汇率，财务上也是可以接受的」）。
- **理论一月两个汇率**（L231-232）：
  - `ENTRY`（入账汇率）：月初 1 号，月内所有外币入账业务取它折本位币；
  - `CLOSING`（月末汇兑汇率）：月末做汇兑损益结转时取它重估货币性项目。
- **逐日实时为可选高级项**：仅当企业有进出口且能对接官方汇率源（如外管局/银行 API）时启用 `SPOT`（L178「能跟官方网站拉通实时获取也可以；但老 ERP 一般做不到，准则也不强制」）。默认关闭。
- **资金类可加 `SETTLEMENT`（结算时汇率）**：见 §5.1，资金类科目两种汇率可选（入账时 / 结算时），由业务 + 财务两方选（L178）。

### 3.4 汇率维护界面（行业惯例）
- 列表：公司 / 货币对（from→to）/ 汇率类型（入账/月末/结算）/ 生效月（或日）/ 汇率值 / 维护人 / 维护时间。
- 录入：选公司 → 选货币对 → 选类型（默认入账）→ 选生效月 → 填汇率。月末关账前提醒补录 `CLOSING` 汇率（缺失则调汇无法跑，硬阻断，见 §7）。
- 缺省提示：某月发生外币业务但该月该货币对无 `ENTRY` 汇率 → 凭证折算时报错「{月} {货币对} 入账汇率未维护，请先维护汇率」（绝不静默取 1，见 §7）。

## 4 · 凭证行的原币 + 本位币双金额（★录音 N1）

> 录音 L223-226 明确：「借贷方你要给原币的金额，也要给本位币的金额」「借贷要给两个金额」。v1 凭证行只存一对 `debit/credit` + `exchange_rate`、靠汇率推算本位币 —— **错**。本位币金额必须独立落库，不靠事后推算（避免精度漂移、便于报表直接汇总本位币）。

### 4.1 VoucherEntry 字段扩展（➕）

| 字段 | 类型 | 必填 | 默认 | 说明 | 状态 |
|------|------|------|------|------|------|
| debit | Numeric(16,2) | — | 0 | **本位币借方金额**（报表/科目余额按此汇总） | ✅已有（语义=本位币） |
| credit | Numeric(16,2) | — | 0 | **本位币贷方金额** | ✅已有（语义=本位币） |
| currency | String(3) | — | "CNY" | 原币种 | ✅已有 |
| exchange_rate | Numeric(12,6) | — | 1 | 折算汇率（原币→本位币，留痕用） | ✅已有 |
| **original_debit** | Numeric(16,2) | — | 0 | ➕**新建：原币借方金额** | ❌需新建 |
| **original_credit** | Numeric(16,2) | — | 0 | ➕**新建：原币贷方金额** | ❌需新建 |

> 关系：`debit = original_debit × exchange_rate`（本位币 = 原币 × 汇率），但**两者都落库**（不是只存原币 + 汇率）。本位币业务时 `original_* = debit/credit`、`currency = 本位币`、`exchange_rate = 1`。
>
> ➕封板确认：是否复用现有 `debit/credit` 为本位币列 + 新增 `original_debit/original_credit` 为原币列（推荐，下游报表零改动）；或反过来。**与财务确认列语义命名**。本 PRD 默认：`debit/credit = 本位币`、`original_debit/original_credit = 原币`。

### 4.2 双金额录入/校验规则
- 录入外币凭证行：选币种 → 系统自动带出该月 `ENTRY` 汇率 → 填原币金额 → 自动算本位币金额（可人工微调，留痕汇率）。
- **借贷平衡校验以本位币列为准**（Σ本位币借 = Σ本位币贷，整单平），原币列不要求跨币种平衡（多币种凭证只在本位币层平衡）。
- 单行只能借或贷其一非零（原币与本位币同步，不能一边借一边贷）。

## 5 · 汇率取数口径（被引用方契约 · ★录音 N5）

### 5.1 按科目类别的汇率取数口径表（L178 核心）

| 科目类别 | 取哪个汇率 | 是否随期重估（调汇） | 录音依据 |
|----------|-----------|---------------------|----------|
| **固定资产**（外币） | **发生当天汇率**，入账后**锁定不再变动** | **否**（不参与月末调汇） | L178「你像固定资产这种，如果涉及到外币的，它就是按照你发生当天的汇率去取，后面就不会再变动了」 |
| **资金类**（货币资金/银行存款/库存现金等） | **两种可选**：①入账时汇率 / ②结算时汇率（`SETTLEMENT`），由业务 + 财务两方选 | **是**（货币性项目月末按 `CLOSING` 重估，属通则推导，录音未明示资金类重估口径，待财务确认） | L178「资金类的有两种处理方式，一种就是你入账的时候用的汇率，另外一种就是你结算的时候用的汇率……取决于业务和财务两方，它可以选择」 |
| **往来类**（应收/应付/预收/预付/其他应收应付） | 入账时 `ENTRY` 汇率 | **是**（货币性项目，月末调汇） | 货币性项目通则；MEMORY 往来含税双方一致 |
| **收入/费用**（损益类）、**成本**（成本类，独立大类） | 业务发生月 `ENTRY` 汇率 | **否**（已实现，不重估，只在结转损益处理） | 通则；五大类口径见科目模块（成本类是独立大类，非损益类） |
| **非货币性其他**（库存商品等） | 入账时汇率，锁定 | 否 | 通则 |

> 系统实现：每个外币科目（或科目类别）配「汇率取数策略」（入账锁定 / 入账+月末重估 / 结算时），默认按上表。资金类的「入账 vs 结算」二选一项暴露给财务配置（L178「取决于业务和财务两方，它可以选择」）。

### 5.2 月末汇兑损益取数口径（调汇 · L189-199）

> 汇兑损益的 **effect 实现归「期末结账」模块**；本模块只定义取数公式与触发口径。

- **触发**：月末关账前，对所有「需重估的货币性项目」科目跑调汇（资金类 + 往来类，固资/损益不参与）。
- **公式**（L199）：
  ```
  调汇差额 = Σ(该科目原币余额 × 月末 CLOSING 汇率) − 该科目当前本位币余额
  ```
  即「一开始入账那个汇率折出的本位币余额」与「月末汇率重估的本位币余额」之差（L193）。
- **差额去向**：记入损益（L193「如果有变动的话，就记到他的损益里面去」）。差额为正/负分别对应汇兑收益/损失。**具体记入的科目编码（如 CAS 财务费用-汇兑损益 / HKFRS FX gain/loss）录音未指定，待财务确认**。
- **生成调汇凭证**：系统自动生成一笔调汇凭证（货币性项目科目 ↔ 汇兑损益科目），多为系统自动（L208「汇兑损益和结转损益大多数系统都做到自动结转，有固定逻辑」）。

### 5.3 顺序铁律（L207-208）★
```
月末过程顺序：  …各模块关账 → 生成凭证 → 过账 → ① 汇兑损益（调汇）→ ② 结转损益 → 各模块结账 → 总账结账
                                                      └─ 汇兑损益在前 ──┘ └─ 结转损益在后 ─┘
```
- **汇兑损益（调汇）必在结转损益之前**（L207「汇兑损益在前，结转损益在后，它们有顺序要求」）。
- 「过账直接调汇」= 结转汇兑损益（L265 专家确认），不是另一个动作。
- 调汇是月末货币性项目重估；结转损益是把收入费用结转本年利润（两个独立动作，前者属本模块取数口径、后者属期末结账）。

## 6 · 逆操作（录音口径 · N7 无蓝冲）

> 录音 L318-319：资深会计「不懂蓝冲，只听过红冲」。本模块不使用「蓝冲」提法。涉及汇率/本位币/调汇的逆操作只有两类：

| 场景 | 方法 | 说明 | 留痕 |
|------|------|------|------|
| 汇率录错（当月、未结账） | **直接改/删汇率记录**；相关凭证当月未结账→反过账后改原币/汇率重过账 | L303-304「当月填错=直接删/改，没结账凭证都能删」 | 改汇率审计留痕 |
| 调汇凭证错（未结账） | **反过账**（POSTED→AUDITED，状态逆向校验，effect 冲回余额）→ 改 → 重过账 | L319「反过账只是状态校验，逆向操作」；**只能逐月返**（L319「7 月结账只能反 7 月，不能反 6/5/4 月」） | 谁/何时反过账留痕 |
| 调汇凭证错（已结账/跨月） | **红冲**（原凭证不删，生成反向单据冲销）+ 重做正确调汇 | L325「红冲=认可原业务但金额要更新；当月红冲一正一负没有任何影响，一般次月冲；系统逻辑不区分是否跨月」 | 原单 + 反向单配对，净额=0（L325「当月一正一负」） |
| 本位币设错（已建账） | 不可逆改 → 重新建账（迁数据），`FINANCE_DIRECTOR`+`BOSS` 会签 | 强约束 | 重大动作留痕 |

> 注：汇率主数据本身不走状态机、不需「红冲」；红冲只针对**调汇凭证**（已过账后跨月发现错误时）。

## 7 · 校验 / 边界 / 错误处理（绝不静默）

| 规则 | 触发点 | 失败提示 |
|------|--------|----------|
| 本位币已锁定不可改 | 设本位币（建账后） | 「本位币 {cur} 已锁定（公司已建账/已有过账凭证），不可变更；如需变更请重新建账」 |
| 入账汇率缺失 | 外币凭证折本位币 | 「{月} {from}→{to} 入账汇率未维护，请先维护该月汇率」（**禁止静默取 1**） |
| 月末汇率缺失 | 月末调汇 | 「{月} {from}→{to} 月末(CLOSING)汇率未维护，无法做汇兑损益，请补录」（硬阻断，调汇不可跑） |
| 汇率 >0 | 维护汇率 | 「汇率必须大于 0」 |
| 双金额一致性 | 凭证行保存 | 「本位币金额 ≠ 原币 × 汇率（差 {d}），请核对」（允许人工微调但需确认） |
| 借贷平衡（本位币） | 提交审核/过账 | 「借贷不平（本位币）：借 {x} ≠ 贷 {y}，差额 {d}」 |
| 固资汇率不可重估 | 月末调汇 | 固资外币科目跳过调汇（按 §5.1 入账锁定），不参与重估 |
| 调汇顺序 | 结转损益前 | 「请先完成汇兑损益（调汇）再做结转损益」（顺序铁律 L207） |
| 反过账逐月 | 反过账调汇凭证 | 「只能反最近已结期({月})；{更早月}不可反过账」（L319） |

## 10 · 验收用例（Given-When-Then）

- **U1 本位币锁定**：G:内地公司已建账、本位币=CNY / W:改本位币为 USD / T:拒绝，提示「已锁定，需重新建账」。
- **U2 月度入账汇率**：G:6 月维护 USD→CNY ENTRY=7.20 / W:6 月任一笔 USD 外币凭证入账 / T:统一取 7.20 折本位币，原币 100 USD → 本位币 720 CNY，两金额都落库。
- **U3 入账汇率缺失硬阻断**：G:7 月无 USD→CNY ENTRY 汇率 / W:录 7 月 USD 凭证 / T:报错「7 月 USD→CNY 入账汇率未维护」，**不静默取 1**。
- **U4 双金额校验**：G:外币行原币 100、汇率 7.20、本位币手填 730 / W:保存 / T:提示「本位币 ≠ 原币×汇率（差 10）」需确认。
- **U5 借贷平衡按本位币**：G:多币种凭证（USD 行 + HKD 行），本位币层 Σ借=Σ贷、原币层不平 / W:提交审核 / T:通过（只校验本位币平衡）。
- **U6 固资不调汇**：G:外币固资科目入账锁定 7.20 / W:月末调汇（CLOSING=7.05）/ T:固资科目跳过、本位币余额不变；资金类/往来按 7.05 重估。
- **U7 月末汇兑损益**：G:USD 银行存款原币余额 1000、入账本位币 7200、月末 CLOSING=7.05 / W:跑调汇 / T:差额 = 1000×7.05 − 7200 = −150，记汇兑损失（具体科目编码待财务确认），生成调汇凭证。
- **U8 顺序铁律**：G:本期未做汇兑损益 / W:直接点结转损益 / T:拒绝「请先完成汇兑损益（调汇）」。
- **U9 月末汇率缺失**：G:月末无 CLOSING 汇率 / W:跑调汇 / T:硬阻断「月末汇率未维护，无法汇兑损益」。
- **U10 调汇凭证跨月红冲**：G:6 月调汇凭证已结账、发现汇率录错 / W:发起冲销 / T:系统走红冲（原凭证不删、生成反向凭证）+ 重做；当月红冲一正一负净 0。
- **U11 反过账逐月**：G:7 月调汇凭证已过账、6 月已结账 / W:反过账 6 月调汇 / T:拒绝「只能反 7 月」（L319）。
