## ① 模块定位

把"销售出库 → 应收债权成立（暂估应收/确认应收）→ 收入确认（满足准则条件）→ 收款核销 → 账龄/坏账"做成**业务单据闭环**。本模块产出的是**业务单据**（应收单 / 收款单 / 结算单），不直接写凭证；单据经业财翻译规则生成**凭证草稿**，统一流到「总账」审核、过账、写余额。

铁律（录音）：
- **先有应收才有凭证**，应收来源=**销售出库**（L346）。
- 应收含**两部分：暂估应收（未开票，按订单金额暂估）+ 确认应收（开票/确认后）**（L352）。
- **收入确认归本模块**，按会计准则满足确认条件即确认，**与税务解耦**（L405-406）。
- 收款单与应收应付**同源**，只是侧重不同：出纳关注付到哪个账户/哪个银行/什么方式+银企直连+日记账；应收应付关注往来单位/业务实质/摘要（L361-364）。→ **真实收付动作的台账归「出纳与资金」模块**，本模块通过结算单引用收款单完成**核销**。

边界划分：
- **预收款（AdvanceReceipt）的"收取"**在「出纳与资金」；本模块只处理**预收冲抵应收**这一核销动作。
- **真实收款流水（BankReceipt）录入**在「出纳与资金」；本模块的"收款单"在概念上=对该流水的应收侧引用，落地用结算单（ARSettlement）把收款源勾到应收单。
- **销项税额拆分 / 真票对接税局**在「税务」模块；本模块只在收入确认时按行级税率把含税额拆出收入与销项（解耦，见 §④）。
- **凭证审核/复核/过账/反过账/反审核/红冲过账**在「总账」；本模块只触发"生成凭证草稿"。

### 对象 2-4：暂估应收（动作/标记，落在应收单上）— ➕扩展，★资金与税分开

> 录音 L286/L352：发票来之前财务必须入账→按**订单金额**暂估；发票到→按发票金额入账+冲掉原暂估（**红冲**）。"资金跟税是两件事，分开讲"：资金侧=应收暂估（往来科目）；税侧=暂估销项（独立科目，归税务模块）。

| 列 | 内容 |
|---|---|
| **①对象** | 应收单 `is_provisional=True` 的暂估状态（`PROVISIONAL`）；不建独立 model（录音两种实现可二选，见下） |
| **②类型** | **B 交易单据标记** |
| **③数据** | 暂估金额=源销售出库/订单金额；is_provisional 标记 |
| **④动作×权限×时机** | 销售出库即建暂估应收（ACCOUNTANT/系统）；发票/确认到→按确认金额建确认应收 + **红冲原暂估** |
| **⑤逆操作** | 暂估转确认=红冲（原暂估单不删、生成红字反向 + 按确认金额建新单），net 配对 |
| **⑥效果** | 资金侧：暂估应收记往来科目（如 1122 下"暂估应收"明细或专设暂估往来科目，财务确认科目码）；税侧暂估销项独立科目归税务模块（解耦） |
| **⑦回链** | reversed_from_ar_id（红冲链）；source=SHIPMENT |
| **⑧准则/账簿** | 暂估往来科目（财务确认）；税侧不在本模块 |

> 录音两种 ERP 实现（可二选）：①专门"暂估应收单"承载；②销售出库单/应收单打"暂估"标记，确认到后再生成红冲单据。**v2 取②（标记法）**：复用现有 AccountsReceivable + is_provisional，最省 schema。

## ④ ★科目映射取数公式（业务单据→凭证草稿分录）

> 经"凭证模板/业财翻译规则"（运维可配，不动开发）配置；方向永远从 `account.balance_direction` 推（DEBIT/CREDIT），**禁 effect 写死借贷**（引擎已有 Account.balance_direction ✅）。
> ★录音 C5：每条分录**原币金额 + 本位币金额**两个都给（外币时 functional = 原币×当日汇率）。
> 业财翻译可配项（录音 L 配置段）：借贷方科目（可写死）+ 金额（灵活：取出库单成本/发票含税or不含税or税额）+ 摘要 + 辅助核算 + 日期（单据创建日 vs 业务发生日，可选）。**只对需生成凭证的单据做映射**（销售出库/应收确认/收款），订单类不进凭证。
> 所有草稿统一送**总账**审核过账（本模块不写 AccountBalance）。

### 4-1 确认收入（暂估转确认 / 直接确认，与税解耦）

| 触发 | dr/cr | 科目 | 原币金额 | 本位币金额 | 税处理 | 财务确认 |
|---|---|---|---|---|---|---|
| 满足准则确认收入 | DR | 1122 应收账款 | amount(含税) | amount×rate | INCLUSIVE，辅助核算=customer | ✓ |
| | CR | 6001 主营业务收入 | amount−tax | (amount−tax)×rate | EXCLUSIVE | ✓ |
| | CR | 222102 销项税额 | tax_amount | tax_amount×rate | TAX_ONLY，税率 LINE→MATERIAL→COMPANY；★税侧可不在本模块、归税务 | ✓ |
| 先收/先开票，未确认 | DR 1122 / CR 2203 预收账款 | 含税 | ×rate | 过渡，确认时再转 6001（贸易简化，合同资产/负债见 gap-R8）| ✓ |
| 暂估转确认（红冲原暂估）| 原暂估分录×(−1) NEGATE + 按确认金额重做 | 1122/相关 | 红字负数 + 新正数 | 同 | 原单不删、反向单冲（录音红冲口径）| ✓ |

### 4-2 收款核销（`finance.settle_ar` + `finance.draft_ar_voucher`）

|---|---|---|---|---|---|
| 银行收款 | DR | 1002 银行存款（辅助核算=银行账户维度，见下注）| Σ本次核销 settle | ×rate | ✓ |
| | CR | 1122 应收账款 | 同上（辅助核算=customer）| ×rate | ✓ |
| 现金收款 | DR 1001 库存现金 / CR 1122 | settle | ×rate | ✓ |
| 收票冲应收 | DR 1121 应收票据 / CR 1122 | note.amount | ×rate | ✓ |
| 预收冲抵应收 | DR 2203 预收账款 / CR 1122 | min(预收余额, AR未清额)| ×rate | ✓ |
| 外币收款汇兑差 | DR/CR 660301 财务费用-汇兑损益 | (收款日rate−入账rate)×外币额 | 差额本位币 | ✓ |

> ★录音"银行账户辅助核算"：银行存款名下 10 个账户**不建 10 个科目**，用"辅助核算维度"挂账户（L 科目段）。→ 1002 借方挂**银行账户辅助核算维度**，不是 10 个子科目。本模块凭证草稿借方需带该辅助核算（取自收款单 bank_account）。
> ★录音"汇兑损益在前、结转损益在后"：月末调汇（汇兑损益）→ 再结转损益。本模块只产出实现汇兑差（收款时点），月末未结外币 AR 余额的调汇在「期末/调汇」按本对象 functional vs 原币×期末汇率取数（不在本模块定义）。

## ⑥ 校验 / 边界 / 错误处理（失败精确提示，绝不静默）

| # | 规则 | hard_rule 条件 | 失败提示 |
|---|---|---|---|
| V1 | 借贷平衡（双金额）| 凭证草稿 Σ借=Σ贷，**原币与本位币各自平** | 「凭证不平衡：借 {d} ≠ 贷 {c}（{币种}），差额 {diff}」|
| V2 | 期间锁（自然月）| 目标自然月 AccountingPeriod.status=OPEN | 「{period} 已结账，禁止生成/过账；如需更正请用红冲/调整凭证」|
| V3 | 删改边界 | **未结账**可删/改单据与凭证草稿（非未过账，C3）| 「该期已结账，不可删改，请走红冲」|
| V4 | 核销不超额 | settle ≤ AR未清额(amount−paid) 且 ≤ 收款源可用余额 | 「核销额 {x} 超出应收未清 {y}」/「超出收款单可用余额 {z}」|
| V5 | 一收多销勾对平 | 全额结算时 Σ(本批 settle)=本次收款额 | 「本批核销合计 {s} ≠ 收款额 {r}，请勾平」|
| V6 | 必输 | amount>0、due_date、currency、exchange_rate>0、functional_amount=amount×rate | 「{字段}必填且 {约束}」|
| V7 | 发票号唯一 | (company_id, invoice_number) 唯一 | 「发票号 {n} 在本公司已存在」|
| V8 | 暂估转确认配对 | 红冲原暂估 + 按确认金额重做，net 应收方向一致 | 「暂估未冲清，请先红冲原暂估单 {no}」|
| V9 | 红冲不删原单 | 红冲=生成反向单，**原单保留** | （引擎校验：禁删已红冲原单）|
| V10 | 坏账前置 | 转坏账需 FINANCE_MANAGER 核定+BOSS 审批，outstanding>0 | 「无未清余额无需转坏账」/「待 BOSS 审批」|
| V11 | 外币双金额 | 本位币入账=amount×当日 rate；缺汇率拒 | 「缺当日汇率，请先在 ExchangeRate 维护 {from}/{to}/{date}」|
| V12 | 收入与税解耦 | 满足准则即确认收入，不强制同时税务确认 | （提示性：收入已确认，税务确认请在税务模块单独处理）|
| V13 | 含税往来一致 | 往来款一定含税且双方一致（集团内核对/合并前提）| 「内部往来口径不一致（一家含税一家不含税），无法核对」|

**边界**：
- 跨期收款（收款日属下自然月）→ 凭证记收款日所属 OPEN 自然月。
- 外币 AR 未清在期末由「调汇」按本对象 functional vs 原币×期末汇率取数（**汇兑损益在前、结转损益在后**，录音顺序）。
- 预收冲抵后预收余额不足→部分冲抵，余额仍挂 1122。
- 内部单位往来（集团内 A-B）：A 对你应收、你对 A 应付可勾对，一笔业务集团内只需一家做、另一家复核（往来模块支撑，本模块提供应收侧数据；强支撑合并报表）。

| ACCOUNTS_RECEIVABLE | CONFIRMED/COLLECTING→VOUCHER_DRAFTED | `finance.draft_ar_voucher` | ➕改现 seed hook（现仅建 DRAFT 凭证，硬写 1002/1122 单金额）| 改为：双金额(原币+本位币)+辅助核算(customer/银行账户)+送总账，**不在本模块过账** |
| ACCOUNTS_RECEIVABLE | 核销 | `finance.settle_ar` | ❌新建 | 回写 paid_amount/status，引用收款源（BankReceipt/Note/AdvanceReceipt）|
| ACCOUNTS_RECEIVABLE | →BAD_DEBT | `finance.accrue_bad_debt` | ❌新建 | 账龄取数计提/转销，草稿送总账 |

| **凭证草稿** | **送总账** | **总账模块** `finance.post_voucher`/`unpost_voucher`/`red_reversal` | **❌总账新建（不在本模块）**| ★录音 C1/C8：过账层写 AccountBalance、反过账逐月、红冲过账全在总账 |

> 调用范式：业务 effect → 业财翻译规则取分录（含双金额）→ 同事务建 Voucher **草稿**（业财同源）→ **送总账**（本模块到此为止；过账由总账触发 `finance.post_voucher`）。

## ⑧ UI（对齐行业惯例，不创新）

### 屏 1 · 往来核销工作台（高频作业=密集网格键盘流）
对齐金蝶/用友"收款核销"双栏：
```
┌─ 客户:[搜▼]  币种:[USD▼]  收款单/票据:[BR-2026-018 $50,000 可用$50,000] ──────┐
│ 左：未核销应收（勾选）          │ 右：本次核销分配                            │
│ ☑ 发票号    到期日   未清额     │ 发票号    分配额(可改)   核销后余额        │
│ ☑ SI-1001  06-10   $30,000    │ SI-1001  [$30,000]      $0                 │
│ ☑ SI-1002  06-20   $25,000    │ SI-1002  [$20,000]      $5,000(部分)       │
│  [自动按到期日FIFO分配]        │ 已分配 $50,000 / 收款 $50,000  ✅平(绿)     │
│                                │ 差额 $0           [生成核销]                │
└──────────────────────────────────────────────────────────────────────────┘
```
关键交互：①左未核销+右收款勾对；②勾选带入分配额可手改（部分核销）；③[自动 FIFO];④实时合计+差额（平绿不平红，不平禁生成）；⑤一收多销/一销多收；⑥生成核销→建收款凭证**草稿（送总账）**；⑦下钻发票/原凭证。

### 屏 2 · 收款凭证录入（复用总账凭证录入密集网格）★双金额两列
摘要｜科目(可搜+辅助核算弹层 customer/银行账户)｜**借方原币｜借方本位币｜贷方原币｜贷方本位币**；Tab 横走/Enter 下行/`=`自动配平；实时借贷合计+差额（原币与本位币各一行，平绿不平红）；金额大写；上/下张翻阅；插删行。
> ★凭证审核/过账按钮**不在本屏**——本模块只到"保存草稿+送总账"；审核/过账在总账凭证处理界面。

### 屏 3 · 收入确认（中频，列表+表单，非五步法向导）
左待确认队列（暂估应收，列：客户｜来源出库单｜暂估金额｜控制权状态｜差额）；右确认明细（确认金额不含税｜税额｜确认日｜控制权转移点下拉 发货/签收）+ 凭证草稿分录预览（双金额）。贸易单多为单一履约义务，不强制五步法分摊（复杂场景 gap-R9）。

### 屏 4 · 账龄分析（期间筛选→分桶→下钻）
基准日 + 客户/币种筛选；五桶热力色阶；外币按本位币列示；下钻 AR→总账凭证；右上 [生成坏账计提] 按桶比例预填（向导，低频不可逆）。

### 复用通用壳

- 应收明细/汇总/对账单/到期债权：派生只读→复用 DataExplorer + reports 壳。
- 坏账计提：向导式（取账龄→选比例→预览分录→FINANCE_MANAGER 核定→**草稿送总账**）。

## ⑩ 验收用例（Given-When-Then，含边界）

- **AC1 销售出库建暂估应收**：Given 销售出库放行；When 触发；Then 生成 AR(is_provisional=True, status=PROVISIONAL, amount=出库金额, functional_amount=amount×rate)，**无凭证亦可挂台账**（先有应收）。
- **AC2 暂估转确认（红冲）**：Given 暂估 AR $100；When 发票到、确认 $100；Then 原暂估**不删**、生成红字反向 + 按确认金额建确认应收，确认收入凭证草稿 DR1122/CR6001/CR222102（双金额）送总账。
- **AC3 收入与税解耦（边界）**：Given 满足准则但税务条件未达；When 确认收入；Then 收入已确认（6001），税侧可暂不确认/单独处理，不报错（V12）。
- **AC4 全额收款核销**：Given AR $30,000；When 核销工作台全额勾对→生成核销→生成凭证草稿；Then paid_amount=30000、status=SETTLED，草稿 DR1002(辅助核算=银行账户)/CR1122(辅助核算=customer) 双金额，**送总账待过账**。
- **AC5 部分核销（边界）**：Given AR $25,000，收款 $20,000；When 分配 20000；Then status=COLLECTING、未清 5,000 仍在账龄；再收 5,000→SETTLED。
- **AC6 一收多销（边界）**：Given 一笔 $50,000 核 SI-1001(30000)+SI-1002(20000)；When 同 batch；Then 两 AR 各回写，Σ分配=收款，差额 0 才可生成。
- **AC7 外币收款汇兑差（双金额，边界）**：Given USD AR 入账 rate 7.1、收款日 7.2；When 收款；Then 本位币差额进 660301 汇兑损益，凭证两列（原币 USD/本位币 CNY）各自平。
- **AC8 收票冲应收+兑付**：Given 收商票 $40,000 冲 AR；When HELD→COLLECTED；Then DR1121/CR1122→DR1002/CR1121，草稿送总账。
- **AC9 当月填错直接删改（非红冲，C2/C3）**：Given 当月录错、未结账；When 删/改单据与凭证草稿；Then 成功（不走红冲）；若已结账→拒绝并提示走红冲。
- **AC10 红冲原单不删（C2）**：Given 已确认业务金额要更新（暂估转实）；When 红冲；Then 原单保留、生成反向单据，当月红冲一正一负 net 0。
- **AC11 业务红冲广播（决策13）**：Given 客户退货回链原 AR；When 退货确认；Then 自动红字反向单+凭证草稿(DRAFT) + 每个财务角色收到 Notification(RED_REVERSAL/WARN)，**原单不删**，待总账审核过账。
- **AC12 坏账计提（边界）**：Given 90+ 桶 $10,000、比例 50%、现 1241 余额 $3,000；When 期末计提；Then 增量 DR6701/CR1241 各 $2,000（差额），后续转销 DR1241/CR1122。
- **AC13 期间锁（自然月，C6）**：Given 该自然月已结账；When 试生成/过账；Then V2 拒绝，提示走红冲/调整凭证；**确认无 period_cutoff_day、期间边界=自然月**。
- **AC14 公司隔离**：Given 用户仅 A 公司权限；When 查 aging_analysis；Then 仅 A 公司 AR，绝不合账（合并是报表层手工汇总，不在本模块混账）。
- **AC15 内部往来含税一致（合并前提，边界）**：Given 集团内 A-B 往来；When 一家含税一家不含税；Then V13 拒绝，提示口径不一致无法核对/合并。
