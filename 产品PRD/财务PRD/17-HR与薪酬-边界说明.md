## ① 模块定位（一句话）

**HR / 薪酬本体不属于财务核心**——它是一个可独立成型的产品（员工档案 / 工资结构 / 福利 / 考勤 / 假期 / 个税计算）。财务核心 PRD 在此**只做边界说明**：HR 每月算完工资后，财务侧承接两笔凭证——**工资计提**（借费用 / 贷应付职工薪酬）与**发薪**（借应付职工薪酬 / 贷银行 + 代扣个税 + 代扣社保），让利润表的人力成本与资产负债表的「应付职工薪酬」有数。**财务侧不参与工资怎么算**。

引擎现状一句话（grep 实测）：`Employee` / `Payroll*` / `应付职工薪酬` 业务对象**全部 ❌ 不存在**（仅旧前端 `FlowEditor.jsx` 有占位文案）；财务侧的承接面 = 既有 `Voucher`/`VoucherEntry`/`AccountBalance`/`Account` + 待建的过账层。本文不要求建任何薪酬业务表，只定义**接缝**与**两张凭证模板**。

> ⚠️ 凭证分录口径来源声明：录音 L376/L382 只明确「薪酬归 HR / HR 可独立成产品 / 含考勤·假期·薪酬·员工数据·工资结构·福利·个税且比较复杂」与「费用报销单独成模块、不与薪酬合并」；**录音并未逐字给出「计提 / 发薪」两笔凭证的具体科目与分录**。下文 §④ 的「借费用 / 贷应付职工薪酬 → 借应付职工薪酬 / 贷银行 + 代扣」属 CAS/HKFRS 标准会计处理（与本任务边界口径一致），凡科目细分（社保子目 / 个税过渡 / 个人社保过渡）均已在 §⑪ 标注 **待财务确认**，不作为录音硬断言。

## ③ 接收契约：HR → 财务（财务侧唯一输入）

财务侧不关心工资怎么算，只需要 HR 每月交付一份**可入账的汇总结果**。约定接收口径如下（HR 产品负责产出，可以是接口、导入文件、或 HR 模块内单据）：

### 3.1 计提汇总（按「费用科目 × 部门」归口）—— 驱动计提凭证

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| company_id | Integer | Y | 行级隔离（一公司一套，**绝不合账**；MEMORY 6 公司行级隔离） |
| period（自然月 YYYY-MM） | String(7) | Y | ★会计期间**一律自然月**（录音 L76-103 最大纠正）；非「27 号切分」 |
| expense_account_code | String(20) | Y | 受益对象费用科目（销售费用/管理费用…按部门归口） |
| dept | String | Y | 部门**辅助核算**维度（写入凭证分录的辅助核算，不建独立费用子科目） |
| gross_amount | Numeric(16,2) | Y | 该归口的应发合计（基本+绩效+补贴，HR 已算好） |
| employer_si_hf | Numeric(16,2) | N | 该归口的**单位**社保公积金（进费用，**不进实发**） |
| currency / 本位币金额 | String(3)+Numeric | Y | ★凭证两个金额：**原币金额 + 本位币金额**（外币时两个都给，录音 L223）。海外公司本位币可 USD |

### 3.2 发薪汇总（公司级合计 + 代扣项）—— 驱动发薪凭证

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| total_gross | Numeric(16,2) | Y | 应发合计（=计提的贷方应付薪酬基数） |
| total_net | Numeric(16,2) | Y | 实发合计（真正打款金额） |
| total_personal_iit | Numeric(16,2) | Y | 代扣个人所得税合计（HR 已算） |
| total_personal_si_hf | Numeric(16,2) | Y | 代扣个人社保公积金合计 |
| total_other_deduction | Numeric(16,2) | N | 其他代扣 |
| bank_account（辅助核算维度） | 维度值 | Y | ★银行账户用**辅助核算维度**挂在「银行存款」科目下，**不建 10 个银行科目**（录音 L16-70 辅助核算典型例子） |

> 勾稽（HR 负责保证，财务侧仅校验合计）：`total_net = total_gross − total_personal_iit − total_personal_si_hf − total_other_deduction`。财务侧拿到的是**已勾稽的合计**，不逐人核算。

## ④ ★发薪两笔凭证：科目映射取数（财务侧唯一实质内容）

> 这两笔凭证**只挂在「总账（凭证）」模块**（MEMORY 模块拆分铁律：只有总账挂凭证）。方向永远由 `f(account.balance_direction, dr_cr_role)` 推导，**禁止写死借贷**。具体业财翻译表落到「凭证模板/业财一体化（配置）」专章，本文给口径。

### 4.1 工资计提（权责发生：把人力成本计入当期损益）

| 行 | 借/贷 | 科目 | 金额取数 | 辅助核算 | 说明 |
|---|---|---|---|---|---|
| 1..n | 借 DR | 费用科目（销售费用/管理费用…按归口） | Σ同归口（gross_amount + employer_si_hf） | 部门 dept | 受益对象费用 = 应发 + 单位社保公积金 |
| L | 贷 CR | 应付职工薪酬 | Σ(gross_amount) | — | 应付员工的应发部分 |
| L+1 | 贷 CR | 应付职工薪酬-社保福利（子目）/ 其他应付款 | Σ(employer_si_hf) | — | 单位社保公积金挂应付【gap-1 待财务定子目结构】 |

平衡：Σ借（应发+单位社保）= Σ贷（应付应发 + 应付单位社保）✓

### 4.2 发薪（收付实现：清应付职工薪酬 + 代扣过渡）

| 行 | 借/贷 | 科目 | 金额取数 | 辅助核算 | 说明 |
|---|---|---|---|---|---|
| 1 | 借 DR | 应付职工薪酬 | Σ(total_gross) | — | 冲减计提的应付 |
| 2 | 贷 CR | 银行存款 | Σ(total_net) | **银行账户**（辅助核算） | 实发打款 |
| 3 | 贷 CR | 应交税费-应交个人所得税 | Σ(total_personal_iit) | — | 代扣个税过渡【gap-2 明细科目码待定】 |
| 4 | 贷 CR | 其他应付款-代扣社保公积金 | Σ(total_personal_si_hf + total_other_deduction) | — | 代扣个人社保过渡【gap-3 科目待定】 |

平衡：借（应发）= 贷（实发 + 代扣个税 + 代扣社保个人 + 其他）✓

> ★外币（海外公司发薪外币）：每行同时给**原币金额 + 本位币金额**（本位币 = 原币 × 当月入账汇率），财报按本位币出具（录音 L172-232）。汇率不要求逐日，月维护一个即可。
> ★现金流量：发薪凭证的银行存款行须在凭证录入时**指定现金流量项目**（项目名「支付给职工以及为职工支付的现金」为 CAS 现金流量表标准项，录音未逐字给出，待财务确认），否则现金流量表取不到数（录音 L463：「现金流量表你也需要在会计科目那里做一定的设计，让他们去做凭证的时候可以指定现金流量，随着他们的指定你才能取到数据，不指定你也取不到数」）。

## ⑤ 凭证生命周期 = 归总账，不归 HR/薪酬

| 动作 | 归属 | 说明（录音 L88-160 过账/结账 · L298-325 红冲/反过账/无蓝冲） |
|---|---|---|
| 凭证新增→保存→审核→过账 | **总账（凭证）** | 发薪是资金类，过账前多一步「复核」（录音 L160：「如果是涉及到资金类的，还有一个复核的动作，复核完了之后，就是可以到过账的状态」）。过账在结账之前（L112） |
| 反过账 / 反审核 | **总账（凭证）** | 状态逆向校验，**只能逐月返**（录音 L319/L321：「反过账只能一个月一个月的返，7 月结完账只能返 7 月，返 6/4/5 月都不行」）；必审计留痕谁/何时点反结账与结账（录音 L136/L139） |
| 红冲 | **总账（凭证）** | 原单不删、生成反向单据（录音 L322/L325：「原来那个单据不删，生成一个新的反向的单据」「次月才会冲，当月冲则一正一负没影响」）。当月填错（业务错误）则**直接删/改凭证**（未结账可删），不走红冲（录音 L304） |
| ❌「蓝冲」 | —— | **不存在此术语**。录音 L319 专家原话「你说的蓝冲我不太懂是什么意思，我只听过红冲」；v1 的「蓝冲反过账」一律改为「反过账/反审核」 |
| 可删边界 | **总账** | 未结账即可删（不是未过账）；单据生成的凭证删后须重新生成否则不能结账；手工凭证未结账即可删（录音 L304：「只要你没结账，凭证都是可以删的」） |

> HR/薪酬模块**不持有任何凭证生命周期动作**。HR 交付汇总 → 财务在总账按模板生成凭证草稿 → 总账统一审核/复核/过账/反过账/红冲。

## ⑦ 校验 / 边界 / 错误处理（财务承接面）

| 规则 | 触发点 | 精确提示 |
|---|---|---|
| 借贷平衡 Σ借=Σ贷 | 过账前（全局不变量） | 「借方合计 {x} ≠ 贷方合计 {y}，差额 {d}，禁止过账」 |
| 会计期间=自然月 | 选 period | 「会计期间须为自然月（{YYYY-MM} 1 日~月末），不存在 27 号切分期间」（录音 L76 纠正） |
| 期间锁（关账/结账） | 过账/反过账 | 「会计期间 {period} 已结账，禁止过账；如需冲销请用红冲」；反过账「只能反 {当期}，不能反更早月」 |
| 汇总合计勾稽 | 接收 HR 汇总 | 「发薪实发 {net} ≠ 应发−代扣 {calc}，请 HR 侧核对后重交」（财务侧只校合计，不逐人） |
| 原币+本位币双金额 | 外币发薪 | 「外币分录须同时提供原币金额与本位币金额」（录音 L223，禁止只存原币靠汇率推算） |
| 现金流量项目必填 | 银行存款行过账 | 「银行/现金科目分录须指定现金流量项目，否则现金流量表取不到数」 |
| 公司隔离 | 全程 | `_company_filter(user)`；跨公司不可见/不可操作；**绝不合账** |
| 幂等 | 生成凭证 effect | 同 source_doc + period 已有凭证→no-op 返回既有，防重复建凭证 |

**边界**：①跨期发薪 → 按 period_id 选定**自然月**期间入账，已结账期用调整凭证 / 红冲；②一人多公司 → 各公司各自计提发薪，**绝不合账**（合并报表是报表层手工汇总，见报表章）；③海外公司本位币 USD → 直接入账无折算，员工垫付外币 → 原币×汇率折本位币。

## ⑩ 验收用例（仅财务承接面 · Given-When-Then）

1. **工资计提入账**：Given 某内地公司 8 月 HR 交付计提汇总（管理部门归口管理费用：应发 30000、单位社保 6000）→ When 财务在总账按计提模板生成凭证、主管审核过账 → Then 凭证「借 管理费用 36000（辅助核算=管理部）/ 贷 应付职工薪酬 30000 + 应付职工薪酬-社保 6000」，借贷平衡，`AccountBalance` 对应期间 `period_debit/period_credit` += 36000。
2. **发薪入账**：Given 上单已计提（应发 30000、代扣个税 1500、代扣个人社保 3000、实发 25500）→ When 财务按发薪模板生成凭证、出纳复核、主管过账 → Then 「借 应付职工薪酬 30000 / 贷 银行存款 25500（辅助核算=工行 XX 账户、现金流量项目=支付给职工的现金）+ 应交税费-个税 1500 + 其他应付款-代扣社保 3000」，平衡，应付职工薪酬余额归零。
3. **边界·自然月期间**：Given 用户尝试把计提期间设成「7-27 至 8-27」→ When 选期间 → Then 拦截「会计期间须为自然月，不存在 27 号切分」（27 号是关账时点，非期间边界）。
4. **边界·已结账冲正**：Given 8 月计提凭证已过账、8 月已结账 → When 需更正 → Then 反过账被拦截（已结账只能逐月返且本期已结账）→ 改走红冲：原凭证不删，生成红字反向凭证，发生额净额=0。当月若未结账填错 → 直接删/改凭证（不走红冲，不提「蓝冲」）。
5. **边界·外币发薪**：Given 香港公司本位币 USD、发薪 USD → Then 直接入账无折算；若员工垫付 HKD 代付 → 原币 HKD 金额 + 本位币 USD 金额（=原币×当月汇率）两个金额都存，财报按 USD 出具。
6. **边界·公司隔离**：Given 员工同属 A、B 两公司 → When 各公司发薪 → Then A、B 各自计提发薪入各自账套，跨公司不可见、绝不合账（合并报表层另行手工汇总）。
7. **边界·费用报销不在本文**：Given 用户在 HR/薪酬入口找「差旅报销」→ Then 引导至**费用管理（11）**；本承接面不处理报销。
8. **边界·薪资隐私**：Given 财务会计查工资 → Then 仅见「按费用科目×部门」归口合计，**看不到逐人薪资**（逐人明细隐私在 HR 产品侧）。
