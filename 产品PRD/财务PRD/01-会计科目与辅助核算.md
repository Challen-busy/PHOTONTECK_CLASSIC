## 1 · 模块定位

会计科目表是整套财务 ERP 的**第一块地基**。它定义「钱往哪个口袋记、余额朝哪个方向走、报表从哪些科目段取数」。本模块封板内容 = **科目主数据 + 辅助核算维度 + 现流/往来标识 + 双准则种子**。封板后下游所有交易模块（应收/应付/出纳/税务/存货/固资）的凭证映射、所有报表的取数公式，都引用本模块的科目集与辅助核算定义。

录音定调三条铁律：
1. **余额方向是最关键属性**，借贷增减全部由「余额方向 + 增减语义」推导，**禁止在任何 effect 里写死借/贷**。
2. **辅助核算替代冗余子科目**：维度（账户/客户/供应商/部门/项目）挂在科目上，而不是为每个值建一个科目。
3. **「科目类别」用处不大**：报表取数靠 **编码范围/取数公式 + 现流标识 + 往来标识 + 余额方向**，不靠那个右侧下拉。

### 2.2 对象二 · 辅助核算（A 主数据 · ❌全新建）★录音核心，v1 缺失

| 列 | 内容 |
|----|------|
| ① 对象 | `AuxiliaryDimension`（维度定义，❌新建）+ `AccountAuxLink`（科目挂维度，❌新建）；A 主数据 |
| ② 类型 | **A 主数据**。维度本身是配置；维度值来源 = 已有主数据表（Customer/Supplier/Project ✅）或新建轻量码表（账户/部门） |
| ③ 数据 | 见 §4 字段字典 |
| ④ 动作×角色×时机 | 建维度、给科目挂维度（**最多 2 个**）：`FINANCE_MANAGER`；建账期配置 + 日常微调 |
| ⑤ 逆操作 | 已被凭证引用的维度挂载不可删，只可停用；不冲销 |
| ⑥ 效果 | 科目挂了维度 → 凭证录该科目行时**强制录维度值**（否则提交审核被拦）；维度值 → 明细账/往来核对/合并报表的分组键 |
| ⑦ 回链 | `AccountAuxLink.account_id`→Account；`AuxiliaryDimension.value_source`→指向值表（CUSTOMER/SUPPLIER/PROJECT/BANK_ACCOUNT/DEPARTMENT）；`VoucherEntry.aux_dim1_value_id/aux_dim2_value_id`（➕） |

**维度值来源映射（value_source）：**

### 3.3 五大类 ↔ 实现枚举映射 + 余额方向速查（★E1 纠错）

录音 L25（专家纠正 L22 口误「资产/负债/收入/费用/损益」后的版本）：五大类 = **资产类 / 负债类 / 成本类 / 权益类 / 损益类**（收入与费用归损益类，成本类是独立大类）。引擎 6 枚举是实现细分，顶层报表按五大类聚合：

| 五大类（录音顶层口径） | 实现枚举 `account_type` | 典型 balance_direction | 增加记 | CAS 编码段首位 |
|----------------------|------------------------|----------------------|--------|---------------|
| **资产类** | ASSET | **DEBIT** 借 | 借 | 1 |
| **负债类** | LIABILITY | **CREDIT** 贷 | 贷 | 2 |
| **权益类** | EQUITY | **CREDIT** 贷 | 贷 | 4 |
| **成本类** | COGS（+ 制造类 EXPENSE） | **DEBIT** 借 | 借 | 5 |
| **损益类 - 收入** | REVENUE | **CREDIT** 贷 | 贷 | 6（60xx） |
| **损益类 - 费用** | EXPENSE | **DEBIT** 借 | 借 | 6（64xx/66xx/67xx） |

> 口诀：资产/成本/费用 = 借方增；负债/权益/收入 = 贷方增。**方向永远从 `balance_direction` 出发推导借贷**（科目映射框架据此自动推借贷增减，禁止在 effect 里写死借/贷）。
> ★成本类（5xx）与损益-费用类（64xx/66xx）在录音里是分开的：成本类是独立大类，费用归损益类。引擎用 COGS 表「成本类」、EXPENSE 表「损益-费用」，报表聚合时各归各的大类。

## 5 · 子科目规则（★E6 纠错 · 无限级 / 无代码 / 沿用父属性）

录音 L61-70 原话落地（无代码 L62/64、沿用父属性 L64、无限级 L69-70）：

| 规则 | 录音依据 | 系统实现 |
|------|---------|---------|
| **无限级** | 「对，无限，系统上一般不做限制」 | `parent_id` 自引用树，无层级上限；`level` 仅展示用 |
| **用户自建子科目没有代码** | 「它是没有代码的，对，相当于数据结构上的增加」 | 用户自建子科目 `code` 可空/系统自动派生「父码-序号」内部码（不展示给用户当业务码）；系统种子科目才有标准 CAS 码 |
| **沿用父属性、不可冲突** | 「子科目要沿用父科目的属性，不能跟父冲突。银行存款余额方向是借，下面也一定要借，不能设成贷」 | 建子科目时**强制继承** parent 的 `balance_direction`/`account_type`/`is_contra`/`is_current_account`/辅助核算维度挂载；UI 锁死这些字段不可改；保存前校验子≠父则拒绝 |
| 父变叶 | — | 一旦科目有子科目，自动 `is_leaf=False`（不可再挂分录，分录下沉到叶子） |

校验 hard_rule（建/改子科目）：
- `child.balance_direction == parent.balance_direction`（否则报「子科目余额方向必须与父科目一致（父={父}借/贷）」）
- `child.account_type == parent.account_type`、`child.is_contra == parent.is_contra`
- `child.aux_links ⊇ parent.aux_links`（维度沿用，可加不可减、不可改父维度）

## 6 · 报表取数口径（★E4/E5 纠错 · 编码范围 + 现流/往来/方向，不读「科目类别」）

录音 L52 给了两种取数法，本系统采用 ②为主 + ①为辅：

| 取数法 | 录音原话 | 本系统采用 | 用途 |
|--------|---------|-----------|------|
| ① 标准公式（改科目/期间/范围抽数） | 「有的系统有自己一套总账取数公式，标准化提炼后只改科目/期间/范围」 | 辅：三大表的固定行用「科目段 + 期间」公式模板 | 资产负债表/利润表固定行 |
| ② 科目编码 in 范围 | 「有些软件根据科目编码自由配置，编码 in 哪些范围去抽」 | **主**：报表行 = `account.code IN [范围]` + 余额方向取净额 | 灵活报表/明细账/科目余额表 |

**驱动报表取数的 4 个字段（录音明确「科目类别用处不大」，不在内）：**
1. **余额方向**（`balance_direction`）→ 决定取借方净额还是贷方净额；
2. **科目编码范围/取数公式**（`code IN [...]`）→ 决定哪些科目进哪一行；
3. **现金流量项目**（`cashflow_item_code`）→ ★现金流量表唯一取数依据（凭证录入时落到行上，没标就取不到）；
4. **往来标识**（`is_current_account`）→ 往来核对表/合并报表抵消的取数池。

> 取数公式示例（科目余额表/资产负债表「应收账款」行）：`SUM(AccountBalance.closing_debit - closing_credit WHERE account.code LIKE '1122%' AND period=X)`，方向按 `balance_direction=DEBIT` 取借方净额。明细账下钻再按 `VoucherEntry.aux_dim1_value_id`（客户）分组。

## 10 · 校验 / 边界 / 错误处理（绝不静默，Fail Loud）

| 规则 | 触发点 | 失败提示 |
|------|--------|----------|
| code 账套内唯一 | 新增系统级科目 | 「科目代码 {code} 在本账套已存在」 |
| 子科目方向=父方向 | 建/改子科目 | 「子科目余额方向必须与父科目一致（父 {code}={借/贷}）」 |
| 子科目属性沿用父 | 建子科目 | 「子科目的类型/备抵/往来标识不可与父科目冲突」 |
| 辅助核算 ≤2 维 | 给科目挂维度 | 「单科目最多挂 2 个辅助核算维度（录音硬上限）」 |
| 非叶不可挂分录 | 凭证提交审核 | 「科目 {code} 为非明细（汇总）科目，不能挂分录」 |
| 有维度必录值 | 凭证提交审核 | 「科目 {code} 已挂辅助核算 {维度名}，第 {n} 行必须录维度值」 |
| 往来科目必挂客户/供应商维度 | 标 is_current_account | 「往来类科目须挂 CUSTOMER 或 SUPPLIER 维度」 |
| 已用科目不可删 | 删除科目 | 「科目 {code} 已有余额/已被凭证引用，只能停用不能删除」 |
| 备抵方向自洽 | 保存 is_contra 科目 | 「备抵科目方向应与所属大类相反（如累计折旧应为贷方）」 |

边界：
- **银行账户绝不建独立科目**（★E2）：试图为单个银行户建 Account → 引导改为「在 1002 下新增 BANK_ACCOUNT 维度值」。
- **「科目类别」下拉不参与任何取数**（★E4）：报表配置界面不暴露该字段作为取数维度。

## 13 · 验收用例（Given-When-Then）

1. G: 用户想为新开的 HSBC 账户记账 / W: 试图新增「1002001 银行存款-HSBC」独立科目 / T: 系统引导改为「在 1002 的 BANK_ACCOUNT 维度新增账户值」，**不创建新科目**（★E2）。
2. G: 在「1002 银行存款」(借方)下建子科目 / W: 把子科目余额方向设为「贷」/ T: 拒绝，提示「子科目余额方向必须与父科目一致（父 1002=借）」（★E6）。
3. G: 给某科目挂第 3 个辅助核算维度 / W: 保存 / T: 拒绝，提示「单科目最多挂 2 个维度」。
4. G: 1122 应收账款已挂 CUSTOMER 维度 / W: 凭证录 1122 行不选客户即提交审核 / T: 拒绝，提示「第 n 行必须录维度值（客户）」。
5. G: 现金流量表取数 / W: 某收现凭证行所在科目未标 cashflow_item_code / T: 该笔在现金流量表取不到 → 系统在期末检查报「{科目} 有现金流量发生但未配现流项目」（★E8 不静默）。
6. G: 累计折旧 1602 / W: 保存时方向误设借 / T: 提示「备抵科目方向应与所属大类相反（累计折旧应为贷方）」（★E7）。
