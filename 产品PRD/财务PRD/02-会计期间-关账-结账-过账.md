# 02 · 会计期间 / 关账 / 结账 / 过账（四件事拆开）

> **本模块按录音事实库重构（录音=事实，优先级高于任何旧 PRD 猜测）。** 范围 = ①会计期间（**自然月**，准则全球统一、不可按公司切换）②关账（锁数据库 / 可先关业务账）③过账（**凭证**层动作）④结账（**各模块各有 + 总账最后**，锁财务录入）⑤逆操作（反结账 / 反过账逐月返）⑥结账周期弹性。
>
> **三个独立动作（录音 L88-160 反复强调）必须分清，绝不能混为一谈：**
> - **过账（Post）** = 单张**凭证**的动作（凭证生命周期最后一步：录入保存 → 审核 →〔资金类多一步**复核**〕→ **过账**）。过账在结账**之前**。
> - **关账（Lock）** = 把该期间数据库**锁定**（增 / 删都不可），27 号是关账时点；**可先关业务账**（只锁业务数据、财务仍可补凭证）。
> - **结账（Close）** = 不允许再录**财务**数据、结账到下一期；**每个模块各有自己的结账动作**（存货模块 / 应收应付模块 / …），人为点。
>
> **模块内顺序铁律**：`先关账（锁数据）→ 把该生成凭证的生成凭证 → 再结账`。各模块凭证全生成并过账 → 各模块结账 → 所有模块完 → **总账才能结账**。
>
> **AccountingPeriod 状态机（v2 核心修正）**：`OPEN —关账→ LOCKED —结账→ CLOSED`；反结账 `CLOSED → OPEN`。（v1 只有 OPEN/CLOSED 两态，**漏掉了"关账=锁数据"这一独立中间态**，本版补 LOCKED。）
>
> **引擎现状已 grep 核实（2026-06-17，`backend/models.py:1224/1236`、`scripts/seed.py:453-534`、`services/finance_commands.py`、`services/phase1_effects.py`）：**
> - ✅ `FiscalYear`(status OPEN/CLOSED)、`AccountingPeriod`(status / closed_by_id / closed_at)、`AccountBalance`(opening/period/closing 六栏)、`Voucher`/`VoucherEntry`、`ExchangeRate` 模型**已存在**。
> - ✅ VOUCHER 主流程状态机含 `POSTED`(过账)/`FX_ADJUSTED`(调汇)/`PL_TRANSFERRED`(结转损益)/`CLOSED`(结账) 四态 + 边**已 seed**；VOUCHER_ADJUSTMENT(调整期/第13期)状态机**已 seed**。
> - ❌ `AccountingPeriod.status` 当前只用 OPEN/CLOSED，**无 LOCKED（关账态）**；**无 `business_locked` 标记（先关业务账）**——需扩展（值域 / 标记列，见 §③）。
> - ❌ `finance.post_voucher`/`unpost_voucher`/`lock_period`/`unlock_period`/`close_module`/`close_period`/`open_period` effect/command **全部不存在**（grep `services/` 仅有 `finance.create_*` 业务→凭证草稿 effect、`finance_commands.py` 仅 AR/AP/信用类 command）。
> - ❌ 各模块结账动作、关账锁、结账前校验 validator、反结账逐月返校验 **未建**。
> - ❌ 财务角色 CASHIER/ACCOUNTANT/FINANCE_MANAGER 未加（依赖基石 01 模块）。
>
> 标注：✅引擎已有 / ➕扩展现有（改 seed / 加值域 / 加标记列） / ❌需新建（纯扩展件，`@register_*` 注册，核心 registry/workflow/commands 字节级零 diff）。**取数 / 校验硬软度 / 弹性窗口口径凡未定 = 标「财务确认 gap」。**

## ① 模块定位

**一句话**：把"期间—关账—过账—结账"四件独立的事按准则正确建模，让每个公司、每个业务模块能按"先锁数据 → 补齐凭证 → 逐模块结账 → 总账收口"的节奏推进，并提供逐月可逆的反结账 / 反过账与审计留痕。

**本模块是基石 01 之后、所有交易模块（销售 / 采购 / 出纳 / 存货 / 固资）之上的"时间轴控制器"**：它不产生业务数据，只控制"某一期间能不能录、能不能改、能不能过账、能不能进下一期"。期末调汇 / 结转损益 / 期初建账的**算法批处理**归 10-期末结账模块；本模块只定义**期间状态、关账锁、过账动作、结账编排、逆操作**这套时间轴骨架。

期末是 ERP 财务最不可逆、最高风险的批处理 → 结账 / 反结账走**向导式 + 留痕 + 会签**。

## ② 本模块对象清单（对象范式）

涉及 **4 个对象**：(A) 会计年度+会计期间〔主数据/状态源〕、(B) 凭证过账动作〔挂 VOUCHER 状态机的边+effect〕、(C) 关账锁〔期间状态翻转 OPEN→LOCKED + 可选先锁业务账〕、(D) 各模块结账 + 总账结账〔编排 + 期间状态翻转 LOCKED→CLOSED〕。

### 对象 B · 凭证过账（Post Voucher）

> **过账 = 凭证的动作，不是期间的动作（录音 L112）。** 凭证生命周期：`录入保存 → 审核 →〔资金类凭证多一步"复核"〕→ 过账`。过账是凭证最后一步，过账后凭证不可改（要改先反过账）。过账常**批量集中点**一次全选过账（录音 L160），紧接着就结账，过账与结账连着。

| 列 | 内容 |
|----|------|
| ① 对象 | `Voucher`/`VoucherEntry`（model ✅）；过账=挂 VOUCHER 状态机 `AUDITED →〔复核〕→ POSTED` 边 + effect。**B 交易凭证** |
| ② 类型 | **B 交易凭证单据**（凭证生命周期：审核 / 复核 / 过账 / 反过账 / 红冲**只属于总账模块**，见基石 01 与 §⑦ 接缝） |
| ③ 数据 | Voucher 头 + VoucherEntry 行；**借贷方每行给两个金额：原币金额 + 本位币金额**（外币时两个都给，不是只存原币+汇率推算，录音 L223）；见 §③ |
| ④ 动作×角色×时机 | **审核** `DRAFT→AUDITED`：`ACCOUNTANT`（与 §④ 状态机表一致；职责分离=制单/审核/过账分离，具体角色绑定待财务确认）。**复核（仅资金类凭证）** `AUDITED→REVIEWED`：`CASHIER`/`FINANCE_MANAGER`（资金类多这一步，录音 L160）。**过账** `〔AUDITED 或 REVIEWED〕→POSTED`：`FINANCE_MANAGER`。时机 = 该模块凭证全生成后、结账前，批量集中过账（录音 L160「批量点一下全部选中过账，紧接着点结账」） |
| ⑤ 逆操作 | **反过账** `POSTED→AUDITED`（期间 OPEN/LOCKED 可、CLOSED 须先反结账）；**反审核** `AUDITED→DRAFT`。两者 = **状态逆向校验，须填原因+留痕**。**反过账只能逐月返**（录音 L319 原话：「反过账的动作，你只能一个月一个月的返…7 月份结完账只能返 7 月份的，6/5/4 月都不行」）：7 月结账只能反 7 月，不能跳反 6/5 月。**★无"蓝冲"术语**（见 §⑤ 红框） |
| ⑥ 效果 | 过账 effect `finance.post_voucher`（❌新建）：把每行 debit/credit（**原币 + 本位币双金额**）累加进对应 `AccountBalance.period_debit/period_credit`，按 balance_direction 重算 closing。`finance.unpost_voucher` = 精确逆运算 |
| ⑦ 回链 | `Voucher.source_doc_type`+`source_doc_id`（回链业务单）；`posted_by_id`/`posted_at`；业务单据经凭证模板生成凭证草稿 → 流到总账过账 |
| ⑧ 准则 / 账簿 | `company_id` 账套；CAS/HKFRS 共用同一过账结构；过账写本位币（财报必本位币出具） |
| +拓 | 借贷平衡 hard_rule；过账不可改；期间锁（LOCKED 期是否可过账 = 关账粒度 gap，见 §⑥）；资金类复核为可配开关 |

> **★ v2 新增「复核」态**：录音 L160「凭证第一步是新增保存，是审核，**如果涉及到资金类的，还有一个复核的动作**，复核完了之后才到过账状态」。v1 状态机只有 DRAFT→AUDITED→POSTED，**漏了资金类复核**。v2 在 VOUCHER 状态机加可选 `REVIEWED` 态：非资金类凭证 `AUDITED→POSTED` 直达；资金类（voucher_type∈资金类 / 含 1001/1002/1012 现金银行科目）走 `AUDITED→REVIEWED→POSTED`。复核人=出纳/财务（财务确认开关）。

### 对象 C · 关账（Lock Period · 锁数据库）★ v2 全新独立对象

> **关账 = 把该期间数据库锁定（增 / 删都不可，录音 L97）。** 27 号是关账时点（录音 L88）。**可先关业务账**（只锁业务数据、财务仍可补凭证，录音 L127「先关业务账的话，那就是业务的数据关了」）。关账 ≠ 结账：关账只锁数据；关账到结账中间还要做"把该生成凭证的生成凭证"（录音 L139）。

| 列 | 内容 |
|----|------|
| ① 对象 | `AccountingPeriod`（model ✅）；状态翻转 `OPEN→LOCKED` + 可选 `business_locked` 标记（先关业务账）。**E 期末过程** |
| ② 类型 | **E 期末过程动作**（专用屏=期间管理 / 关账面板，人为点） |
| ③ 数据 | `AccountingPeriod.status='LOCKED'` + `locked_by_id`/`locked_at`（❌新增列）；可选 `business_locked Boolean`（先关业务账，❌新增列） |
| ④ 动作×角色×时机 | **关账（全锁）** `OPEN→LOCKED`：`FINANCE_MANAGER`+`BOSS`。**先关业务账** = 单独动作只置 `business_locked=True`（业务模块录入冻结、财务模块仍开）：`FINANCE_MANAGER` / 业务负责人。时机 = 27 号（或公司自定关账日，**这是人为动作不是期间边界**） |
| ⑤ 逆操作 | **反关账** `LOCKED→OPEN`（解锁数据库允许补 / 改）：`FINANCE_MANAGER`+`BOSS`，**审计留痕**（谁 / 何时点反关账，录音 L136）。先关业务账可单独解锁 `business_locked=False` |
| ⑥ 效果 | `finance.lock_period`（❌新建）：`status='LOCKED'`，hard_rule `period_writable` 拦截该期所有业务 / 财务单据的增删改（除非反关账）；**先关业务账**：只拦业务单据增删改，财务凭证仍可录 / 改 / 过账（为"把该生成凭证的生成凭证"留通道，录音 L139） |
| ⑦ 回链 | `AccountingPeriod.locked_by_id`/`locked_at`；WorkflowLog 留痕；hard_rule 拦截点广播全系统 |
| ⑧ 准则 / 账簿 | `company_id` 账套各自关账；关账时点公司可不同（27 / 25 / 月末，**人为制度，非准则强制**） |
| +拓（v2 核心） | **关账是结账的前置步骤（顺序铁律：关账 → 生成凭证 → 结账）。** 关账粒度（全锁 vs 先关业务账）= 录音明确支持二段式：先关业务账锁住业务录入，财务补完凭证后再全锁 + 结账 |

> **★ v2 新增对象（v1 完全缺失）**：v1 把"关账"和"结账"混为 OPEN→CLOSED 一步，**丢掉了录音反复强调的"关账=锁数据库"这个独立中间动作**。录音 L88/L97/L127/L139 四处独立证据：①27 号关账是锁录入窗口；②关账=锁数据库增删不可；③可先关业务账；④关账→生成凭证→结账是模块内顺序。故 v2 必须有 LOCKED 中间态。

### 对象 D · 各模块结账 + 总账结账（Close Module → Close GL）★ v2 重构为"逐模块"

> **结账 = 不允许再录财务数据、结账到下一期（录音 L127）。** **每个模块都应该有自己的结账动作**（存货模块 / 应收应付模块 / …，录音 L121）：点该模块结账 = 该模块本期业务处理完毕、该生成的凭证已生成 → 该模块可进入下一期。**所有模块该生成凭证的都生成完 → 总账才到可结账状态（录音 L139）。** 结账是人为点的（录音 L124）。

| 列 | 内容 |
|----|------|
| ① 对象 | `AccountingPeriod`（model ✅）+ ❌新增 `ModuleCloseStatus`（各模块×期间结账台账）；状态翻转：各模块结账 → 全模块结账 → 总账 `LOCKED→CLOSED`。**E 期末过程** |
| ② 类型 | **E 期末过程编排**（专用屏=结账看板：各模块结账状态行 + 总账结账按钮） |
| ③ 数据 | `ModuleCloseStatus`(❌新建)：company_id / period_id / module（INVENTORY/AR_AP/CASH/FIXED_ASSET/TAX/EXPENSE…）/ status(OPEN/CLOSED) / closed_by_id / closed_at；总账：`AccountingPeriod.status='CLOSED'` + closed_by_id/closed_at（✅已有列） |
| ④ 动作×角色×时机 | **各模块结账**（如"存货结账""应收应付结账"）：各模块 `FINANCE_MANAGER` / `ACCOUNTANT`，时机 = 本模块凭证全生成并过账后。**总账结账** `LOCKED→CLOSED`：`FINANCE_MANAGER`+`BOSS`（会签），时机 = **所有模块均已结账**后。**结账周期弹性**：贸易类一般次月 1~2 号完成（录音 L148） |
| ⑤ 逆操作 | **反结账** `CLOSED→OPEN`（录音 L133「点反结账→打开期间→可反过账 / 反审核 / 删除」；L139「记录谁点的反结账/结账、什么时候点」）：`FINANCE_MANAGER`+`BOSS`会签 + **审计强留痕**；**只能逐月返**（录音 L319 明文是对"反过账"——一个月一个月返；反结账逐月返为顺推：反结后才能反过账，故反结账亦应逐月、不可跳期，工程草案，**逐月返硬软度待财务确认**）。模块结账也可单模块反结 |
| ⑥ 效果 | **各模块结账** `finance.close_module`（❌新建）：校验本模块本期凭证全过账 → 写 ModuleCloseStatus.CLOSED；本模块禁再录财务数据。**总账结账** `finance.close_period`（❌新建）：校验全模块已结 + 试算平衡 + 损益已结转 → `status='CLOSED'`、写 closed_by/at、**本期 closing → 下期 opening**（资产 / 负债 / 权益延续，损益类已结转故 closing=0）。**反结账** `finance.open_period`（❌新建）：`status='OPEN'`、留痕、下期 opening 同步回退 |
| ⑦ 回链 | `ModuleCloseStatus`(模块×期间)；`AccountingPeriod.closed_by_id`/`closed_at`；`AccountBalance.period_id` 链下期；WorkflowLog 留痕反结账 |
| ⑧ 准则 / 账簿 | `company_id` 账套**各自独立结账**（绝不合账，6 公司行级隔离）；公司可不同结账进度；合并报表是**报表层手工汇总**，不影响各自结账 |
| +拓 | **期间锁全局源**：CLOSED 期任何 Voucher 过账被 hard_rule 拦；跨期改账走调整期（VOUCHER_ADJUSTMENT 第 13 期）或反结账 |

> **★ v1 错 → v2 正**：v1 §10 把结账写成 VOUCHER 状态机单条 `PL_TRANSFERRED→CLOSED` 一步到位的"期末结账"。**录音 L121「每个模块都应该有自己的结账动作」被 v1 漏掉。** v2 重构为两层：**各模块结账（存货 / 应收应付 / 出纳 / 固资…各一个动作）→ 总账结账**。各模块结账完才能总账结账（录音 L139 顺序铁律）。

## ④ 关键状态机表

### AccountingPeriod 状态机（v2 核心·三态）

| state | 名称 | 含义 | 允许的录入 | 进入动作 | 角色 |
|-------|------|------|-----------|----------|------|
| **OPEN** | 开放 | 本期可正常录入业务+财务、可过账 | 业务✓ 财务✓ 过账✓ | 建期间(initial) / 反关账 / 反结账 | — |
| **LOCKED** | 已关账 | 数据库锁定（增删不可）；先关业务账=仅锁业务、财务仍可补凭证 | 全锁:业务✗财务✗ ／ 先关业务账:业务✗财务✓过账✓ | **关账** OPEN→LOCKED | FINANCE_MANAGER+BOSS |
| **CLOSED** | 已结账 | 不允许再录财务数据；不可过账；closing→下期opening | 业务✗ 财务✗ 过账✗ | **总账结账** LOCKED→CLOSED | FINANCE_MANAGER+BOSS会签 |

**边（transition）：**

| label | from→to | roles | hard_rules | effects |
|-------|---------|-------|-----------|---------|
| 关账（全锁） | OPEN→LOCKED | FINANCE_MANAGER+BOSS | 须填确认 | ➕`finance.lock_period`（全锁） |
| 先关业务账 | OPEN(business_locked=F→T) | FINANCE_MANAGER | — | ➕`finance.lock_business`（仅业务） |
| 反关账 | LOCKED→OPEN | FINANCE_MANAGER+BOSS | 须填原因+留痕 | ➕`finance.unlock_period` |
| 总账结账 | LOCKED→CLOSED | FINANCE_MANAGER+BOSS会签 | **结账前校验清单全绿**（§⑥）+**全模块已结**+**前序期已CLOSED** | ➕`finance.close_period`（closing→下期opening） |
| 反结账 | CLOSED→OPEN | FINANCE_MANAGER+BOSS会签 | **只逆最新已结期（逐月返）**+须填原因+留痕+下期未占用 | ➕`finance.open_period` |

> 注：`OPEN→CLOSED` **不允许直达**（必须先 LOCKED）——这是 v2 相对 v1 的硬约束，强制"先关账锁数据 → 再结账"顺序。

### VOUCHER 状态机·过账相关边（✅已 seed，➕标补的边/态/effect）

| label | from→to | roles | hard_rules | effects |
|-------|---------|-------|-----------|---------|
| 提交审核 | DRAFT→AUDITED | ACCOUNTANT | 借贷平衡、金额>0单边、叶子科目、必输、**期间≠CLOSED** | — |
| 审核退回 | AUDITED→DRAFT | FINANCE_MANAGER | 填退回原因 | — |
| ➕**复核（仅资金类）** | AUDITED→REVIEWED | CASHIER/FINANCE_MANAGER | 资金类凭证（含现金/银行科目）必经此步 | — |
| 过账（非资金类） | AUDITED→POSTED | FINANCE_MANAGER | 借贷平衡、**期间 OPEN 或 LOCKED-先关业务账**、职责分离 | ➕`finance.post_voucher` |
| 过账（资金类） | REVIEWED→POSTED | FINANCE_MANAGER | 同上 | ➕`finance.post_voucher` |
| ➕**反过账** | POSTED→AUDITED | FINANCE_MANAGER | **期间≠CLOSED**（已结账须先反结账）、**逐月返**、填原因+留痕 | ➕`finance.unpost_voucher`（冲回 AccountBalance） |
| ➕**反审核** | AUDITED→DRAFT | FINANCE_MANAGER | 填原因 | — |

> **过账与期间锁的交互（关键 gap，§⑥-3）**：录音 L139「关账→生成凭证→结账」意味着**关账（LOCKED）后仍需能过账凭证**（因为关账只锁业务录入，财务还要把凭证生成+过账后才结账）。故 `period_writable` hard_rule 对 LOCKED 期的拦截 = **仅拦业务单据增删 + 拦凭证的新增/改，但放行已生成凭证的过账**；CLOSED 期才全拦过账。

## ⑤ 逆操作精确规则（★删除「蓝冲」·录音 L304-319）

> **★ v1 错 → v2 正（术语级纠正）**：v1 §4.5/§10 大量使用「**蓝冲**」二分法（未结账=蓝冲 / 已结账=红冲），并自称「已 WebSearch 核实 K3 标准术语」。**录音 L304-319 资深会计（金蝶用户）明确：「我不懂蓝冲，只听过红冲」——「蓝冲」不是标准术语，删除。** 逆操作正确三分法如下：

| 场景 | 正确方法 | 操作链 | 留痕 | 适用 |
|------|---------|--------|------|------|
| **当月填错（业务错误，期间未结账）** | **直接删 / 改凭证** | 凭证可删的边界=**未结账**（不是未过账）：①单据生成的凭证可删，但**删后必须重新生成否则不能结账**；②手工凭证未结账即可删；③已过账未结账→**反过账后**可删/改 | 凭证删除日志 | 当期业务错账 |
| **状态逆向（已过账要改）** | **反过账 / 反审核** | `反过账 POSTED→AUDITED`（冲回余额）→`反审核 AUDITED→DRAFT`→直接改/删→重新审核→过账。**反过账只能逐月返**（7月结账只能反7月，不能反6/5/4月） | WorkflowLog（谁/何时/原因） | 已过账未结账的修正 |
| **认可原业务但金额要更新（如暂估转实）** | **红冲** | **原单不删、生成反向单据**（一正一负）。系统逻辑**统一不区分跨月**：当月红冲=一正一负净 0（录音 L325「当月冲了相当于当月一正一负，没有任何影响」）；一般次月冲（录音 L325「一般红冲就是次月才会冲」）。〔跨年度损益错账走「以前年度损益调整」科目 = **待财务确认**，录音未涉及〕 | 红字+蓝字配对，净额=0 | 暂估转实 / 跨期更新 |

> **三件事的区别（录音 L304-319 原话还原）**：
> - **当月填错 → 直接删/改**（未结账可删），**不走红冲**；
> - **反过账/反审核 = 状态逆向校验**（逐月返），不是红冲；
> - **红冲 = 原单不删、生成反向单据**，用于"认可原业务但金额要更新"（如暂估转实，发票来按发票金额入账 + 红冲冲掉原暂估）；系统逻辑**不区分跨月**。
> - **删除 v1 所有「蓝冲」提法**（v1 §4.5 表第一行「蓝冲」、§4.11 UI「冲销▼蓝冲/红冲」下拉、§10 「蓝冲性质解锁」措辞）。反过账/反审核就叫反过账/反审核，不叫蓝冲。

**红冲 command**（❌新建 `finance.red_reversal`）：入参=原 voucher_id → 生成反向 Voucher 草稿（每行 debit/credit 取反 / 反向、source_doc 回链原凭证、is_auto_generated=True、description="红冲 {原凭证号}"）→ 审核过账。**业务红冲**（退货/退款）顺 source_doc 回链找原凭证 → 自动生成红字草稿 → 财务审核（必广播全财务，见基石 01 §4.6）。

## ⑥ 校验规则 + 边界 + 错误处理（绝不静默）

### 关账校验（hard_rule，OPEN→LOCKED）

| # | 规则 | 失败提示 |
|---|------|----------|
| 1 | 前序期已关账/结账 | 「上一期 {y}-{p} 未关账，须先按顺序关账」 |
| 2 | 关账须填确认 | 「关账将锁定本期数据，确认？（仅财务可补凭证，业务录入将冻结）」 |
| 3 | 反关账须填原因+留痕 | 「反关账须填写原因（审计留痕：{user}/{time}）」 |

### 各模块结账校验（hard_rule，写 ModuleCloseStatus.CLOSED）

| # | 规则 | 失败提示 |
|---|------|----------|
| 1 | 本模块本期凭证全生成 | 「{模块}有 {n} 笔单据未生成凭证，请先生成」 |
| 2 | 本模块本期凭证全过账 | 「{模块}有 {n} 张凭证未过账（{凭证号}），请先过账」 |
| 3 | 本模块顺序（先关账后结账） | 「本期尚未关账，须先关账锁数据再结账」 |

### 总账结账前校验清单（hard_rule，LOCKED→CLOSED，全绿才放行）

| # | 规则 | 校验点 | 失败提示 |
|---|------|--------|----------|
| 1 | **全模块已结账** | 所有 ModuleCloseStatus（INVENTORY/AR_AP/CASH/FIXED_ASSET/…）=CLOSED | 「{模块} 尚未结账，总账须待全部模块结账后才能结账」 |
| 2 | **无未过账凭证** | 当期无 status∈{DRAFT,AUDITED,REVIEWED} 的 Voucher | 「当期有 {n} 张凭证未过账：{凭证号}」 |
| 3 | **试算平衡** | Σ本期 closing_debit = Σ closing_credit（误差<0.01） | 「试算不平：借 {x} ≠ 贷 {y}，差额 {d}，禁结账」 |
| 4 | **汇兑损益已做**（多币种） | 已生成并过账当期调汇凭证（无外币跳过） | 「外币货币性科目未调汇，请先期末调汇」（详见 10-期末结账） |
| 5 | **结转损益已做** | 损益科目本期 closing=0（顺序：汇兑损益在前、结转损益在后） | 「损益未结转，请先结转损益」（详见 10-期末结账） |
| 6 | **前序期已结账** | 上一期 status=CLOSED（顺序结账不跳期） | 「上一期 {y}-{p} 未结账，须按顺序逐期结账」 |

### 反结账校验（CLOSED→OPEN）

| 规则 | 提示 |
|------|------|
| 须填原因+留痕 | 「反结账须填写原因（审计留痕：谁/何时点反结账）」 |
| **只逐月返**（只逆最新已结期） | 「只能反结最新已结期 {y}-{p}，不可跳期反结（反过账只能逐月返）」 |
| 下期未占用 | 「下期 {y}-{p+1} 已有过账凭证/已结账，需先逐期反结」 |
| 会签 | 「反结账需 BOSS 会签」 |

### 期间口径校验（v2 删除项的反向守门）

| 规则 | 提示 |
|------|------|
| **期间必自然月** | 建期间时 start_date/end_date 强制 = 当月1日/月末日，**拒绝任何人工自定义边界**「会计期间一律按自然月，准则统一不可改」 |
| **禁 period_cutoff_day** | 配置项中**不得出现**截止日/分割月；27 号只在"关账动作"出现 |

> **待财务确认 gap（汇总见 §⑪）**：①LOCKED 期对"已生成凭证的过账"放行/拦截的精确粒度；②各模块结账的模块清单（是否含税务/费用模块）；③汇兑/结转的硬软度（草案=硬）；④结账弹性窗口（草案=次月1~2号，可配警示不强拦）。

```
│  [总账结账] (全绿才亮,需BOSS会签)        弹性:贸易类次月1-2号完成          │
└──────────────────────────────────────────────────────────────────────────┘
```

**专用屏 3：凭证录入(双金额+资金类复核)** —— 借贷网格**原币列+本位币列两列**(录音L223)、资金类凭证审核后多"复核"按钮、键盘流(Tab横走/Enter下行/`=`配平)、实时借贷平衡。详 UI 见基石 01 §4.11，本模块仅强调:①原币/本位币双金额列;②资金类多复核态。

**关键交互**:期间网格三态徽标(OPEN绿/LOCKED黄/CLOSED灰);关账与结账分两个独立按钮(不可一步OPEN→CLOSED);先关业务账=独立子动作;反关账/反结账二次确认+原因必填+会签;反结账入口仅在最新已结期(逐月返);各模块结账状态行可下钻看未过账凭证;总账结账按钮全绿才亮。

## ⑩ 验收用例（Given-When-Then，含边界）

**会计期间（自然月）**
- G:建 2026 年度 / W:批量生成期间 / T:生成 12 期，第6期 start=2026-06-01 end=2026-06-30（**非 05-28~06-27**）。
- 边界 G:试图把第6期起止改成 05-28~06-27 / W:保存 / T:拒绝「期间一律自然月，准则统一不可改」。
- 边界 G:配置界面 / W:找截止日/分割月配置 / T:**不存在该配置项**（已删 period_cutoff_day）。

**关账（锁数据）**
- G:第6期 OPEN / W:FINANCE_MANAGER+BOSS 点关账(全锁) / T:status=LOCKED、写 locked_by/at；该期业务单据增删被 period_writable 拦。
- G:第6期 OPEN / W:点"先关业务账" / T:business_locked=True；业务录入冻结，**财务仍可录凭证+过账**（为"关账→生成凭证→结账"留通道）。
- G:第6期 LOCKED 全锁、有 1 张待过账凭证 / W:过账 / T:**放行**（LOCKED 不拦已生成凭证过账，只拦录入；CLOSED 才全拦）。
- G:第6期 LOCKED / W:反关账+填原因 / T:status=OPEN、WorkflowLog 留痕(谁/何时)。

**过账 + 资金类复核**
- G:资金类凭证(含1002银行) AUDITED / W:直接过账 / T:拒绝「资金类凭证须先复核」；走 AUDITED→REVIEWED→POSTED。
- G:非资金类凭证 AUDITED / W:过账 / T:直达 POSTED，写 AccountBalance（原币+本位币双金额）。
- G:已过账凭证、期间未结账 / W:反过账 / T:AccountBalance 精确冲回、status→AUDITED、可改；逐月返。
- 边界 G:7月、6月均已过账 / W:直接反过账6月凭证(7月未反) / T:拒绝「反过账只能逐月返」。

**各模块结账 + 总账结账（顺序铁律）**
- G:第6期 OPEN（未关账） / W:点存货模块结账 / T:拒绝「须先关账锁数据再结账」（关账→生成凭证→结账顺序）。
- G:存货模块有 2 张凭证未过账 / W:存货结账 / T:拒绝，列凭证号。
- G:出纳模块未结账 / W:总账结账 / T:校验#1 失败「出纳尚未结账，总账须待全部模块结账后」。
- G:全模块已结+清单全绿 / W:FINANCE_MANAGER+BOSS 会签总账结账 / T:status=CLOSED、写 closed_by/at、本期 closing→下期 opening。
- G:期间 CLOSED / W:对该期凭证过账 / T:hard_rule 拦「已结账须反结账或走调整期」。

**反结账（逐月返）**
- G:第6期 CLOSED 为最新已结期、第7期无过账 / W:FINANCE_MANAGER+BOSS 反结账+原因 / T:status→OPEN、留痕、下期 opening 回退。
- 边界 G:第5、6期均 CLOSED / W:直接反结第5期 / T:拒绝「只能反结最新已结期，不可跳期」。

**逆操作（无蓝冲）**
- G:当月手工凭证填错、期间未结账 / W:删凭证 / T:允许直接删（未结账可删），**不走红冲/不提"蓝冲"**。
- G:单据生成的凭证、删后 / W:试图结账 / T:拒绝「该单据凭证已删，须重新生成否则不能结账」。
- G:暂估应付、发票到 / W:按发票金额入账+红冲原暂估 / T:原暂估单不删、生成反向单据，净额按发票更新（红冲=认可原业务但金额更新）。
