## 一、★ 三层分层结构（先读这张，理解模块如何咬合）

财务 ERP 自下而上分 6 层，下层是上层的契约来源，**任何模块开工前对齐 02 基石的共享契约（科目码 / 状态码 / 角色 / 取数格式）**：

```
┌─ 第 6 层 · 合并报表（多主体）──────────────────────────────────────┐
│   16 合并报表（报表层手工合：干净求和 / 不干净抵消，依赖往来+辅助账） │
├─ 第 5 层 · 报表与账表（结账后生成·派生只读）─────────────────────────┤
│   15 财务报表（资产负债表/利润表/现金流量表）· 账表（科目余额表/明细账/总分类账）│
├─ 第 4 层 · 期末收口（向导式批处理）────────────────────────────────┤
│   14 期末结账与期初建账（关账→生成凭证→过账→汇兑损益→结转损益→各模块结账→总账结账）│
├─ 第 3 层 · 业务单据模块（只挂业务单据·经业财翻译生成凭证草稿→总账）───┤
│   06 应收 · 07 应付 · 08 出纳与资金 · 09 往来 · 10 税务/发票          │
│   11 费用 · 12 存货核算与成本 · 13 固定资产 · 17 HR/薪酬(边界)         │
├─ 第 2 层 · 总账凭证内核（唯一挂凭证）+ 业财翻译发动机 ─────────────────┤
│   04 总账与凭证（DRAFT→审核→[资金类复核]→过账→反过账→反审核→红冲）    │
│   05 凭证模板与业财一体化映射（业财翻译表=业务单据↔凭证 两映射）        │
├─ 第 1 层 · 基础设置（基石主数据）───────────────────────────────────┤
│   01 会计科目与辅助核算 · 02 会计期间/关账/结账/过账 · 03 币种/本位币/汇率│
│   00 编写规范与对象范式（宪法 + 对象范式 + 模块总纲）                  │
└──────────────────────────────────────────────────────────────────┘
```

**层间数据流（铁律）**：业务单据（第3层）→ 业财翻译规则（05）生成**凭证草稿** → 流到**总账**（04，唯一挂凭证）→ 审核/[资金类复核]/过账 → 写 `AccountBalance` → 期末收口（14）→ 报表/账表（15）派生 → 合并（16）报表层手工汇总。**财务唯一写入路径 = 凭证过账**；台账/报表永远是派生只读。

## 二、★ 新结构总览（18 模块表 · 标「挂凭证 / 挂单据」）

> 核心列 = **挂凭证 or 挂单据**（铁律 1）：只有「总账」挂凭证，其余全挂业务单据 / 报表 / 配置。

| 文件 | 模块 | 层 | 挂什么 | 主对象（doc_type / model） | 录音边界要点 |
|---|---|---|---|---|---|
| **00** | 编写规范与对象范式 | 宪法 | —（规范）| 对象范式 / 5 类对象 / 8 件套 / 模块总纲 | 三条头号铁律 + 26 条修订决策 + 全局不变量 |
| **01** | 会计科目与辅助核算 | 1 基础 | 主数据 | `Account` / `AuxiliaryDimension` / `AccountAuxLink` | 五大类；余额方向最关键；现金流量项目+往来标识；辅助核算 1~2 维（银行账户=维度非科目）；子科目无限级/无代码/沿用父属性 |
| **02** | 会计期间/关账/结账/过账 | 1 基础 | 主数据+期间动作 | `FiscalYear` / `AccountingPeriod` / `ModuleCloseStatus` | 期间=自然月不可配；27 号=关账时点非边界；过账/关账/结账三独立动作；OPEN→LOCKED→CLOSED；反结账逐月返 |
| **03** | 币种/本位币/汇率 | 1 基础 | 主数据（被引用）| `Company.currency`(本位币) / `ExchangeRate` | 一企一本位币（公司级可配）；月维护1汇率即可；固资入账锁定不调汇；资金类两汇率可选；汇兑损益在前 |
| **04** | 总账与凭证 | 2 内核 | ★**凭证** | `Voucher` / `VoucherEntry` · VOUCHER；`AccountBalance` | **唯一挂凭证**；原币+本位币双金额；资金类多一步复核；过账写 AccountBalance（缺 finance.post_voucher） |
| **05** | 凭证模板与业财一体化映射 | 2 内核 | 配置（驱动凭证生成）| `VoucherTemplate` / `VoucherTemplateLine` / `map_to_voucher_entries` | 业财翻译表=两映射；可配科目/金额(含税·不含税·税额)/摘要/辅助核算/日期；订单类不进凭证 |
| **06** | 应收管理 | 3 单据 | **单据** | `AccountsReceivable` / `ARSettlement` · ACCOUNTS_RECEIVABLE | 先有应收才有凭证；来源=销售出库；暂估应收+确认应收；★**收入确认归应收**（与税解耦）|
| **07** | 应付管理 | 3 单据 | **单据** | `AccountsPayable` / `PaymentRequest` / `APSettlement` · ACCOUNTS_PAYABLE | 暂估应付；特殊业务(员工借款)手录；供应商信用=录音无依据待确认 |
| **08** | 出纳与资金 | 3 单据 | **单据** | `CashReceipt` / `CashPayment` / `CashJournal` / `BankReconciliation` / `FundTransfer` / `BankAccountDim` | 与应收应付同源侧重不同；银行账户=辅助核算维度非科目；连不上银行→网银手动付；日记账与1001/1002勾稽 |

| **10** | 税务与发票 | 3 单据 | **单据（发票）** | `SalesInvoice` / `PurchaseInvoice` / `VatAssessment` / `IncomeTaxProvision` | 税务独立·核心=发票；采购发票可扫描；销项/进项税；不打通税局可不做(A/B 档)；收入确认≠税务确认 |
| **11** | 费用管理 | 3 单据 | **单据** | `ExpenseClaim` / `ExpensePayment` / `Employee` / `ExpenseCategory` | ★**不与薪酬合并**（独立模块）；类别→费用科目映射+部门辅助核算 |

| **14** | 期末结账与期初建账 | 4 期末 | **过程批处理（E）**| FX 调汇批 / 结转损益批 / 期间结账 / 期初建账 | 七步顺序；汇兑损益在前结转损益在后；反结账留痕逐月；只上线首期建期初；完全替代金蝶切换 |
| **15** | 财务报表与账表 | 5 报表 | **报表（D·派生只读）**| `BALANCE_SHEET` / `INCOME_STATEMENT` / `CASH_FLOW` / 账表 / `ReportLineDef` / `CashflowItem` | 结账后生成；★现金流量表靠凭证现金流量项目取数(直接法)；账表=科目余额表/明细账/总分类账 |
| **16** | 合并报表（多主体）+ 多组织账套与权限 | 6 合并 | **报表层手工汇总**| `ConsolidationGroup` / `ConsolidationWorksheet` / `EliminationEntry` / `CrossCompanySummary` | ★可手工合(干净求和/不干净抵消)；依赖往来+辅助账；6 公司行级隔离≠不出合并；并排看板≠合并报表 |

## 四、★ 全局不变量清单（录音版 · 每模块逐条遵守）

来自 `00-编写规范与对象范式.md §七`，14 条铁律：

1. **借贷平衡**：过账前 `Σ借 = Σ贷`(hard_rule)，不平禁过账、精确提示差额。借贷增减由 **余额方向 + 增减** 推导，禁写死借贷。
2. **过账不可改 / 凭证可删边界 = 未结账**(不是未过账)：POSTED 不可直接改；未结账可删/改(单据生成的删后须重新生成)；已过账未结账→反过账→反审核(逐月)。AccountBalance/账表/报表 = 只读派生。
3. **期间锁两层**：**关账** = 锁数据库(该期增删不可·27 号人为)；**结账** = 锁财务录入(结账到下期)。已结账期不可过账。
4. **业财同源**：业务单映射生成凭证草稿与业务单**同事务原子**，要么都成要么都回滚。
5. **财务唯一写入路径 = 凭证过账**：任何科目余额变动必经 Voucher→VoucherEntry→AccountBalance(唯一例外 = 期初建账写 opening 列，仅第一个期间一次性)。
6. **凭证双金额**：外币分录借贷方各存**原币金额 + 本位币金额**两个值(非汇率推算)。
7. **现金流量项目 + 往来标识**：科目须标识现金流量项目(否则现金流量表取不到数)+ 往来类标识。
8. **审计四本**：`CommandLog` / `WorkflowLog` / `AgentLog` / `WorkflowDefAuditLog` 全程留痕；反结账/反过账必留痕(谁/何时)。
9. **字段防火墙**：成本/买价对 SALES + SALES_ASSISTANT 隐藏(字段级 mask)。
10. **公司行级隔离**：`company_id` NOT NULL，查询走 `_company_filter(user)`，**绝不混账**；但**合并报表可在报表层手工汇总**(行级隔离 ≠ 不出合并报表)。
11. **逆操作不留脏数**：未结账直接删/改原凭证；红冲 = 原单留 + 反向单(一正一负净 0)，用于金额更新(暂估转实)。**禁用「蓝冲」**。
12. **业务红冲必广播**：退货/退款/采购退货自动生成红冲反向凭证草稿(顺 source_doc 回链) + 站内通知全部财务(`Notification.recipient_role`)。
13. **往来含税且双方一致**：往来款一定含税、集团内双方口径一致，否则不能核对/合并(税差只在进不进成本/收入)。
14. **期末顺序固定**：各模块关账 → 生成凭证 → 过账 → **汇兑损益 → 结转损益** → 各模块结账 → 总账结账。
