## ★★ 零、三条头号铁律（写任何模块前先背下，违者全篇作废）

### 铁律 1 ──「只有总账挂凭证，其余模块只挂业务单据」

> 录音 L334-526：**只有"总账"模块挂凭证；其他所有模块只挂业务单据。** 业务单据经"凭证模板/业财翻译规则"生成凭证草稿 → 流到【总账】统一审核过账。

- 凭证的**全部生命周期**（新增保存 → 审核 → [资金类多一步"复核"] → 过账 → 反过账 → 反审核 → 红冲）**只属于总账模块**，不下放到任何业务单。
- 各业务模块（应收/应付/出纳/存货/固资/薪酬/费用/税务/往来）**只持有业务单据**（应收单、付款单、采购入库单、固资卡片、发票…），业务单据的状态机里**没有"过账"这一步**。
- 业务单据 → `AccountMappingRule`（业财翻译规则 / 凭证模板）→ **生成凭证草稿** → 草稿落到总账 → 在总账走审核/过账。
- ★只对**需要生成凭证的单据**（采购入库单、销售出库单、付款单、收款单、发票、折旧、计提…）做业财映射；**订单类单据（销售订单/采购订单）不进凭证**。

```
业务模块（B 类业务单据）          总账模块（唯一挂凭证）
┌─────────────────┐   映射     ┌──────────────────────────────┐
│ 采购入库单 已审核 │──────────▶│ 凭证草稿 DRAFT               │
│ 销售出库单 已发货 │  Account  │   → 审核 → (资金类:复核) → 过账│
│ 付款单/收款单    │  Mapping  │   → 反过账 → 反审核 → 红冲     │
│ 发票/折旧/计提   │  Rule     │   → 过账后写 AccountBalance    │
└─────────────────┘           └──────────────────────────────┘
   只有业务单据状态机             凭证生命周期 100% 在这里
```

> **v1错 → v2正**：v1 §二把 AR/AP/预收预付/付款申请/票据/发票全列为「B 交易凭证单据」并说它们"有完整生命周期状态机、可过账"——错。这些都是**业务单据**，**它们自己不过账**；过账只发生在它们映射出的**总账凭证**上。v2 把"可过账"从所有业务单收回，唯一归总账。

### 铁律 2 ──「会计期间 = 自然月；27 号是关账时点，不是期间边界；关账/结账/过账是三个独立动作」

> 录音 L76-160：会计期间**一律自然月**（每月 1 日~月末日 / 自然年），准则全球统一、**不可按公司切换**。"27 号"是**关账动作的时点**（人为锁当月录入窗口），**不是会计期间的边界**。

- **会计期间 = 自然月**：1 日 ~ 当月末日。准则通用、**全公司统一、不可按公司切换**。
- ❌ **绝不能**把期间设成"28~次月27"；❌ **绝不能**有 `period_cutoff_day` 作为期间边界；❌ **绝不能**"12月27分割财年"。
- "上月 27 → 本月 27"指的是**业务关账窗口**（人为锁录入），**会计期间仍按自然月计**。
- **关账 ≠ 结账 ≠ 过账**，是三个独立动作：

| 动作 | 是什么 | 作用对象 | 时机 | 谁触发 |
|---|---|---|---|---|
| **过账** | 凭证的动作 | **凭证** | 凭证：新增保存→审核→[资金类复核]→**过账**；过账在结账**之前** | 总账（ACCOUNTANT 提，FINANCE_MANAGER 过；资金类多一步复核） |
| **关账** | **锁数据库**（该期间增/删都不可）；可先关业务账 | 该期间数据 | 27 号（人为时点） | 各模块人为点 |
| **结账** | 不允许再录**财务**数据，结账到下一期 | 财务模块/期间 | 各模块**各有自己的结账动作**（存货模块、应收应付模块…），人为点 | 各模块负责人 |

- **模块内顺序铁律**：先**关账**（锁数据）→ 把该生成凭证的**生成凭证**→ 过账 → 再**结账**。
- **总账结账顺序**：各模块凭证全生成并过账 → 各模块结账 → 所有模块完 → **总账才能结账**。
- **反结账** = 打开期间（回到该期可反过账/反审核/删除），**必审计留痕**（谁/何时反结账与结账）。
- **反过账只能逐月返**（7 月结账只能反 7 月，不能反 6/5/4 月）。
- 结账周期弹性：贸易类企业一般**次月 1-2 号**完成。

> **v1错 → v2正**：v1 决策 9「会计期间可配，默认截止 27 号、会计年度 12 月 27 号分割」**全错**——期间是自然月、不可配、27 号只是关账时点。v1 把关账/结账/过账混为一谈（README §五把"反审核/反过账/反结账"全堆一处）也**纠正为三个独立动作**，且过账只在凭证、关账锁库、结账锁财务录入，三者顺序固定。

### 铁律 3 ──删除「蓝冲」术语；逆操作 = 未结账删改 / 反过账反审核 / 红冲

> 录音 L223 等：资深会计 + 金蝶用户明确"**不懂蓝冲，只听过红冲**"。**系统里没有"蓝冲"这个术语**。

逆操作三件事（**取代** v1 的"蓝冲 vs 红冲"二分法）：

| 场景 | 正确做法 | 不是 |
|---|---|---|
| 当月填错（业务错误）| **直接删 / 改凭证**（凭证可删的边界 = **未结账**，不是未过账）| ❌ 不走任何"冲" |
| 已过账、未结账 | **反过账**（POSTED→AUDITED）→ 反审核（AUDITED→DRAFT）→ 改 / 删 → 重过账 | ❌ 不叫蓝冲 |
| 反过账 / 反审核 | 状态逆向校验，**逐月返**（7 月只能反 7 月）| ❌ 不能跨月乱反 |
| **红冲** | 原单**不删**、生成**反向单据**（一正一负净 0）。用于"**认可原业务但金额要更新**"（如暂估转实）。系统**统一不区分跨月**；当月红冲一正一负净 0，一般次月冲 | ❌ 红冲≠删改，原单保留 |

- **凭证可删的边界 = 未结账**（不是未过账）：① 单据生成的凭证可删，但删后**必须重新生成**否则不能结账；② 手工凭证未结账即可删。已过账未结账 → 反过账后可删/改。
- **没有"蓝冲"**：v1 自称"WebSearch 核实 K3 标准术语"得出的"蓝冲"提法 **= 错，全文删除**。

> **v1错 → v2正**：v1 决策 12 + README §五「蓝冲 vs 红冲（已 WebSearch 核实金蝶 K3 标准术语）」**整段错**——"蓝冲"不存在，删除。v2 逆操作改为：未结账直接删改 / 反过账反审核（逐月）/ 红冲（原单留、生反向单、用于金额更新如暂估转实）。

## 一、统一对象范式（PRD 骨架·固定 10 列 + 拓展）

每一个财务对象（doc_type / model）都必须按下表写满。这是 PRD 的最小信息单元，也是 dev 的取数清单。

| # | 列名 | 写什么 | 映射的引擎构件（grep 已核实） |
|---|------|--------|------------------------------|
| ① | **对象** | doc_type 码 + model 类名 + 中文名 | `__doc_types__` 元组（被 `core/registry.py:doc_model_map()` 启动扫描）；纯子表/主数据标 `__queryable__=True` 进 `table_map()` |
| ② | **对象类型** | 5 类之一：A 主数据 / B 交易**业务单据** / C 台账余额派生只读 / D 报表派生定期 / E 期末过程批处理（见 §二）。★注意：**B 类业务单据自己不过账**（铁律 1）；只有总账的凭证过账 | 决定有无专用状态机、是否只读派生、是否映射出凭证 |
| ③ | **对象数据** | 字段级数据字典：头 + 行子表 + **辅助核算维度**（科目级可挂 **1 个或 2 个**维度，系统可配——录音明示：往来 customer/supplier、银行账户；部门/项目/币种为常见维度，**待财务确认是否启用**）。每字段：类型/长度/必填/默认/枚举/精度/唯一/FK。★金额 `Numeric(16,2)`、单位成本 `Numeric(16,4)`、汇率 `Numeric(12,6)`；★**凭证分录借贷方各给两个金额：原币金额 + 本位币金额** | SQLAlchemy `Column`；公司隔离用 `company_id`（NOT NULL, indexed）；审计字段来自 `AuditMixin` |
| ④ | **动作 ×（角色，时机/前置）** | 状态机每条出边：`{label, from, to, roles[], hard_rules[], editable_fields[], effects[]}`。★业务单据的边里**没有"过账"**；映射出凭证草稿用 effect 串 `map_to_voucher_entries` | `WorkflowDefinition.states` JSONB 的 `next[]`；`execute_transition` 校验 `allowed_roles`/`editable_fields`/`hard_rules` 再跑 `effects[]` |
| ⑤ | **逆操作** | **（按铁律 3）**：未结账 → 直接删/改（凭证可删边界=未结账）；已过账未结账 → 反过账→反审核（逐月）→改/删→重过账；**红冲**=原单留、生反向单、用于金额更新（暂估转实）。**禁用"蓝冲"** | 逆向出边（POSTED→AUDITED 反过账、AUDITED→DRAFT 反审核）+ 红冲 `@register_command` 生成反向 Voucher 草稿 |
| ⑥ | **效果** | 过账更新 `AccountBalance`（period_debit/credit→closing）；业务单映射生成往来/凭证草稿；推台账/报表数据 | `@register_transition_effect("finance.xxx")`，签名固定 `(db, doc_type, doc, to_state, user, command_log_id) -> list[str]` |
| ⑦ | **关联回链** | `source_doc_type` + `source_doc_id`（凭证回链业务单）、`voucher_id`（业务单回链凭证）、`command_log_id`、FK 链 | `Voucher.source_doc_type/source_doc_id`；AR/AP/InventoryTransaction.`voucher_id`；`routers/chain.py`、`routers/data.py` |
| ⑧ | **准则 / 账簿** | CAS（内地 3 家）/ HKFRS（香港 3 家，本位币可 USD）差异；`company_id` 账套（**公司行级隔离，绝不混账**——但**可手工出合并报表**，见铁律延伸） | `Account.company_id` + `_company_filter(user)`（`routers/reports.py` 已用） |
| ⑨ | **现金流量项目 / 往来标识** | 科目须标识：① **现金流量项目**（做凭证时指定，否则现金流量表取不到数）；② **往来类科目**标识 | `Account` 上的现金流量项目标识 + 往来标识（➕扩展） |
| ⑩ | **对象特有规则 / 口径 / 钩子 / 准则差异** | 暂估口径、红冲回链规则、收入确认时点、调汇范围、坏账钩子、辅助核算维度数（1 或 2）等 | `@register_transition_validator` 硬拦 + effect 内联逻辑 |

**逐列详解要点：**
- ③ 必须给到「可直接建表/可直接 seed」的精度。★**凭证分录必须存原币 + 本位币两个金额**（外币时两个都给，**不是只存原币 + 汇率推算**——录音 L172-232 明确）。当前引擎 `VoucherEntry` 只有 `debit/credit + currency + exchange_rate`，**缺 local-amount 列 ➕ 须扩**。
- ③ **辅助核算**：科目级可挂 1 个或 2 个维度（系统可配）。典型例：银行存款名下 10 个账户 → **不建 10 个科目**，而用"辅助核算维度"挂账户。
- ④ 一条边 = 状态机一行。`roles[]` 留空 `[]` = 系统/自动触发边。`effects[]` 是**字符串名数组**（引擎按名查 `_EFFECTS`），不是函数引用。**业务单据的边里不出现"过账"动作**。
- ⑤ 必须明确「未结账直接删改 / 反过账反审核逐月 / 红冲」三路径，并标谁有权（默认 ACCOUNTANT 提、FINANCE_MANAGER 过；资金类多一步复核）。**禁止"蓝冲"提法**。
- ⑥ **财务唯一写入路径 = 凭证过账**。AccountBalance / 报表 / 台账永远是派生只读，禁止任何旁路直写。

## 二、对象类型 5 分类（行为 + 引擎落点 + 逆操作处理）

| 类 | 名称 | 行为特征 | 有无专用屏 | 引擎落点 | 逆操作如何处理 |
|----|------|----------|-----------|----------|--------------|
| **A** | 主数据 | 长期存在、被引用、少变更（科目/期间/币种/汇率/科目映射模板/角色配置/银行账户作辅助核算维度） | 配置类用通用壳 `DocEditor`；科目表用专用「左树右明细」 | `__queryable__=True`；变更走 Admin 审计 | 不冲销，只「停用/版本化」。生效日切换准则模板 |
| **B** | 交易**业务单据** | 有自己的业务生命周期状态机、**自身不过账**；经映射规则生成总账凭证草稿（应收单/应付单/收款单/付款单/结算单/发票/采购入库单/销售出库单/固资卡片/折旧/计提/费用报销/红冲反向单） | **有专用屏**（业务单录入；往来核销左右栏） | `__doc_types__` + `WorkflowDefinition.states`；映射 effect 串 `map_to_voucher_entries` 生成凭证草稿 | 业务单按自身规则改/作废；**对应凭证的逆操作（删改/反过账/红冲）在总账做**。业务退货 → 自动生成红冲反向单据草稿 + 广播全财务 |
| **总账·凭证** | （B 的特例，**唯一挂凭证者**）记账凭证 Voucher/VoucherEntry | 完整凭证生命周期（保存→审核→[资金类复核]→过账→反过账→反审核→红冲），强审计 | 凭证录入密集网格 + 借贷实时平衡 + **原币/本位币两列** + 辅助核算弹层 | `Voucher` doc_type=VOUCHER；过账 effect 写 `AccountBalance`（**❌ 缺，须新建 finance.post_voucher**） | **唯一逆操作发生地**：未结账删改 / 反过账反审核（逐月）/ 红冲（反向单） |
| **C** | 台账/余额派生只读 | **不可手工写**，由凭证过账派生（AccountBalance、InventoryValuation、往来余额、信用占用、累计折旧台账） | 无专用屏，用 `DataExplorer` 下钻 | effect 计算后写入；查询走 `routers/reports.py` + `_company_filter` | 不直接冲销。逆操作发生在源凭证，台账随过账/反过账自动重算 |
| **D** | 报表派生定期 | **结账后生成**，口径固定、可反复出（三大表 + 账表 + 试算 + 科目余额 + 账龄 + 现金流量）；★现金流量表靠**凭证录入时指定现金流量项目**取数，取数固定公式 | 复用现有 reports 壳 + 期间/科目范围筛选 | `routers/reports.py`（已有 trial_balance/account_balance/aging）；取数公式驱动 | 不冲销。源改了重出表 |
| **E** | 期末过程批处理 | 一次性向导式批量动作。★顺序：各模块**关账 → 生成凭证 → 过账 → 汇兑损益 → 结转损益 → 各模块结账 → 总账结账**（**汇兑损益在前，结转损益在后**） | **向导式**（分步 + 前置校验清单，UI 律唯一向导例外） | `@register_transition_effect` 批处理 + `@register_transition_validator` 前置清单 | **反结账**（逐月、必留痕）解锁期间；批处理生成的凭证按凭证逆操作规则处理 |

## 三、dev-ready 开发可开工 8 件套（硬标准）

凡有专用状态机/专用屏的对象（B 类业务单据、总账凭证、部分 E 类），PRD 必须给齐以下 8 件，缺一不可开工：

| # | 件套 | 「可开工」硬标准 |
|---|------|------------------|
| 1 | **字段级数据字典** | 每字段有类型/长度/必填/默认/枚举/精度/唯一/FK，可直接建 `Column`。★凭证分录含**原币金额 + 本位币金额两列** + 现金流量项目 + 辅助核算 |
| 2 | **精确状态机表** | `state码 × 节点allowed_roles × 边editable_fields × hard_rules精确条件 × 触发effect名`，可直接 seed JSONB。★业务单据状态机**不含"过账"**；凭证状态机含 审核/[资金类复核]/过账/反过账/反审核 |
| 3 | **★科目映射取数公式** | `业务单×动作 → 借/贷科目代码 + 金额取数公式(含税/不含税/税额) + 税率来源 + 暂估/红冲处理 + 现金流量项目 + 辅助核算 + 日期(单据创建日 vs 业务发生日,可选)`，每条标「财务确认」。方向由 `balance_direction` + 增减推导，不写死借贷 |

## 七、全局不变量清单（每模块逐条遵守）

1. **借贷平衡**：过账前 `Σ借 = Σ贷`（hard_rule），不平禁过账，精确提示差额。借贷增减由 **余额方向 + 增减** 推导，不写死。
2. **过账不可改 / 凭证可删边界 = 未结账**：`Voucher.status=POSTED` 不可直接改；未结账可删/改（单据生成的删后须重新生成）；已过账未结账 → 反过账→反审核（逐月）。AccountBalance / 账表 / 报表 = **只读派生**。
3. **期间锁两层**：**关账** = 锁数据库（该期增/删不可，27 号人为）；**结账** = 锁财务录入（结账到下期）。已结账期不可过账。
4. **业财同源**：业务单映射生成凭证草稿与业务单**同事务原子**，要么都成要么都回滚。
5. **财务唯一写入路径 = 凭证过账**：任何科目余额变动必经 Voucher→VoucherEntry→AccountBalance（**唯一例外 = 期初建账写 opening 列，仅第一个期间一次性**）。
6. **凭证双金额**：外币分录借贷方各存**原币金额 + 本位币金额**两个值（非汇率推算）。
7. **现金流量项目 + 往来标识**：科目须标识现金流量项目（否则现金流量表取不到数）+ 往来类标识。
8. **审计四本**：`CommandLog` / `WorkflowLog` / `AgentLog` / `WorkflowDefAuditLog` 全程留痕；反结账/反过账必留痕（谁/何时）。
9. **字段防火墙**：成本/买价对 SALES + SALES_ASSISTANT 隐藏（字段级 mask）。
10. **公司行级隔离**：`company_id` NOT NULL，查询走 `_company_filter(user)`，**绝不混账**；但**合并报表可在报表层手工汇总**（行级隔离 ≠ 不出合并报表）。
11. **逆操作不留脏数**：未结账直接删/改原凭证；红冲 = 原单留 + 反向单（一正一负净 0，余额不受影响），用于金额更新（暂估转实）。**禁用"蓝冲"**。
12. **业务红冲必广播**：退货/退款/采购退货自动生成红冲反向凭证草稿（顺 source_doc 回链）+ 站内通知全部财务（`Notification.recipient_role`）。
13. **往来含税且双方一致**：往来款一定含税、集团内双方口径一致，否则不能核对/合并。
14. **期末顺序固定**：各模块关账 → 生成凭证 → 过账 → **汇兑损益 → 结转损益** → 各模块结账 → 总账结账。
