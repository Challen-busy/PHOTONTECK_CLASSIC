## ① 模块定位（一句话）

把**期间费用**（员工报销/差旅/招待/办公/通讯/运费等）从**报销单**这一业务单据原子驱动，经**费用类别→费用科目映射 + 部门辅助核算**生成**凭证草稿**，草稿流到**总账**审核/过账后写 `AccountBalance`——没有本模块，利润表的「销售费用/管理费用/财务费用」缺一块、净利润算不全。

**本模块是独立模块**（与「凭证处理」「往来管理」平级），**不含薪酬**（薪酬→HR）。本模块只挂业务单据（报销单 + 付款单），**凭证的生命周期归总账**。

引擎现状一句话：报销/员工模型**全部 ❌不存在**（v1 提到的 `FlowEditor.jsx` 占位文案不构成实现）；凭证过账层 `finance.post_voucher`/红冲/反过账 **❌尚未落地**（MEMORY：唯一缺凭证过账层 `finance_posting.py`）；`AccountMappingRule`（业财翻译表）**❌未建**；`VoucherEntry` **❌无本位币双金额列、无辅助核算列**。本模块**复用** Voucher/VoucherEntry/AccountBalance/Account 表结构 + 取号 + 会签 + 通知 + 行级隔离 + 审计四本，其余按 §⑨ 标 ➕扩展 / ❌新建。

## ② 财务对象清单（统一对象范式 8 列 · 类型用 A~E 五类）

> 类型五类：A 主数据 / B 交易单据 / C 台账余额 / D 报表 / E 期末过程。
> 本模块对象：**B-费用报销单 EXPENSE_CLAIM**、**B-费用付款单 EXPENSE_PAYMENT**（出纳付款，与出纳资金同源、侧重不同；本模块视图侧重往来/费用实质）、子表 **ExpenseClaimLine**、**A-员工 Employee**（报销人锚点 + 辅助核算）、配置 **A-费用类别 ExpenseCategory**（类别→默认费用科目+默认税率字典）。
> 凭证（Voucher）**不在本模块对象清单**——它归总账（见 §⑦ 接缝，本模块只产「凭证草稿」交付总账）。

## ④ ★科目映射取数公式（业务单×动作→借/贷科目+金额取数，全走 AccountMappingRule，标【财务确认】）

> 框架（=录音「业财翻译表」）：业务 effect → `map_to_voucher_entries(db, company_id, source_doc_type, trigger_action, doc)` → 建 **Voucher 草稿**（业财同源同事务）→ 草稿交**总账**跑 `finance.post_voucher`。方向永远 `f(account.balance_direction, dr_cr_role)` 推导，**禁止 effect 写死借贷**。
> 录音「业财翻译表 = 两个映射」：可写死部分（借/贷**科目**：类别→费用科目）+ 灵活部分（**金额**取含税/不含税/税额）+ **摘要** + **辅助核算**（部门）+ **日期**（单据创建日 vs 业务发生日，可选）。本模块映射只对**报销单/付款单**做（订单类不进凭证）。
> ★分录金额双列：amount（原币）+ amount_base（本位币=amount×exchange_rate）。

### 4.1 EXPENSE_CLAIM · 生成凭证草稿（trigger_action=CLAIM，FIN_REVIEW→VOUCHER_DRAFTED）

| 行 | dr_cr | 科目 | account_source | amount_formula（原币；本位币=×汇率） | tax_handling | 辅助核算/摘要 |
|---|---|---|---|---|---|---|
| 1..n | DR | 行 `expense_account_code` | LINE | Σ同科目同 dept 行（deductible ? amount_excl : amount_incl） | EXCLUSIVE(可抵不进费用)/INCLUSIVE(不可抵进费用) | 辅助核算=dept；摘要=报销事由 |
| T | DR | 进项税额（应交税费-进项税额） | FIXED | Σ行（deductible ? tax_amount : 0） | TAX_ONLY，仅专票可抵 | — |
| L | CR | 其他应付款（REIMBURSE 垫付）/ 银行存款（即付，按付款账户辅助核算） | CONFIG by pay_method | Σ行（amount_incl） | NONE | 辅助核算=报销人（往来）/银行账户 |

平衡：Σ借(费用+进项税) = Σ贷(其他应付款/银行) ✓

> HKFRS 公司（无进项抵扣）：deductible 恒为 False → 税并入费用，无进项税行（按 company.region 切模板）。
> 科目编码以基石公司科目表为准；具体明细码（进项税额子目、其他应付款子目）**待财务确认**填 AccountMappingRule。

### 4.2 EXPENSE_PAYMENT · 付款凭证草稿（trigger_action=PAY，VOUCHER_DRAFTED→PAID）

| 行 | dr_cr | 科目 | amount_formula | 辅助核算 |
|---|---|---|---|---|
| 1 | DR | 其他应付款 | Σ行(amount_incl) | 报销人（清往来） |
| 2 | CR | 银行存款 | Σ行(amount_incl) | 付款银行账户（辅助核算维度，非独立科目；MEMORY：银行 10 账户用辅助核算不建 10 科目） |

### 4.3 红冲（已结账期更正 · 归总账 command）

★录音口径：**当月填错且未结账→直接删/改报销单 + 删凭证草稿**（不走红冲）。仅**已结账期**需更正→走总账 `finance.red_reversal`：原 voucher 不删 → 生成红字反向 Voucher（每行金额取负、source_doc_type=`VOUCHER`+source_doc_id=原 id、is_auto_generated、description=「红冲 {原凭证号}」）→总账审核过账。当月红冲=一正一负净 0，一般次月冲；系统逻辑统一不区分跨月。本模块只**触发**红冲意图，红冲生命周期归总账。

> v1 错→v2 正：删除「未结账走蓝冲反过账」整段——未结账直接删/改，无蓝冲术语。

### 4.4 暂估侧关联（费用模块边界说明）

费用报销一般**票据齐全才报销**（凭票即时），通常不涉暂估。若发生「先入账后到票」（如预提费用），按 MEMORY 暂估口径走：发票来之前按金额暂估、发票到按发票金额入账 + 红冲原暂估——但**暂估主战场在采购/应付/存货成本模块**，费用模块不重复建模，仅在跨期票据场景按所属期间（自然月）入账。

## ⑥ 校验规则 + 边界 + 错误处理（失败精确提示，绝不静默）

| 规则 | 触发点 | 精确提示（失败） |
|---|---|---|
| 报销勾稽 incl=excl+tax；tax=deductible?excl×rate:0 | EXPENSE 提交（逐行） | 「第 {n} 行含税额 {incl} 与不含税+税额 {calc} 不符」 |
| 本位币换算 amount_base=amount×rate | 生成草稿前 | 「第 {n} 行本位币金额缺失/与原币×汇率不符」（★双金额不变量） |
| 借贷平衡 Σ借=Σ贷（原币 + 本位币各自平） | 生成凭证草稿（hard_rule，全局不变量1） | 「借方合计 {x} ≠ 贷方合计 {y}，差额 {d}，禁止生成凭证」 |
| 期间锁 period.status=OPEN（未关账） | 所有生成草稿/付款边 | 「会计期间 {period} 已关账/结账，禁止操作；如需更正请走总账红冲」 |
| 期间=自然月 | period 解析 | 据 posting_date 落自然月期间；★不得用「28~次月27」类边界（v1 错口径，已删） |
| 职责分离（开关） | 复核边 | 「职责分离已开启：复核人不能等于制单人 {name}」 |
| 必输 | 提交边 | 「{字段} 必填」（行/科目/金额/期间/报销事由/票据张数） |
| 票据张数>0 | 财务复核 | 「报销单无附票，请补录票据张数后再复核」 |
| 进项可抵扣一致性 | 复核 | deductible=True 但无专票→「专票未认证，进项不可抵扣，请取消勾选或并入费用」 |
| 撤草稿前置：凭证未过账 | VOUCHER_DRAFTED→FIN_REVIEW | 已被总账过账→「凭证已过账，请先在总账反过账（逐月返）后再撤回」 |
| 直接删/改限未结账 | 报销单删/改 | 「期间已结账，不可删改；请走总账红冲」（★无蓝冲提法） |
| 红冲触发须留痕 | 触发总账红冲 | 必经 command_log + Notification，否则视为失败 |
| 公司隔离 | 全查询 | 走 `_company_filter(user)`，跨公司单据不可见/不可操作 |
| 幂等 | create_*_voucher effect | 已存在 source_doc_type+source_doc_id 对应 Voucher→no-op 返回既有（防退回重触发重复建草稿，参照 `confirm_payment_request_settlement` 幂等守卫） |

**边界**：①跨期票据（票据月≠当前期）→按 posting_date 落自然月期间入账，跨已结账期用总账更正/红冲；②负数报销（退费）→红字行/红冲；③外币报销→amount×exchange_rate 折本位币双金额，费用类按入账即时汇率，差额不进汇兑；④冲备用金（ADVANCE_OFFSET）→冲预付/挂账；⑤员工借款（特殊业务）→不在本模块，财务侧手录往来；⑥一人多公司→报销按所在 company 各自建单，绝不合账。

### 8.1 费用报销单屏（专用屏，对齐金蝶「费用报销单」）

- **关键交互**：①按类别带科目/税率/可抵；②实时合计（原币+本位币双行）；③审批流可视进度条（提交→主管→财务复核→生成凭证草稿→出纳付款）；④驳回必填原因；⑤复核前「预览凭证」弹层（借费用[+进项税]/贷其他应付款，借贷+本位币双平衡校验，平才让生成草稿）；⑥本人只见本人单。

## ⑩ 验收用例（Given-When-Then，含边界 · 已按录音口径纠正）

1. **正常报销·垫付后付（可抵专票）**：Given 内地公司员工差旅含税 1130（不含税 1000、可抵专票税 130，pay_method=REIMBURSE）→ When 财务复核生成凭证草稿→交总账过账→出纳付款 → Then 草稿「借 6601 1000 + 借 进项税额 130 / 贷 其他应付款 1130」；总账过账后付款「借 其他应付款 1130 / 贷 银行存款 1130」；借贷平衡（原币+本位币各平）。
2. **不可抵扣（普票/HKFRS）**：Given 招待费含税 1060，deductible=False → Then 「借 6602 1060（含税全进费用）/ 贷 其他应付款 1060」，无进项税行（HKFRS 公司同此口径，税并入费用）。
3. **外币报销双金额**：Given 香港公司员工 USD 报销含税 200、本位币 USD → 直接入账无折算；若垫付 HKD 200 汇率 0.128 → Then 凭证草稿原币 HKD 200 / 本位币 USD 25.6 双列，借贷两套各自平衡。
4. **边界·勾稽不平**：Given 报销行含税≠不含税+税额 → When 提交 → Then 校验拦截「第 2 行含税额 ≠ 不含税+税额」，禁止进审批。
5. **边界·当月填错（无蓝冲）**：Given 报销单已生成凭证草稿、所在期未结账、发现金额录错 → When 撤回草稿（凭证未过账）→ Then `discard_claim_voucher` 删草稿、退 FIN_REVIEW、直接改→重生成；**全程不出现蓝冲**（v1 错→v2 正）。
6. **边界·已过账未结账更正**：Given 凭证已在总账过账、期未结账 → When 撤草稿 → Then 拦截「凭证已过账，请先在总账反过账（逐月返）后再撤回」。
7. **边界·已结账期更正（红冲）**：Given 报销凭证已过账、所在期已结账 → When 需更正 → Then 报销单不可直接删改（提示「期间已结账，请走总账红冲」）；总账 `red_reversal` 生成红字反向凭证，原单不删，净额=0。
8. **边界·期间=自然月**：Given posting_date=2026-06-30（6 月末，企业 27 号已关账 6 月录入窗口）→ Then 落 6 月自然月期间；若 6 月已结账→拦截期间锁；★不得因「27 号」把期间切成「28~次月27」（v1 错口径已删）。
9. **边界·职责分离**：Given segregation_of_duties=ON，复核人=制单人（同一 FINANCE 账号）→ When 复核 → Then 拦截「复核人不能等于制单人」；OFF（小公司）→ 放行。
10. **边界·非本人报销不可见**：Given 员工 A 登录 → When 查报销台账 → Then 仅见 A 本人单，B 的单不可见（行级 owner 过滤）。
11. **红冲广播**：Given FINANCE_MANAGER 触发已结账报销红冲 → Then `Notification` 广播 recipient_role ∈ {ACCOUNTANT,FINANCE_MANAGER,FINANCE_DIRECTOR}「报销单 EX-2606-003 已触发红冲，待总账过账」。
12. **幂等**：Given 报销 FIN_REVIEW→VOUCHER_DRAFTED 后撤回又再生成 → Then create_claim_voucher 发现既有 source_doc 凭证→no-op 不重复建草稿。
13. **冲备用金**：Given 员工领备用金 5000、报销含税 3000（ADVANCE_OFFSET）→ Then 借 费用 3000 / 贷 预付（备用金）3000，不再走出纳付款。

## ⑫ 薪酬边界说明 + 发薪凭证如何入账（★只做边界，不在本模块实现）

> 录音铁律：**薪酬不与费用合并**，归 HR（HR 可独立做产品，含考勤/假期/薪酬，薪酬只是其一小部分；员工档案/工资结构/福利/个税）。本 PRD（财务核心）**只做边界说明 + 发薪凭证如何入账**，不建 PAYROLL_RUN 模型/状态机/effect。v1 在此模块建的整套薪酬对象（PAYROLL_RUN/PayrollLine + 计提/发放状态机 + payroll.* effect）**全部移出本模块**。

**边界**：
- 员工档案、工资结构、福利、个税、考勤、假期 → **HR 模块**（独立产品），不在财务核心。
- 财务核心只关心：薪酬最终如何**变成凭证入账**（落到费用/应付职工薪酬/应交税费）。

**发薪凭证如何入账（仅口径，落地在 HR→总账接缝或财务手录）**：
- 计提（权责发生）：借 费用（6601/6602/6603，按受益对象/部门归口）/ 贷 应付职工薪酬。
- 发放（收付实现）：借 应付职工薪酬 / 贷 银行存款 + 贷 应交税费（代扣个税）+ 贷 其他应付款（代扣社保个人部分）。
- 单位社保公积金计入费用、个人部分从应发扣回经其他应付款代缴（CAS 明细科目待 HR/财务确认；HKFRS 无强制社保科目，按 region 模板）。
- 来源：HR 模块生成发薪业务单据 → 经业财翻译表生成**凭证草稿** → 流到**总账**过账（与费用模块同范式，凭证生命周期归总账）；或财务作为「特殊业务手录凭证」（L400 口径，HR 未上线时）。
- ★本模块不实现以上，仅记录边界与入账口径，供 HR 模块对接时复用同一套「业财翻译表 + 总账过账」框架。
