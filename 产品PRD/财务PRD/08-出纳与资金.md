# 08 · 出纳与资金（Cashier & Treasury）

> 本模块 = 财务核心模块之一，**挂业务单据，不挂凭证**（★录音模块拆分铁律：只有"总账"挂凭证，其他所有模块只挂业务单据 → 单据经凭证模板/业财翻译规则生成凭证草稿 → 流到总账统一审核/复核/过账）。
>
> **范围（录音锁定口径）**：真实的钱进钱出——**收款单 + 付款单**（真实收付）；现金日记账 / 银行日记账（与 1001 库存现金 / 1002 银行存款 勾稽）；资金调拨（同公司账户间转钱）；银行对账（与银行流水勾对、出余额调节表）；票据（应收/应付票据生命周期）。
>
> **与"应收应付"的关系（★录音原话，本模块定位的核心）**："出纳与资金"和"应收应付"**同源、侧重不同**——
> - **应收应付模块**关注：往来单位 / 业务实质 / 摘要（钱"为什么"动、对谁动、欠多少）。
> - **本模块（出纳）**关注：付到/收自**哪个账户**、**哪个银行**、**什么收付方式**（转账/支票/电汇…）、**银企直连**、**现金日记账 / 银行日记账**（钱"实际"怎么动、流水登账、勾稽对平）。
> - 一笔收款/付款单可由应收应付侧生成，也可出纳侧直接登；二者**不双重记账**，靠 `source_doc` 回链同一笔业务。**连不上银行 → 网银手动付**（银企直连是增强，不是前提）。
>
> **凭证生命周期不在本模块**（★录音纠正 v1 最大错误）：本模块单据只生成**凭证草稿**经业财翻译规则推到总账；审核 / **复核（资金类多一步）** / 过账 / 反过账 / 反审核 / 红冲，**全部属于总账模块**，本 PRD 只画"单据 → 凭证草稿"的接缝，不在本模块画凭证状态机。
>
> **会计期间 = 自然月**（★录音纠正）：每月 1 日~月末，全球准则统一、**不可按公司切换**、**无 period_cutoff_day**。所谓"27 号"是**关账动作时点**（人为锁当月录入窗口），不是会计期间边界——本模块所有日记账/收付单的"期间锁"判定基于**自然月期间 + 关账标记**，绝不按 28~次月 27 切。
>
> **引擎现状（已 grep 核实 `backend/models.py` 2026-06-17）**：
> - ✅已有模型：`BankReceipt`（银行收款流水 `bank_receipt`，status=UNALLOCATED）、`NotesReceivable`（应收票据 `notes_receivable`，status=HELD）、`ARSettlement`（核销，可挂 `bank_receipt_id`/`note_id`）、`AdvanceReceipt`/`AdvancePayment`/`PaymentRequest`（预收/预付/付款申请）、`Account`/`Voucher`/`VoucherEntry`/`AccountBalance`/`ExchangeRate`/`AccountingPeriod`、`Notification`（recipient_role 广播）。
> - ➕扩展：见 §⑨。★**关键修正**：`VoucherEntry` 现仅有 `debit`/`credit` **一个金额** + `currency`/`exchange_rate` ——录音要求**原币金额 + 本位币金额两个都给**（外币时），故凭证行的"双金额"是 ➕扩展（在总账模块落地，本模块只负责在收付单上同时带出原币 amount 与汇率供翻译规则取数）。
> - ❌需新建：`BankAccountDim`（★见 §②对象 2.1 的口径变更——银行账户做**辅助核算维度**而非独立主数据+10 科目）、`CashReceipt`/`CashPayment`（收款单/付款单，真实收付）、`CashJournal`/`BankJournal`（现金/银行日记账，本模块**派生账本视图**为主）、`BankReconciliation`+`BankStatementLine`、`FundTransfer`、`NotesPayable`。全部纯扩展（`@register_command`/`@register_transition_effect`/`@register_transition_validator` + `__doc_types__` + `WorkflowDefinition.states`），core/registry.py / workflow.py / commands.py 字节级零 diff。

标注口径：✅引擎已有 / ➕扩展现有 / ❌需新建。

## ① 模块定位

出纳侧重"钱实际怎么动"（账户/银行/收付方式/日记账），与"应收应付"同源、侧重不同（录音 L363-364）。本模块只管**货币资金的真实收付 + 登账 + 对账 + 调拨 + 票据保管**，全部以**业务单据**承载（"出纳是钱的看门人、会计是账的记录者"的职责分离硬约束录音未提，见用例 12 标注"待财务确认"）：

- 真实收款 → **收款单（CashReceipt）**；真实付款 → **付款单（CashPayment）**。来源既可由应收应付侧带出（关注往来/实质），也可出纳手工补录（关注账户/方式）。
- 每笔收付即时进**现金日记账 / 银行日记账**逐笔登账（出纳视角的流水账本）。
- 收付单经**业财翻译规则**生成凭证草稿 → 推送**总账**统一审核（资金类含复核）/过账，过账后写 1001/1002 科目余额。
- 月末出纳导入/录入银行流水，与银行日记账逐笔勾对，找未达账项，出**余额调节表**证明账实相符。
- 资金调拨管同公司账户间转钱（行内/跨行/提现/存现/购结汇）；票据管应收/应付票据全生命周期。

**核心勾稽（本模块验收红线，过账之后成立）**：
```
现金日记账期末余额        == 1001 库存现金 科目余额
银行日记账期末余额（按账户维度汇总）== 1002 银行存款 科目余额
余额调节表：调节后双方相等
```

> 银行账户**不作 1001/1002 的余额唯一来源**——本模块日记账是出纳侧逐笔流水账本（派生视图），科目余额仍由总账过账（`finance.post_voucher`）唯一写入（不变量：财务唯一写入路径=凭证过账）。

## ② 对象清单（对象范式）

> 类型枚举：A 主数据 / B 交易单据 / C 台账余额 / D 报表 / E 期末过程。本模块**无 E**（期末过程归"期末结账"模块），**无凭证对象**（凭证归总账）。

### 对象 2.1 · 银行账户 = 辅助核算维度（★录音重大纠正：v1 错→v2 正）

> **v1 错**：v1 把 BankAccount 建成独立主数据 + 给每个账户绑一个 1002 下明细科目（10 个账户→10 个明细科目）。
> **v2 正（录音 L16-70）**：专家明确"银行存款名下 10 个账户**不建 10 个科目**，而用**辅助核算维度**挂账户"。→ 银行账户应是 **1002 科目上的一个辅助核算维度值**（科目级可挂 1~2 个维度，系统可配），不是独立科目，也不必为每户造明细科目。

| 列 | 内容 |
|----|------|
| ① 对象 | `BankAccountDim`（model `bank_account_dim`，作为辅助核算维度"银行账户"的维度值表）；`__queryable__=True`（A 主数据，无 doc_type，不走状态机） |
| ② 类型 | **A 主数据 / 辅助核算维度值**。复用通用壳 DataExplorer（左列表右明细）+ 辅助核算维度框架 |
| ③ 数据 | 见 §3.1。关键：account_no 银行账号、bank_name 开户行、currency 户币种、is_cash（现金户标记）。**不再有 gl_account_code**（不绑独立科目）——它本身就是挂在 1001/1002 上的辅助核算值；日记账/收付单/对账/调拨经 `bank_account_id` 引用维度值 |
| ④ 动作×角色×时机 | 新增/改/停用：`CASHIER`+`FINANCE_MANAGER`（建账期维护）。已发生流水的账户不可删，只可停用（is_active=False） |
| ⑤ 逆操作 | 不冲销。误建→停用 |
| ⑥ 效果 | 启用后进入日记账/收付单/调拨/对账的账户下拉池；凭证草稿在 1001/1002 行上挂该辅助核算值 → 科目余额可按账户维度下钻（=出纳要的"勾稽到具体账户"，但不靠造科目） |
| ⑦ 回链 | CashReceipt/CashPayment/CashJournal/BankJournal/FundTransfer/BankStatementLine 经 `bank_account_id` 反链；凭证行经辅助核算维度反链 |
| ⑧ 准则/账簿 | CAS/HKFRS 一致；`company_id` 账套隔离（绝不跨公司用账户）；现金户绑 1001、银行户绑 1002（用辅助核算区分户，不区分科目） |

## ⑥ 校验规则 + 边界 + 错误处理（失败精确提示，绝不静默）

**全局不变量落地（逐条，按录音口径）**：
1. 借贷平衡：凭证草稿过账前 Σ借=Σ贷（过账校验在**总账**，本模块生成草稿时即先自检），不平提示「凭证不平衡，借 X 贷 Y 差额 Z」。
2. 过账后不可改：收付单已挂总账凭证且**凭证已过账**后，金额/账户/方向只读；改须先在**总账**反过账/反审核（逐月返）或红冲。★本模块不提供"蓝冲"。
3. 期间锁：journal_date 所属 AccountingPeriod（**自然月**）status=CLOSED **或该期已关账**时禁登账，提示「该期间已结账/已关账，请用次期或调整凭证」。★期间边界=自然月，**绝不按 27 号切**。
4. 业财同源：收付单确认 effect（落日记账 + 生成凭证草稿）同事务原子（要么都成要么都回滚）。
5. 财务唯一写入路径：日记账 running balance 派生（查询 Σ），1001/1002 科目余额只由总账 `finance.post_voucher` 写 AccountBalance；本模块**不写 AccountBalance**。
6. 审计留痕：每次登记/勾对/调拨/票据动作经 execute_transition→WorkflowLog + CommandLog；反结账/反过账由总账留痕（谁/何时/反哪期）。
7. 字段防火墙：本模块无成本/买价字段，不涉及销售毛利隐藏。
8. 公司隔离：所有对象 company_id NOT NULL，查询走 `_company_filter(user)`；调拨 from/to 必须**同 company_id**（跨公司报错「资金调拨不可跨公司，跨公司请走往来挂账」）。
9. 含税一致（往来）：收付单 counterparty 涉往来时金额**含税**且双方一致（录音 L538-558：往来一定含税且一致，否则核对/合并对不上）。
10. 业务红冲广播：票据退票（DISHONORED）+ 错误调拨红冲 → 生成红字凭证草稿（推总账）+ Notification 广播 recipient_role=FINANCE/CASHIER/ACCOUNTANT/FINANCE_MANAGER。★红冲=原单不删、生成反向单据；系统逻辑统一不区分跨月（当月红冲一正一负净 0，一般次月冲）。

**对象级校验**：

| 校验点 | 规则 | 错误提示 |
|--------|------|---------|
| 收付金额 | amount>0 | 「金额必须大于 0」 |
| 账户类型匹配 | 现金日记账→is_cash=True；银行日记账→is_cash=False | 「现金收付须选现金账户」 |
| 余额勾稽 | 日记账期末余额≠对应科目余额（过账后） | 对账时 WARN「日记账与 1001/1002 差 X，请核查未过账单据」 |
| 调拨账户 | from≠to 且同公司 | 「转入转出账户不能相同」/「资金调拨不可跨公司」 |
| 调拨余额 | from 余额≥amount+fee（可配硬拦） | 「转出账户余额不足，当前 X 需 Y」 |
| 对账平衡 | adjusted_balance 双方相等（容差 0.01） | 「对账未平，调节后差额 X，请检查未达账项/漏勾对」 |
| 勾对一致 | statement_line.amount==journal.amount 且方向一致 | 「勾对金额/方向不一致，无法勾对」 |
| 票据到期 | 托收 journal_date≥maturity_date；贴现日<maturity_date | 「未到期不可托收」/「已过期不可贴现」 |
| 期间/关账锁 | 所属自然月期间 OPEN 且未关账 | 「期间已结账/已关账，禁止操作」 |

**余额调节表公式（4 类未达账项）**：
```
银行端调节：对账单余额 + (③企业已收银行未收) − (④企业已付银行未付) = 调节后余额
企业端调节：日记账余额 + (①银行已收企业未收) − (②银行已付企业未付) = 调节后余额
两端调节后余额必须相等（容差 0.01）→ 否则禁完成对账
```
> ①②（银行已动、企业未登）是出纳次月需补登收付单/日记账的；③④（企业已登、银行滞后）无需补账，下月银行自然到账后自动平。

**边界**：①外币户日记账原币+折算双记，期末货币性项目调汇（汇兑损益）归**期末结账**模块，本模块只保证原币余额准确、按当月维护汇率折本位。②跨期对账（流水跨自然月）→按 statement_date 归期，①②类未达项次月补登。③一笔银行收款核销多张发票（ARSettlement 一对多）→收款单一笔、核销多行，凭证草稿借 1002 一行、贷 1122 按客户多行。④调拨手续费=0 时不生成费用行。⑤连不上银行（bank_link_status=MANUAL）→付款单走网银手动付，出纳事后回填到账确认。

## ⑩ 验收用例（Given-When-Then，含边界）

1. **现金收款登记→勾稽**
   Given 客户现金回款 ¥5000 核销 AR#88；When CASHIER 登收款单 IN ¥5000（来源 AR_SETTLEMENT，现金户），确认 → 落现金日记账 + 生成凭证草稿借1001/贷1122 推总账，会计审核过账；Then 现金日记账期末结存 +5000 == 1001 科目余额变动，勾稽徽标绿。

2. **银行付款外币双金额折算**
   Given 付供应商货款 USD 10000、当月汇率 7.10、本位币 CNY；When CASHIER 登付款单 OUT amount=10000 currency=USD rate=7.10（来源 PAYMENT_REQUEST，转账方式），确认；Then 凭证草稿借2202/贷1002 **原币各 USD 10000 + 本位币各 CNY 71000 两个金额都给**，1002（USD 户辅助核算）原币减 10000。

3. **资金调拨同公司不影响总资产**
   Given 招行 CNY 户转工行 CNY 户 ¥100000；When FUND_TRANSFER BANK_TO_BANK 审批 EXECUTED；Then 凭证草稿借 1002(工行辅助核算)/贷 1002(招行辅助核算) 各 10 万（同科目不同辅助核算值），总货币资金不变，两户日记账各一笔。

4. **跨币种购汇折算差额（边界）**
   Given USD 户购 CNY，amount=USD 1000、rate=7.10、to_amount=CNY 7100，手续费 CNY 20；When EXECUTED；Then 借 1002(CNY)7100 + 6603 手续费 20 / 贷 1002(USD 折本位)+差额轧入 660303 汇兑损益使借贷平。

5. **银行对账 4 类未达账项（核心边界）**
   Given 对账单余额 ¥120000、日记账余额 ¥118000；存在「企业已付银行未付」¥3000 +「银行已收企业未收」¥1000；When CASHIER 勾对算调节表（企业端 118000+1000=119000，银行端 120000−3000=117000）；Then 系统检测不平差 2000，红色「对账未平，请检查漏勾对/未达项」，禁提交。

6. **银行对账平衡通过**
   Given 调整未达项使两端调节后均 =117000；When CASHIER 完成对账；Then status=BALANCED，绿「已平」，FINANCE_MANAGER 复核。

7. **票据贴现（边界）**
   Given 持有银行承兑 ¥100000、maturity 90 天后、贴现率 3.6%/年；When CASHIER 申请贴现、FINANCE_MANAGER 复核、推总账过账；Then 凭证草稿借 1002 实得 ¥99100 + 6603 贴现息 ¥900 / 贷 1121 ¥100000（天数基准待财务确认）。

8. **票据退票广播（红冲口径）**
   Given 商业承兑到期托收被拒（DISHONORED）；When CASHIER+ACCOUNTANT 处理；Then 生成**红字反向凭证草稿**借 1122/贷 1121 ¥100000（★红冲=原单不删、反向单据，不区分跨月）+ Notification 广播 FINANCE/CASHIER/ACCOUNTANT/FINANCE_MANAGER「票据 NR-xxx 退票」。

9. **期间锁=自然月（边界，★纠正）**
   Given 6 月**自然月**期间已 CLOSED（或已关账）；When CASHIER 试图登 6 月收付单；Then 报错「该期间已结账/已关账，请用次期或调整凭证」。★若有人把期间设成"5/28~6/27"则视为错误配置——期间必须自然月。

10. **过账后改账拦截（不变量 2，★无蓝冲）**
    Given 收款单已挂总账凭证且**凭证已过账**；When CASHIER 改金额；Then 字段只读、提示「已过账不可改，请在总账反过账/反审核（逐月返）或红冲」——★不出现"蓝冲"字样。

11. **跨公司调拨拦截（不变量 8）**
    Given from=公司 A 账户、to=公司 B 账户；When 提交调拨；Then 报错「资金调拨不可跨公司，跨公司请走往来挂账」。
