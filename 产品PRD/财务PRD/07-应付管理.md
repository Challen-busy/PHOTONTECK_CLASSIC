> **★模块拆分铁律（录音 L334-526）**：本模块是「应付管理」——**只挂业务单据（应付单+付款单+结算单+暂估应付），不挂凭证**。凭证的整个生命周期（新增→审核→[资金类复核]→过账→反过账→红冲）**只属于「总账」模块**。本模块的业务单据经「凭证模板/业财翻译规则」生成凭证草稿，流到总账统一审核过账。
>
> **v1错→v2正 · 全局四纠**：
> 1. **删除「蓝冲」**（录音 L316-319 专家原话「你说的蓝冲我不太懂是什么意思，我只听过红冲」）。逆操作改为三件事：①当月填错（业务错误）→直接删/改凭证（未结账可删，不走红冲）；②反过账/反审核=状态逆向校验（逐月返）；③红冲=原单不删、生成反向单据（认可原业务、金额要更新）。v1 自称「WebSearch 核实 K3 标准蓝冲vs红冲」=错。
> 2. **凭证生命周期不下放本模块**（v1 把审核/过账/反过账写进 PI/AP 对象=错）。本模块只「生成凭证草稿 + 回链 voucher_id」，审核过账动作归总账。
> 3. **凭证金额=原币+本位币双金额**（录音 L223「借贷方要给原币金额，也要给本位币金额」），v1 多处只写「外币折本位币」隐含汇率推算=不够，应**两个金额都存**。
> 4. **采购发票（PurchaseInvoice）不属于本模块**——录音 L412/L430 明确「销售发票挂应收、采购发票挂应付，由独立『税务』模块承载」。本模块**消费**采购发票的审核结果（生成应付/暂估冲回），但发票录入/扫描提取/税局对接放【09 税务模块】。本稿仅描述「采购发票→应付」接缝，不再把 PI 当本模块主对象。

## ① 模块定位

**应付管理 = 采购履约「钱端」的往来真相源**（挂业务单据，不挂凭证）。链路：货到即**暂估应付**（财务以发票为确认，发票来之前不能不入账，按订单金额暂估——录音 L286）→ 采购发票到达（税务模块）→ 财务审核**冲回暂估、转实应付**（红冲暂估、按发票金额入账，录音 L286/L295）→ **应付台账**记欠多少/何时到期 → **付款单 + 结算单**核销（真实收付的「打款到哪个账户/哪个银行」动作侧重在【08 出纳与资金】，本模块侧重往来单位/业务实质/到期）→ **账龄与到期债务监控** + **供应商信用占用/释放**。

每张应付单/付款单/结算单经凭证模板生成凭证草稿，由总账统一审核过账写 `AccountBalance`。**含税铁律（录音 L553）**：往来款一定含税且双方一致（不可一家含税一家不含税），往来一致才能与集团内单位勾对、支撑合并报表。

**与相邻模块边界**：
| 事项 | 归属 | 录音依据 |
|------|------|---------|
| 凭证审核/复核/过账/反过账/红冲 | **总账** | L334「只有总账挂凭证」 |
| 采购发票录入/扫描提取/税局开真票/进项税确定 | **税务模块** | L430/L442 |
| 真实付款（银企直连/网银手动付/银行日记账） | **出纳与资金** | L361/L364 |
| 内部单位 A-B 往来勾对/合并支撑 | **往来管理（独立小模块）** | L241/L259 |
| 员工借款等特殊业务 | 本模块**手录应付单** | L400「财务有常规业务之外的特殊业务，比如员工借款」 |

## ④ ★科目映射取数公式（业务单×动作 → 借/贷分录）

> 全部经「**凭证模板/业财翻译规则**」配置（录音 L508-526：两个映射=业务单据↔凭证，运维自由配，不动开发）。可配项=借贷方科目（可写死）+金额（灵活：取入库单成本/发票含税or不含税or税额）+摘要+辅助核算+**日期（单据创建日 vs 业务发生日，可选）**。**借贷方向由科目「余额方向」自动推导**（录音 L29 最关键属性），禁止 effect 写死方向。
> **取数**：INCLUSIVE=含税额；EXCLUSIVE=不含税额=amount/(1+税率)；TAX_ONLY=税额=amount/(1+税率)×税率；NEGATE=红字取负。**凭证每行原币+本位币双金额**（录音 L223）。
> **税口径（录音 L553）**：往来含税且双方一致；税进不进成本的区别=可抵扣→不进成本、不可抵扣→进成本；**暂估不暂估进项税**（暂估只估资金侧、税侧发票到才确认，录音 L286「资金跟税是两件事，分开讲」）。

## ⑥ 校验规则 + 边界 + 错误处理（失败精确提示，绝不静默）

| # | 规则 | 触发动作 | hard_rule 精确条件 | 失败提示 |
|---|------|---------|-------------------|---------|
| 1 | 借贷平衡 | 凭证过账前（总账）| Σ借=Σ贷（原币与本位币各自平）| 「凭证不平：借{x} 贷{y} 差额{d}，禁过账」 |
| 2 | 货到无票才暂估 | 应付单暂估 | 无对应已审采购发票时方可暂估 | 「该笔已有发票，请走转实而非暂估」 |
| 3 | 发票与暂估一致性 | 转实确认 | abs(发票额−暂估额)/暂估额 ≤ 容差%（公司可配）| 「发票额{x}与暂估{y}差异超容差，请核对或确认差异入成本」 |
| 4 | 暂估冲回幂等 | 转实确认 | is_provisional=True 且未冲回 | 「该暂估已冲回，重复操作被拦截」 |
| 5 | 付款额≤应付余额 | 付款/核销 | amount ≤ (应付 amount − paid_amount) | 「付款额{x}超应付余额{y}」 |
| 6 | 往来含税一致 | 内部往来勾对 | 双方均含税、口径一致（录音 L553）| 「内部往来须双方含税一致，无法勾对」 |
| 7 | 信用额度占用（**录音无依据·待财务确认**，见对象5注）| 暂估/转实 | used_amount+本次 ≤ credit_limit（换本位币）| 「供应商{s}信用超额：{u}+{n}>{l}」（达阈值发预警）|
| 8 | 期间锁 | 任何过账（总账）| AccountingPeriod=OPEN（**自然月**，27号关账≠期间边界，录音 L88）| 「{period}期已结账/已关账，跨期请反结账或下期处理」 |
| 9 | 职责分离 | 制单/复核/付款 | seg=ON 时 created_by≠reviewer≠cashier | 「制单/复核/付款须不同人（职责分离已开）」 |
| 10 | 反过账逐月返 | 反过账（总账）| 只能反当前最新已结期，不能跨月返（录音 L319）| 「只能反{latest}期，不能反{target}期」 |
| 11 | 当月填错走删改 | 业务错误更正 | 未结账→直接删/改单+重生凭证（**非红冲**，录音 L304）| 「当月业务错误请直接删改并重新生成凭证」 |
| 12 | 外币双金额必填 | 外币应付/付款 | currency≠本位币 时 exchange_rate>0 且 amount_base>0 | 「外币业务须录原币+本位币两个金额」 |

**边界**：①一票多付（PARTIAL 累加 paid_amount 至 PAID）；②一付多票（结算批 APSettlement）；③跨期付款（汇差进 660301）；④暂估差异为负（发票<暂估，红字调存货）；⑤预付转抵后尾款（部分抵+部分付）；⑥已结账期红冲跨年走 6901 以前年度损益调整；⑦内部单位往来勾对（A 对你应收/你对 A 应付→一家做、另一家复核，录音 L241，归往来管理协同）。

## ⑩ 验收用例（Given-When-Then，含边界）

**UC1 货到无票暂估**：Given PO 已下单、GR 入库、无发票 / When 财务暂估入账 / Then 建暂估应付单 is_provisional=True、生成暂估凭证草稿（借 1405 / 贷 2202xx 不含税估价，**不暂估进项税**），占用供应商信用，invoice_number=GR 临时号。

**UC2 采购发票审核冲回转实（平账）**：Given 暂估应付存在、税务侧发票审核通过、发票额=暂估额 / When FINANCE_MANAGER 转实确认 / Then 一笔凭证草稿：借 2202xx 暂估取负（红字）、借 222101 进项税、贷 2202 应付账款（含税），借贷平衡，is_provisional=False、voucher_id 回填，信用占用从暂估转实净额不变。

**UC3 暂估差异（发票>暂估）**：Given 暂估 1000、发票不含税 1050 / When 转实 / Then 借 1405 差异 +50 调存货，平账。

**UC4 暂估差异为负（发票<暂估）**：Given 暂估 1000、发票不含税 950 / When 转实 / Then 借 1405 差异 -50（红字冲存货），平账。

**UC5 货后付款核销**：Given 应付余额 1000、付款单 1000 已审、出纳已打款 / When 到账确认 / Then 付款凭证草稿借 2202 贷 1002，应付 paid_amount=1000 status=PAID，释放信用，confirmed=True；凭证流总账审核过账。

**UC6 一票多付（部分）**：Given 应付 1000 / When 首付 600 / Then PARTIAL、paid_amount=600；再付 400→PAID。幂等防重复递减。

**UC7 外币付款汇差（汇率二选）**：Given USD 应付入账汇率 7.0、结算汇率 7.1、付 100 USD / When 到账确认 / Then 借 2202 700、贷 1002 710、借 660301 汇兑损益 10（本位币 CNY），**原币 100 USD + 本位币两列都记**，平账。

**UC8 信用超额拦截**：Given 额度 10000、已占 9500、新暂估 800 / When GR 暂估入账 / Then 报错「信用超额：9500+800>10000」，转采购侧预付；占用达 80% 阈值发预警。

**UC9 采购退货业务红冲（必广播）**：Given 进项已过账、采购退货 / When 退货红字处理 / Then 顺 source 找原进项凭证生成红字草稿（2202/1405/222101 全 NEGATE，**原单不删**），站内通知全部财务（ACCOUNTANT/FINANCE_MANAGER/FINANCE_DIRECTOR），总账审核过账后净额=0。

**UC10 当月填错走删改·非红冲**：Given 当月暂估应付单录错供应商、未结账 / When 更正 / Then **直接删/改该单并重新生成凭证**（不走红冲，录音 L304）；若已过账→总账先反过账（逐月）再删改。

**UC11 反过账逐月返**：Given 7月已结账、6月已结账 / When 试图反过账 6 月 / Then 拦截「只能反7月，不能反6月」（录音 L319）。

**UC12 预付转抵**：Given 预付 200（借1123）、应付转实 1000 / When 结算单核销 / Then 应付转实贷 2202 1000、预付转抵借 2202 200/贷 1123 200，尾款 800 走付款单。

**UC13 员工借款手录**：Given 财务发生员工借款特殊业务（录音 L400）/ When 手录应付单 source_type=EMPLOYEE_LOAN / Then 建应付单+对应凭证草稿，往来辅助挂员工，不依赖 PO。

**UC14 内部往来含税一致**：Given 集团内 A（内地含税）对你应付、你对 A 应收 / When 勾对 / Then 须双方均含税一致方可核对（录音 L553），一家做、另一家复核（录音 L241），支撑合并。
