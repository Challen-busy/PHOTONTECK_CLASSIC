# 06 · 存货核算与成本（Inventory Costing）子模块 PRD

> 对齐 Layer0 基石契约 v1（2026-06-17 封板）。科目码/状态码/角色/取数格式以基石 digest 为唯一来源；与基石冲突=停下讨论。
> 本模块属 **Layer1 交易模块**：只负责把「存货数量×成本」事实转成**凭证过账**写入 AccountBalance；过账统一调 `finance.post_voucher`（基石❌新建），映射统一走 `AccountMappingRule`（基石❌新建）。

---

## ① 模块定位（一句话业务价值）

**让 6 家贸易公司在发货当下就能看见利润点**——出库即按移动加权平均结转暂估成本（销售毛利实时可见），月末再用实际成本差异冲回调平，把"成本事后才算清"（金蝶坑）压成一个可审计、可红冲、可勾稽的存货核算闭环；存货科目余额永远 = 存货收发存明细账。

引擎现状一句话：**WMS 数量×价值台账（移动加权）已完整实现并在状态机里跑**（`wms.apply_goods_receipt_costs`/`wms.apply_shipment_costs` 更新 `InventoryValuation` + 写 `InventoryTransaction`），但 **`InventoryTransaction.voucher_id` 恒为空、从不生成 Voucher、从不写 AccountBalance**——本模块 = 给这条已存在的成本数据流**补一座通向总账的桥（凭证过账层）+ 暂估/差异/红冲三套会计政策**。

## ② 财务对象清单（统一对象范式 8 列 + 拓展）

### 对象 1 · InventoryValuation 存货计价台账（C 台账派生只读）

| 列 | 内容 |
|---|---|
| ①对象 | `InventoryValuation` model；非 doc_type（`__queryable__`），无独立状态机 |
| ②类型 | **C 台账余额派生只读**（移动加权当前成本的权威投影） |
| ③数据 | `id` / `company_id`(FK,NN) / `material_id`(FK,NN) / `cost_method` String(15) 默认`WEIGHTED_AVG`（基石决策=移动加权；labels 值集 WEIGHTED_AVG/MOVING_AVG/FIFO，本项目锁 **MOVING_AVG 语义**——逐笔即时重算，UI 文案应纠为"移动加权平均"，见 gap) / `current_unit_cost` Numeric(16,4) / `total_quantity` Numeric(16,2) / `total_value` Numeric(16,2) / `last_updated`。唯一约束 `(company_id, material_id)`。辅助维度：物料、公司（账套） |
| ④动作× | **无人工动作**。仅由 `_apply_valuation_delta()` 在入/出/调/差异冲回时自动重算：`new_unit=new_value/new_qty`（qty=0→unit=0）。读：会计/出纳/主管/BOSS（成本字段对 SALES+SALES_ASSISTANT **字段防火墙隐藏**） |
| ⑤逆操作 | 无（派生台账，靠源 InventoryTransaction 回滚/红冲重算；负库存保护：new_qty<0→clamp 0，**留警告**见校验） |
| ⑥效果 | 被动被 `wms.py` 写，不挂 effect |
| ⑦回链 | `material_id`→Material；与 InventoryTransaction 同 `(company_id,material_id)` 勾稽 |
| ⑧准则 | 计价口径 CAS/HKFRS 一致（移动加权双标准通用）；company_id 账套隔离 |
| 拓展 | **★平销返利**：返利使采购成本事后下调→不在本台账即时反映，走月末"成本调整凭证"冲回 `total_value`（见对象4 + ⑪gap） |

### 对象 2 · InventoryTransaction 存货核算流水（B 交易凭证驱动 + C 流水）

| 列 | 内容 |
|---|---|
| ①对象 | `InventoryTransaction` model；`__doc_types__=("INVENTORY_COSTING",)` 有 seed 状态机 |
| ②类型 | **B 交易凭证单据**（每笔收/发/调/差异冲回一行，是凭证过账的取数明细源） |
| ③数据 | `id` / `company_id`(FK,NN) / `material_id`(FK) / `warehouse_id`(FK) / `transaction_type` String(20)（**值集 IN/OUT/ADJUST**；➕需扩 **PROVISIONAL_IN 暂估入 / VARIANCE 差异冲回 / RED_REVERSAL 红冲**）/ `transaction_date` Date(index) / `quantity` Numeric(16,2) abs / `unit_cost` Numeric(16,4) / `total_cost` Numeric(16,2) / `reference_type` String(30)（GOODS_RECEIPT_LINE/SHIPMENT_LINE/STOCK_ADJUSTMENT…）/ `reference_id` BigInt / **`voucher_id`** FK→voucher **nullable（现状恒空=核心缺口）** / `period_id`(FK,NN) / `status` String(15) 默认`START`(index) / `created_at` |
| ④动作× | 自动产生（业务 effect 调 `_apply_valuation_delta`）；**过账动作**=会计/主管对该期流水批量生成凭证→审核→过账（INVENTORY_COSTING 状态机现仅"概念流"无 effect，➕需补"生成凭证"挂 `finance.create_voucher_from_inv_costing`）。前置：所属期 OPEN |
| ⑤逆操作 | 蓝冲（未结账：反过账删 Voucher+本流水 status 回 START）/ 红冲（已结账：生成 `transaction_type=RED_REVERSAL` 负数行 + 配对蓝字重做）；作废=反向 ADJUST |
| ⑥效果 | ➕`finance.create_voucher_from_inv_costing`（按期+科目方向聚合生成存货凭证草稿）→`finance.post_voucher` 过账写 AccountBalance，回填 `voucher_id` |
| ⑦回链 | `reference_type+reference_id`→源业务行（GR/SHIPMENT/SA 行）；`voucher_id`→Voucher；Voucher.`source_doc_type=INVENTORY_COSTING`/`source_doc_id`=本行或批次；`command_log_id` 在 CommandLog |
| ⑧准则 | 移动加权双标准；company_id 账套；暂估/实际差异口径 CAS=暂估入账实际冲回（基石决策5） |
| 拓展 | `status` 流转复用 seed 的 INVENTORY_COSTING 18 边概念图，但**真正过账只用其中"生成凭证→总账"两段**，其余核算环节（自制/委外/材料出库）贸易公司不用→**做薄保留占位** |

### 对象 3 · ProvisionalPayable 暂估应付（B 交易凭证）— ➕需新建（薄）

> 基石决策5：**采购入库即暂估应付**；进项发票审核时冲回暂估。本模块负责"暂估入账+冲回"的**存货侧凭证**，应付侧台账由 04 应付模块持有。两侧同事务、互为接缝。

| 列 | 内容 |
|---|---|
| ①对象 | 复用 `AccountsPayable`（基石✅已有，加 `is_provisional` 标记，见 04 模块）+ 本模块只产生**暂估存货凭证** |
| ②类型 | B 交易凭证（无独立 model，凭证由映射规则产出） |
| ③数据 | 取数自 GoodsReceiptLine（actual_quantity × 估价单价）；估价来源优先级：**PO 行 unit_price → 合同价 → 上次入库 current_unit_cost → 手工估**（见⑪gap 需财务确认优先级） |
| ④动作× | GOODS_RECEIPT→STOCKED_IN 自动触发暂估入账（会计/系统）；进项发票审核→冲回（会计/主管） |
| ⑤逆操作 | 进项发票审核=暂估金额取负冲回 + 实际进项重入（NEGATE）；入库红冲=暂估行红字 |
| ⑥效果 | ➕`finance.create_provisional_payable_voucher`（入库时）/ ➕`finance.reverse_provisional_on_invoice`（发票审核时） |
| ⑦回链 | GoodsReceipt→ProvisionalAP→PurchaseInvoice 三段回链；voucher_id 双向 |
| ⑧准则 | CAS/HKFRS：估价入账（IAS2/CAS1 存货按成本，发票前用合理估计），发票实际差异调存货成本/损益 |
| 拓展 | **差异去向**：货已售出→差异进 6401 主营业务成本；货仍在库→差异调 1405 库存商品（见④映射 + gap） |

### 对象 4 · InventoryCostAdjustment 存货成本调整凭证（B 交易凭证）— ➕需新建

> 期末实际成本差异冲回（基石决策4）+ **平销返利**事后成本下调，统一走这张。

| 列 | 内容 |
|---|---|
| ①对象 | 复用 `Voucher`（voucher_type=`COST_ADJ`），不新增 model；明细=新增 `transaction_type=VARIANCE` 的 InventoryTransaction |
| ②类型 | **E 期末过程批处理**（月末跑差异 + 不定期跑返利） |
| ③数据 | 头=期间/调整原因（VARIANCE_REVERSAL 实际差异 / REBATE 平销返利 / VALUATION_LOSS 跌价）；行=物料×差异金额（实际成本−暂估成本）×（在库/已售拆分） |
| ④动作× | 会计编制（期末向导触发或返利单触发）→主管审核→过账。前置：期 OPEN、暂估流水已配对 |
| ⑤逆操作 | 蓝冲反过账（未结账）/ 红冲（已结账，跨年经 **6901 以前年度损益调整**） |
| ⑥效果 | ➕`finance.create_cost_adjustment_voucher`→`finance.post_voucher` |
| ⑦回链 | 返利：→SupplierCredit/采购返利单；差异：→原 GR/暂估流水；voucher_id |
| ⑧准则 | CAS：移动加权下差异据实调整在库与已售部分；返利冲减进货成本（不作收入） |
| 拓展 | **★平销返利金蝶坑**：返利"成本事后才定"——入库时按发票价/估价入存货，返利确认后用本调整凭证**按未售比例冲减库存成本、按已售比例冲减已结转的销售成本**（见④映射末行 + ⑪gap） |

## ③ 关键状态机表（可直接 seed WorkflowDefinition.states）

### 3.1 INVENTORY_COSTING（✅已 seed，➕需补过账 effect + 做薄）

> 现状 seed 是"K3 核算概念全流程"18 边但**无任何 effect**（纯 prompt）。贸易公司只用"外购入库核算 / 产成品出库核算 / 生成凭证 / 转总账"四态，其余材料出库/自制/委外**保留占位不挂 effect**。下表是**做薄并挂 effect 后**的精确状态机（角色按基石替换 FINANCE→ACCOUNTANT 制单 / FINANCE_MANAGER 审过账）。

| state 码 | allowed_roles | editable_fields | hard_rules（精确） | 触发 effect |
|---|---|---|---|---|
| BUSINESS_TRIGGER (initial) | 系统 | — | 由 GR/SHIPMENT 状态机跨流触发 | — |
| PURCHASE_IN_COST 外购入库核算 | 系统/ACCOUNTANT | — | 已有 InventoryTransaction(IN) | （数量价值已由 wms 写） |
| FINISHED_OUT_COST 产成品出库核算 | 系统/ACCOUNTANT | — | 已有 InventoryTransaction(OUT)；OUT 行 unit_cost>0（否则报"无成本不可结转"） | （wms 已写）→进 VOUCHER_GENERATED |
| VOUCHER_GENERATED 生成凭证 | ACCOUNTANT | period_id, voucher_date | 该期所有未生成凭证的流水齐备；Σ取数借=贷 | ➕`finance.create_voucher_from_inv_costing` |
| VOUCHER_AUDITED 凭证审核（➕新增） | FINANCE_MANAGER | — | 制单≠审核（segregation_of_duties ON）；借贷平衡 | — |
| TRANSFERRED_GL 转总账(terminal) | FINANCE_MANAGER | — | 期 OPEN | ➕`finance.post_voucher`（写 AccountBalance，回填 voucher_id） |
| PERIOD_CLOSED 期末结账(terminal) | FINANCE_MANAGER, BOSS | — | 期已 CLOSED | （由 09 结账模块收口） |

> ❌**不为本模块单独建状态机的对象**：暂估/差异/红冲的凭证一律走 **VOUCHER 主状态机**（基石18态）；本模块只产出 Voucher 草稿喂进去。

### 3.2 暂估/差异凭证 → 复用 VOUCHER 状态机（基石）

| 场景 | doc_type | 入口态 | 关键 hard_rule | effect |
|---|---|---|---|---|
| 入库暂估入账 | VOUCHER (type=PROV) | DRAFT→AUDITED→POSTED | 借贷平衡；期 OPEN | `finance.create_provisional_payable_voucher`→`post_voucher` |
| 发票冲回暂估 | VOUCHER (type=PROV_REV) | DRAFT→…→POSTED | 暂估 voucher 须 POSTED 且未冲回 | `finance.reverse_provisional_on_invoice` |
| 期末成本差异冲回 | VOUCHER (type=COST_ADJ) | DRAFT→…→POSTED | 差异=实际−暂估，按在库/已售拆 | `finance.create_cost_adjustment_voucher` |
| 平销返利成本下调 | VOUCHER (type=COST_ADJ) | DRAFT→…→POSTED | 返利单已确认；返利≤已暂估成本 | `finance.create_cost_adjustment_voucher` |
| 业务红冲（退货/采退） | VOUCHER (type=RED) | DRAFT(草稿) | 顺 source_doc 回链找原凭证；**广播全财务** | `finance.create_red_reversal_from_business` |

## ④ ★科目映射取数公式（业务单×动作 → 借/贷分录；科目码=基石契约）

> 统一经 `AccountMappingRule`（基石❌新建）+ `map_to_voucher_entries()`。方向永远从 `account.balance_direction` 推，**禁 effect 写死借贷**。金额 Numeric(16,2)，单位成本(16,4)。**每条标【财务确认】**。

### 4.1 外购入库（GOODS_RECEIPT → STOCKED_IN）— 暂估入账

| line_seq | dr_cr_role | account_code | account_source | amount_formula | tax_handling | provisional_flag |
|---|---|---|---|---|---|---|
| 1 | DR | **1405** 库存商品 | FIXED | Σ(actual_qty × 估价单价) | EXCLUSIVE（不含税） | true |
| 2 | CR | **2202xx** 应付账款-暂估应付 | SUPPLIER | =行1金额（暂估不拆进项税） | NONE | true |

口诀：1405 借增、2202 贷增。**暂估不计进项税**（无合规发票，进项税待发票到票才确认）。【财务确认：估价单价优先级见⑪gap】

### 4.2 进项发票审核（PURCHASE_INVOICE → 审核）— 冲回暂估 + 实际入账

| 步 | line_seq | dr_cr_role | account_code | amount_formula | 说明 |
|---|---|---|---|---|---|
| 冲回 | 1 | DR | 2202xx 暂估应付 | =原暂估金额（NEGATE 等效） | 冲掉暂估贷方 |
| 冲回 | 2 | CR | 1405 库存商品（或6401/差异） | =原暂估金额 | 冲掉暂估借方 |
| 实际 | 3 | DR | 1405 库存商品 | 发票不含税额 | 按实际进货成本入库 |
| 实际 | 4 | DR | **222101** 进项税额 | 发票额×税率 | 税率来源 LINE→MATERIAL→COMPANY |
| 实际 | 5 | CR | **2202** 应付账款 | 发票价税合计 | 正式应付 |

**差异处理**：(实际不含税 − 暂估) = 差异。货仍在库→并入步3的 1405；货已售出部分→差异计 **6401 主营业务成本**（按已售比例拆分，见 4.5）。tax_handling=EXCLUSIVE+TAX_ONLY 组合。【财务确认】

### 4.3 销售出库结转成本（SHIPMENT → SALES_OUTBOUND）— 暂估成本结转（发货当下看利润）

| line_seq | dr_cr_role | account_code | amount_formula | 说明 |
|---|---|---|---|---|
| 1 | DR | **6401** 主营业务成本 | Σ(出库qty × InventoryValuation.current_unit_cost) | 移动加权当前单位成本 |
| 2 | CR | **1405** 库存商品 | =行1金额 | 减库存 |

口诀：6401 借增（COGS DEBIT）、1405 贷减。**出库即结转=发货当下毛利可见**（收入侧由 05 收入模块按 CAS14 控制权转移确认，COGS 与收入配比）。【财务确认：成本结转时点=控制权转移时点，与收入同步，见⑪gap】

### 4.4 库存调整 / 盘盈盘亏（STOCK_ADJUSTMENT → 完成）

| 场景 | DR | CR | amount_formula |
|---|---|---|---|
| 盘亏/报废 | **6701** 资产减值损失（或管理费用，见gap） | **1405** 库存商品 | 短缺qty × current_unit_cost |
| 盘盈 | **1405** 库存商品 | **6711/营业外收入** 或冲管理费用 | 盈余qty × current_unit_cost |
| 跌价计提 | **6701** 资产减值损失 | **1471** 存货跌价准备(备抵CREDIT) | 可变现净值缺口 |

【财务确认：盘亏去向 6701 vs 管理费用 vs 待处理财产损溢 1901】

### 4.5 期末成本差异冲回 / 平销返利（InventoryCostAdjustment → POSTED）

**实际差异冲回（暂估≠实际）**：

| 情形 | DR | CR | amount |
|---|---|---|---|
| 实际>暂估·在库 | 1405 库存商品 | 2202/银行 | 差额×在库比例 |
| 实际>暂估·已售 | 6401 主营业务成本 | 2202/银行 | 差额×已售比例 |
| 实际<暂估·在库 | 2202/银行 | 1405 库存商品 | |
| 实际<暂估·已售 | 2202/银行 | 6401 主营业务成本 | |

**★平销返利（成本事后才定 / 金蝶坑）**——返利冲减进货成本，按"未售/已售"拆：

| 情形 | DR | CR | amount_formula |
|---|---|---|---|
| 返利·货未售 | 2241 其他应付款（或冲应付/银行收返利） | **1405** 库存商品 | 返利额 × (未售qty/总qty) |
| 返利·货已售 | 2241/银行 | **6401** 主营业务成本 | 返利额 × (已售qty/总qty) |
| 返利含税（一般纳税人开红字） | 上同 | **222101** 进项税额转出 | 返利×税率 |

口诀：返利**冲减成本不作收入**（CAS：平销返利按购进金额比例冲减成本，不确认其他业务收入）。已售比例×返利→冲已结转 6401；未售比例→冲在库 1405，并即时重算 `InventoryValuation.total_value`。【财务确认：返利分摊口径=按购进数量比例 vs 销售额比例；红字进项税转出处理，见⑪gap】

### 4.6 红冲（业务退货/采购退货）

红冲遵基石第13条：顺 `source_doc_type/id` 回链找原凭证→生成 `voucher_type=RED` 负数草稿（NEGATE：同科目同方向取负）→财务审核过账→**广播站内通知全部财务**（`Notification.recipient_role` in CASHIER/ACCOUNTANT/FINANCE_MANAGER/FINANCE）。已结账走红字+蓝字配对（发生额净额=0、余额不变）；跨年经 **6901**。

## ⑥ 校验规则 + 边界 + 错误处理（失败精确提示，绝不静默）

| 规则 | 触发点 | hard_rule | 失败提示（精确） |
|---|---|---|---|
| 借贷平衡 | 过账前 | Σ借=Σ贷 | "存货凭证不平：借 {dr} 贷 {cr} 差 {diff}，禁止过账" |
| 期间锁 | 生成/过账 | period.status=OPEN | "{period} 已结账，存货成本调整请用调整凭证或反结账" |
| 出库无成本 | SHIPMENT 结转 | OUT 行 unit_cost>0 | "物料 {sku} 当前移动加权成本=0，无法结转 COGS，请先确认入库成本/暂估" |
| 负库存 | _apply_valuation_delta | new_qty≥0 | clamp 0 但 **WorkflowLog 记警告**"物料 {sku} 出库致负库存，已置0，请核查"（不静默） |
| 暂估优先级缺失 | 暂估入账 | 估价单价>0 | "物料 {sku} 无 PO 价/合同价/历史成本，需手工估价方可暂估入账" |
| 暂估重复冲回 | 发票审核 | 暂估 voucher 未冲回 | "该入库暂估已被发票 {no} 冲回，不可重复" |
| 幂等 | 所有 effect | `_transaction_exists(ref_type,ref_id)` | 已存在→跳过（现状已实现，复用），返回"已处理 skip" |
| 返利超额 | 返利冲减 | 返利≤已暂估/已入成本 | "返利 {amt} 超过该批入库成本 {cost}，请核对返利单" |
| 红冲前置 | 红冲 | 原凭证 POSTED 且未被红冲 | "未找到原过账凭证或已红冲，无法生成红字" |
| SoD | 审核/过账 | created≠auditor≠posted | "制单人不可同时审核/过账（职责分离已开启）" |
| 跨期差异 | 月末差异 | 差异落已结账期 | "差异属已结账期，须经调整凭证(6901)入当期" |

边界场景：① qty=0 不生成流水；② 同一物料同期多次入出→移动加权逐笔即时重算（非全月一次）；③ 跨年返利→6901；④ 货已全部售出后才来发票差异→全部进 6401；⑤ 暂估入库后又退货→先红冲暂估再处理退货。

## ⑩ 验收用例（Given-When-Then，含边界）

1. **入库暂估**：Given PO 行单价 10、入库 100 件无发票；When GR→STOCKED_IN；Then InventoryValuation total_qty+100/total_value+1000/unit=10，生成暂估凭证 DR 1405 1000 / CR 2202暂估 1000 并过账，AccountBalance 1405 +1000、2202 +1000，InventoryTransaction.voucher_id 回填。
2. **发货当下看利润**：Given 库存移动加权单价 10、销售单价 15、发 60 件；When SHIPMENT→SALES_OUTBOUND；Then OUT 流水成本 600，COGS 凭证 DR 6401 600/CR 1405 600 过账，SalesInvoiceLine.cost_amount=600，**毛利 (15-10)×60=300 当下可见**。
3. **移动加权重算**：Given 库存 100@10；先入 100@12 → unit=(1000+1200)/200=11；再出 50 → COGS=50×11=550，结存 150@11=1650。校验逐笔即时（非全月平均）。
4. **暂估冲回+差异已售**：Given 暂估入 100@10、已售 60；When 进项发票到票实际 11；Then 冲回暂估 1000，实际入 DR 1405 1100+DR 222101 进项税/CR 2202；差异 100：在库40件部分 40 进 1405、已售60件部分 60 进 6401。
5. **★平销返利（金蝶坑）**：Given 入库 100@10、已售 70、收返利 200（含税）；When 返利单确认→成本调整凭证；Then 未售30%：DR 2241 60/CR 1405 60；已售70%：DR 2241 140/CR 6401 140；InventoryValuation.total_value −60 即时重算；进项税转出按返利税额。
6. **盘亏**：Given 账面 100 实盘 95；When STOCK_ADJUSTMENT 完成；Then ADJUST 流水 −5×unit，凭证 DR 6701/CR 1405，台账减值。
7. **跨期差异（边界）**：Given 6月已结账，7月才到发票有差异；Then 提示不可入6月，差异经 6901 入7月调整凭证。
8. **未结账蓝冲**：Given 暂估凭证已过账、期 OPEN、发现估价错；When 反过账→改→重过账；Then 不留负数，AccountBalance 复原后重写。
9. **已结账红冲**：Given 暂估凭证已结账后发现错；When 红冲；Then 红字负数+蓝字配对，发生额净额=0、余额不变，跨年经6901，**广播全财务**。
10. **业务红冲广播**：Given 客户退货顺 source_doc 回链找到原 COGS 凭证；When 生成红字草稿；Then 全部财务收到站内通知，财务审核过账后冲回 6401/1405。
11. **负库存边界**：Given 库存 10 误发 15；Then valuation clamp 0、WorkflowLog 记警告（不静默），出库前置校验本应拦截 SHIPMENT。
12. **字段防火墙**：Given SALES 角色查发货单/收发存账；Then 所有成本/单价字段 API 返回缺省、UI 隐藏。
13. **公司隔离**：Given 公司A查询；Then 仅 company_id=A 的台账/流水/凭证可见，绝不串账。
14. **借贷不平拦截**：Given 映射取数 bug 致差 0.01；Then 过账被拒，提示精确差额。
