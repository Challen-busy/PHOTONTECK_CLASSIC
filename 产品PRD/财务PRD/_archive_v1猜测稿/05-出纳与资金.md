# 05 · 出纳与资金（Cashier & Treasury）

> Layer1 交易模块。范围：现金日记账 / 银行日记账 / 银行对账单（勾对未达账项）/ 资金调拨（账户间转账）/ 票据管理（应收应付票据生命周期）/ 出纳 CASHIER 角色（管钱与会计管账分离）。
>
> **一句话价值**：让出纳在系统里完成全部货币资金的收付、登账、对账、调拨、票据保管，日记账逐笔与凭证过账后的 1001/1002 科目余额勾稽对平，金蝶出纳模块下线。
>
> **本模块在三层中的位置**：Layer1 交易层。只依赖 Layer0 基石 digest（科目码 1001/1002/1121/2201、`AccountMappingRule`、`finance.post_voucher`、CASHIER 角色、10 条不变量）。任何科目余额变动必经 Voucher→过账，本模块**不直接写 AccountBalance**（不变量 5：财务唯一写入路径=凭证过账）。
>
> **引擎现状（已 grep 核实 2026-06-17）**：
> - ✅已有：`BankReceipt`（银行收款流水 `bank_receipt`，status=UNALLOCATED）、`NotesReceivable`（应收票据 `notes_receivable`，status=HELD）、`ARSettlement`（核销明细，可挂 bank_receipt_id/note_id）、`AdvanceReceipt`/`AdvancePayment`/`PaymentRequest`（预收/预付/付款申请，DRAFT→…→CONFIRMED/PAID）、`Account`/`Voucher`/`VoucherEntry`/`AccountBalance`/`ExchangeRate`、`Notification`（recipient_role 角色广播）。
> - ➕扩展：`BankReceipt`（补 voucher_id/journal 钩子，作为银行收款流水来源之一，但本模块以新建日记账为主账本）；AR 状态机 CASH_MGMT/NOTES_RECV 节点（`phase1_workflows.py:589-592`）已有「现金到账/票据到账」边，本模块的日记账/票据登记接其后。
> - ❌需新建（本模块主要建造量）：`CashJournal` 现金日记账、`BankJournal` 银行日记账、`BankAccount` 银行账户主数据、`BankStatementLine` 银行对账单行 + `BankReconciliation` 对账批次（含 4 类未达账项 + 余额调节表）、`FundTransfer` 资金调拨单、`NotesPayable` 应付票据 + 票据状态机扩展。全部纯扩展（`@register_command`/`@register_transition_effect`/`@register_transition_validator` + `__doc_types__` + `WorkflowDefinition.states`），core/registry.py / workflow.py / commands.py 字节级零 diff。
>
> 标注口径：✅引擎已有 / ➕扩展现有 / ❌需新建。

## ① 模块定位

出纳是「钱」的看门人，会计是「账」的记录者，二者强制分离（决策 10）。本模块覆盖钱进钱出的执行与登记：每一笔现金/银行收付由出纳 CASHIER 在日记账逐笔登账（来源=收付款单/核销/调拨/票据），会计 ACCOUNTANT 据收付款单制凭证、FINANCE_MANAGER 过账更新 1001/1002 科目余额；月末出纳导入/录入银行对账单，与银行日记账逐笔勾对，找出 4 类未达账项，出余额调节表证明「账实相符」。资金调拨管账户间转钱（同公司本外币户互转、提现、存现），票据管理管应收/应付票据从收到/开出到贴现/背书/到期托收/退票的全生命周期。

**核心勾稽（本模块的验收红线）**：
```
现金日记账期末余额 == 1001 库存现金 科目余额（过账后）
银行日记账期末余额（按 bank_account 分户汇总） == 1002 银行存款 科目余额（过账后）
余额调节表：调节后双方余额相等
```

## ⑥ 校验规则 + 边界 + 错误处理（失败精确提示，绝不静默）

**全局不变量落地（逐条）**：
1. 借贷平衡：所有日记账生成的凭证过账前 Σ借=Σ贷，不平禁过账，提示「凭证不平衡，借 X 贷 Y 差额 Z」。
2. 过账不可改：日记账 POSTED_JOURNAL 且已挂 voucher_id 后，金额/账户/方向只读；改须先蓝冲/红冲凭证。
3. 期间锁：journal_date 所属 AccountingPeriod.status=CLOSED 时禁登账/禁过账，提示「该期间已结账，请用次期或调整凭证」。
4. 业财同源：日记账登账 effect 与凭证草稿同事务原子生成（要么都成要么都回滚）。
5. 财务唯一写入路径：日记账 running balance 是派生（查询时 Σ 算），1001/1002 科目余额只由 `finance.post_voucher` 写 AccountBalance。日记账本身**不写 AccountBalance**。
6. 审计四本：每次登账/勾对/调拨/票据动作经 execute_transition→WorkflowLog + CommandLog。
7. 字段防火墙：本模块无成本/买价字段，不涉及 SALES 隐藏。
8. 公司隔离：bank_account/journal/transfer 的 company_id NOT NULL，查询走 `_company_filter(user)`；资金调拨 from/to 账户必须同 company_id（跨公司报错「资金调拨不可跨公司，跨公司资金往来请走往来挂账」）。
9. 冲销不留脏：日记账未过账作废=VOID 不留负数；已过账错账=随凭证红冲（红字日记账+蓝字重做）。
10. 业务红冲广播：票据退票（DISHONORED）+ 错误调拨红冲 → 自动生成红字凭证草稿 + Notification 广播 recipient_role=FINANCE/CASHIER/ACCOUNTANT/FINANCE_MANAGER（决策 13）。

**对象级校验**：
| 校验点 | 规则 | 错误提示 |
|--------|------|---------|
| 日记账金额 | amount>0 | 「金额必须大于 0」 |
| 账户类型匹配 | CASH_JOURNAL→is_cash=True；BANK_JOURNAL→is_cash=False | 「现金日记账须选现金账户」 |
| 余额勾稽 | 日记账期末余额≠对应科目余额（过账后） | 对账时 WARN「日记账与科目余额差 X，请核查未过账日记账」 |
| 调拨账户 | from≠to 且同公司 | 「转入转出账户不能相同」/「资金调拨不可跨公司」 |
| 调拨余额 | from 账户余额≥amount+fee（可配硬拦） | 「转出账户余额不足，当前 X 需 Y」 |
| 对账平衡 | adjusted_balance 双方相等（容差 0.01） | 「对账未平，调节后差额 X，请检查未达账项/漏勾对」 |
| 勾对一致 | statement_line.amount==journal.amount 且 direction 一致 | 「勾对金额/方向不一致，无法勾对」 |
| 票据到期 | 托收 journal_date≥maturity_date；贴现日<maturity_date | 「未到期不可托收」/「已过期不可贴现」 |
| 期间锁 | 所属期间 OPEN | 「期间已结账，禁止操作」 |

**边界**：①外币户日记账原币+折算双记，期末调汇在 Layer2，本模块只保证原币余额准确。②跨期对账（对账单含跨期流水）→按 statement_date 归期，未达账项跨期次月补登。③一笔银行收款核销多张发票（ARSettlement 一对多）→日记账一笔、核销多行，凭证借 1002 一行贷 1122 按客户多行。④调拨手续费=0 时不生成费用行。

## ⑧ UI（对齐金蝶/用友出纳模块惯例）

### 屏 8.1 · 出纳收付台账（现金/银行日记账，专用屏，密集网格键盘流）

对齐金蝶「出纳→现金日记账/银行日记账」：顶部 Tab 切「现金日记账｜银行日记账」+ 账户下拉 + 期间筛选。主体=密集网格，列：日期 | 摘要 | 方向(收/付) | 对方单位 | 收入 | 支出 | 结存余额(running，系统算) | 来源单 | 凭证号 | 状态。
- **键盘流**：Tab 横走、Enter 下行新增、摘要可向下复制、收/付金额输入即重算结存余额。
- **顶栏汇总**：本期收入合计 / 支出合计 / 期末结存 + 「与科目余额勾稽」徽标（平=绿、差=红并显差额，下钻看未过账日记账）。
- 行内「来源单」可点开回链原单（BankReceipt/PaymentRequest/FundTransfer）；「凭证号」下钻凭证。
- 自动带出：选来源单后自动填日期/金额/对方/摘要，出纳只补确认。

**关键交互清单**：账户+期间筛选 / 收付双栏即时结存 / 来源单一键带出 / 摘要向下复制 / 与科目余额实时勾稽徽标 / 下钻来源单与凭证 / 作废留痕（划线不删）。

### 屏 8.2 · 银行对账（专用屏，左右双栏勾对）

对齐金蝶「出纳→银行对账」：左栏=银行对账单流水（导入/手录），右栏=本企业银行日记账，中间「自动勾对」按钮（按金额+日期+方向匹配）。
- 已勾对行变灰成对显示；未勾对行高亮，出纳逐笔点选配对或标记未达账项类型（4 类下拉）。
- 底部=**银行存款余额调节表**（自动生成）：左=对账单余额±未达项、右=日记账余额±未达项，调节后两端余额并排，相等=绿色「已平」、不等=红色「未平 差 X」。
- 完成对账须双方相等才可提交 BALANCED；FINANCE_MANAGER 复核。

**关键交互清单**：导入对账单（Excel/手录）/ 自动勾对+手工微调 / 未达账项 4 类标记 / 余额调节表实时计算 / 平/不平红绿徽标 / 不平禁提交。

### 屏 8.3 · 资金调拨 / 票据台账

- **资金调拨**：低频，复用通用壳 DocEditor 单头表单（转出户/转入户/类型/金额/汇率/手续费/摘要）+ 审批流。
- **票据台账**：列表 + 到期提醒（临近 maturity_date 红黄标）+ DocEditor 录票/贴现/背书/托收动作按钮。

### 屏 8.4 · 银行账户主数据

复用通用壳 DataExplorer（左列表右明细），无专用屏。

## ⑩ 验收用例（Given-When-Then，含边界）

1. **现金收款登账→勾稽**
   Given 客户现金回款 ¥5000 核销 AR#88；When CASHIER 在现金日记账登 IN ¥5000（来源 AR_SETTLEMENT），会计制凭证 借1001/贷1122 ¥5000 并过账；Then 现金日记账期末结存 +5000 == 1001 科目余额变动，勾稽徽标绿。

2. **银行付款外币折算**
   Given 付供应商货款 USD 10000，当日汇率 7.10，本位币 CNY；When CASHIER 登银行日记账 OUT amount=10000 currency=USD exchange_rate=7.10，过账；Then 凭证 借2202/贷1002 折本位 71000 CNY，1002 银行存款（USD 户）原币减 10000。

3. **资金调拨同公司不影响总资产**
   Given 招行 CNY 户转工行 CNY 户 ¥100000；When FUND_TRANSFER BANK_TO_BANK EXECUTED 过账；Then 借 1002(工行)/贷 1002(招行) 各 10 万，总货币资金不变，两户日记账各一笔。

4. **跨币种购汇折算差额（边界）**
   Given USD 户购 CNY，amount=USD 1000、rate=7.10、to_amount=CNY 7100，手续费 CNY 20；When EXECUTED；Then 借 1002(CNY) 7100 + 6603 手续费 20 / 贷 1002(USD) 折本位 7100+... 差额轧入 660303 汇兑损益使借贷平。

5. **银行对账 4 类未达账项（核心边界）**
   Given 对账单余额 ¥120000、日记账余额 ¥118000；存在「企业已付银行未付」¥3000（已开支票银行未扣）+「银行已收企业未收」¥1000（银行代收企业未登）；When CASHIER 勾对后算调节表：日记账 118000 + 银行已收企未收 1000 = 119000；对账单 120000 - 企业已付银未付 3000 = 117000... Then 系统检测不平差 2000，红色提示「对账未平，请检查漏勾对/未达项」，禁提交（验证不平拦截）。

6. **银行对账平衡通过**
   Given 调整两类未达项使两端调节后均 =117000；When CASHIER 完成对账；Then status=BALANCED，绿色「已平」，FINANCE_MANAGER 可复核。

7. **票据贴现（边界）**
   Given 持有银行承兑 ¥100000 maturity 90 天后，贴现率 3.6%/年；When CASHIER 申请贴现、FINANCE_MANAGER 复核、过账；Then 借 1002 实得 ¥99100 + 6603 贴现息 ¥900 / 贷 1121 ¥100000（贴现息公式待财务确认天数基准）。

8. **票据退票广播（决策 13）**
   Given 商业承兑到期托收被拒（DISHONORED）；When CASHIER+ACCOUNTANT 处理；Then 自动生成红字/转回凭证 借 1122/贷 1121 ¥100000 草稿 + Notification 广播 FINANCE/CASHIER/ACCOUNTANT/FINANCE_MANAGER「票据 NR-xxx 退票」。

9. **期间锁（边界）**
   Given 6 月期间已 CLOSED；When CASHIER 试图登 6 月日记账；Then 报错「该期间已结账，请用次期或调整凭证」，禁登账。

10. **过账后改账拦截（不变量 2）**
    Given 日记账已过账（voucher POSTED）；When CASHIER 改金额；Then 字段只读、提示「已过账不可改，请蓝冲反过账或红冲」。

11. **跨公司调拨拦截（不变量 8）**
    Given from=公司 A 账户、to=公司 B 账户；When 提交调拨；Then 报错「资金调拨不可跨公司」。

12. **职责分离（决策 10，SoD ON）**
    Given segregation_of_duties=ON；When 同一人 CASHIER 登账 + 又作为会计制凭证 + 过账；Then validator `finance.segregation_of_duties` 拦截「制单/审核/过账须不同人」（小公司可配 OFF 放宽）。
