# 01 · Layer0 基石模块

> Layer0 = 整套财务ERP的地基，**串行先封板**。封板内容：会计科目表 / 会计年度期间 / 币种本位币汇率 / ★凭证内核 / ★科目映射与凭证模板框架 / 财务角色与职责分离。封板后 digest 下发，Layer1 交易模块才能并行开工。
>
> 引擎现状已 grep 核实（2026-06-17）：`Account` / `FiscalYear` / `AccountingPeriod` / `Voucher` / `VoucherEntry` / `AccountBalance` / `ExchangeRate` 模型 **✅已存在**；VOUCHER 18 态状态机 **✅已 seed**（`scripts/seed.py:456`）；但 **过账 effect（写 AccountBalance）❌不存在**、**科目映射框架 ❌不存在**、**财务角色 CASHIER/ACCOUNTANT/FINANCE_MANAGER ❌未加**、**蓝冲/红冲逆向边与 command ❌未建**。这些是基石的主要建造量。
>
> 标注：✅引擎已有 / ➕扩展现有 / ❌需新建。

|-----------|------|--------------|-------------------|--------|--------|
| **1** | 资产 | ASSET | **DEBIT** 借 | 借 | 贷 |
| **2** | 负债 | LIABILITY | **CREDIT** 贷 | 贷 | 借 |
| **3** | 共同 | (按性质) | 按性质 | — | — |
| **4** | 权益（所有者权益） | EQUITY | **CREDIT** 贷 | 贷 | 借 |

### 1.6 准则切换规则
- 公司创建时选模板（`Company.region`：CN→CAS、HK→HKFRS）。
- 切换 = 加载对应模板科目集 + 保留旧码映射表（报表/期初导入据此对应）。
- 各公司在模板下**增设明细**（建子级，`parent_id` 挂上级，`level+1`，`is_leaf` 自动维护）。

### 1.7 UI（左树右明细）
- 左：科目代码段树（1/2/4/5/6 根 → 下钻）；右：选中科目属性（code/name/类型/方向/币种/是否叶/启停）。
- 操作：新增下级、停用、启用、切换模板（顶部）。停用时校验「无余额、无未结引用」。

## 模块 2 · 会计年度 / 期间

### 2.1 对象范式表

| 列 | 内容 |
|----|------|
| ① 对象 | `FiscalYear` + `AccountingPeriod`（model）；A 主数据 |
| ② 类型 | **A 主数据**（期间状态 OPEN/CLOSED 影响过账，是全局期间锁的源） |
| ③ 数据 | 见 §2.2 |
| ④ 动作×角色 | 建年度/建期间：`FINANCE_MANAGER`；开启/关闭期间：`FINANCE_MANAGER`+`BOSS`（结账在 VOUCHER 主流程，见模块 4） |
| ⑤ 逆操作 | **反结账**（CLOSED→OPEN）：`FINANCE_MANAGER`+`BOSS`，解锁期间允许补/改凭证；审计留痕 |
| ⑥ 效果 | 期间 CLOSED → 该期任何 Voucher 过账被拦（期间锁 hard_rule）；本期 closing 余额 → 下期 opening |
| ⑦ 回链 | `Voucher.period_id` FK；`AccountBalance.period_id` FK；`AccountingPeriod.fiscal_year_id` FK |
| ⑧ 准则/账簿 | `company_id` 账套（FiscalYear.company_id）；各公司独立期间 |
| +拓 | **默认截止 27 号、会计年度 12 月 27 号分割**（对齐业务盘点）；启用日对齐期间起点；可配 |

### 2.2 字段字典（✅模型已有）

**FiscalYear**：id / company_id(FK,✓) / year(Int,✓) / start_date(Date,✓) / end_date(Date,✓) / status(String10, 默认OPEN, 枚举 OPEN/CLOSED)。UNIQUE(company_id, year)。
**AccountingPeriod**：id / fiscal_year_id(FK,✓) / period_number(SmallInt,✓, 1-12，调整期=13) / start_date(✓) / end_date(✓) / status(默认OPEN, OPEN/CLOSED) / closed_by_id(FK→user, NULL) / closed_at(DateTime, NULL)。UNIQUE(fiscal_year_id, period_number)。

### 2.3 期间切割规则（可配）
- 默认：每月 28 号 ~ 次月 27 号为一个会计期间；12 月 27 号为年度分割点。
- 启用日（期初导入日）对齐**期间起点**（启用日所在期 = 第 1 期）。
- 配置项（公司级）：截止日（默认 27）、年度分割月（默认 12）。
- 调整期（第 13 期）：年末审计调整专用，走 `VOUCHER_ADJUSTMENT` 状态机（✅已 seed）。

### 2.4 期间锁（全局不变量 #3）
- hard_rule `period_open`：过账前校验 `AccountingPeriod.status == OPEN`，CLOSED 拒过账，提示「期间 {year}-{period} 已结账，请用调整期凭证或先反结账」。
- 跨期改账：禁止直接改已结账期 → 走调整期（VOUCHER_ADJUSTMENT）或反结账后蓝冲。

## 模块 4 · ★凭证内核（核心·B 类·完整对象范式）

> 这是 Layer0 的心脏。`Voucher`/`VoucherEntry` 模型 ✅已有，VOUCHER 18 态状态机 ✅已 seed，但 **过账 effect（写 AccountBalance）❌不存在**、**蓝冲/红冲逆向边与 command ❌未建**。基石的主要建造量在此。

### 4.1 对象范式表

| 列 | 内容 |
|----|------|
| ① 对象 | `Voucher`(头)/`VoucherEntry`(行) ；doc_type=**VOUCHER**（主流程）/ **VOUCHER_ADJUSTMENT**（调整期）；B 交易凭证 |

| ⑤ 逆操作 | **蓝冲**（未结账）：反过账 POSTED→AUDITED、反审核 AUDITED→DRAFT → 直接改/删 → 重过账；**红冲**（已结账）：生成红字负数 Voucher 冲销 + 蓝字重做。财务自行判断选（见 §4.5） |

| PL_TRANSFERRED | 结转损益 | state(E) | ACCOUNTANT | 损益结转 |
| CLOSED | 期末结账 | state(terminal,E) | FINANCE_MANAGER+BOSS | 期间关闭 |
| LEDGER_QUERIED / REPORT_QUERIED | 账簿/财报查询 | cross_module(terminal) | FINANCE系+BOSS | 只读派生 |
| RPT_* | 各报表节点 | report | — | 派生 |

**边（transition）：**

| label | from→to | roles | hard_rules | editable_fields | effects |
|-------|---------|-------|-----------|-----------------|---------|
| 启用科目 | ACCOUNT_READY→DRAFT | ACCOUNTANT | — | — | — |
| 提交审核 | DRAFT→AUDITED | ACCOUNTANT | **借贷平衡**(Σ借=Σ贷)、金额>0、账户为叶、必输(摘要/科目/金额)、**期间OPEN** | — | — |
| 审核退回 | AUDITED→DRAFT | FINANCE_MANAGER | 须填退回原因 | — | — |
| 审核过账 | AUDITED→POSTED | FINANCE_MANAGER | 借贷平衡、期间OPEN、**职责分离**(过账人≠制单人，开关可配) | — | ➕`finance.post_voucher`（写 AccountBalance） |
| ➕**反过账**(蓝冲) | POSTED→AUDITED | FINANCE_MANAGER | **期间OPEN**(已结账禁反过账，须红冲)、须填原因 | — | ➕`finance.unpost_voucher`（冲回 AccountBalance） |
| ➕**反审核**(蓝冲) | AUDITED→DRAFT | FINANCE_MANAGER | 须填原因 | — | — |
| 往来核对 | POSTED→RECONCILED | ACCOUNTANT | — | — | — |
| 过账直接调汇 | POSTED→FX_ADJUSTED | ACCOUNTANT | — | — | ➕`finance.fx_revaluation` |
| 结转损益 | FX_ADJUSTED→PL_TRANSFERRED | ACCOUNTANT | — | — | ➕`finance.pl_transfer` |
| 期末结账 | PL_TRANSFERRED→CLOSED | FINANCE_MANAGER+BOSS | 前置清单(借贷平衡✓/无未过账✓/调汇✓/损益已结✓) | — | ➕`finance.close_period`（period→CLOSED, closing→下期opening） |

> 注：seed 现有边的 roles 写的是 `["FINANCE"]`/`["FINANCE","BOSS"]`。封板时**统一替换为新角色集**（制单 ACCOUNTANT、审核/过账 FINANCE_MANAGER），并保留 FINANCE 作为兼容超集（小公司一人多职责）。`editable_fields` 在 DRAFT 态开放头+行全字段；AUDITED/POSTED 态 `editable_fields=[]`（锁死）。

### 4.4 ★科目映射取数公式（凭证行级·示范，模块 5 是完整框架）
凭证内核本身不写死科目，分录来源有二：
1. **手工凭证**：财务直接选科目填借贷（密集网格）。
2. **业务自动凭证**：由科目映射框架（模块 5）按「业务单×动作」翻译成借贷行，过账时跑 `finance.post_voucher`。

### 4.5 逆操作精确规则（决策 12·已 WebSearch 核实术语）

| 场景 | 方法 | 操作链 | 留痕 | 适用 |
|------|------|--------|------|------|
| **未结账**（期间 OPEN，凭证可改） | **蓝冲** | 反过账(POSTED→AUDITED, effect 冲回余额) → 反审核(AUDITED→DRAFT) → **直接改/删** → 重新提交审核→过账 | 蓝字正常凭证，**不留负数** | 当期错账 |
| **已结账**（期间 CLOSED，凭证不可改） | **红冲** | 生成**红字负数凭证**（同科目同方向、金额取负，冲销原凭证）+ 再做**蓝字正确凭证** | 红字+蓝字配对，**发生额净额=0，余额不受冲销对影响** | 已结期/跨期错账 |
| **跨会计年度**（影响以前年度损益） | 红冲 + 经 **6901 以前年度损益调整** | 不能直接调本年利润，损益错账经 6901 调整，最终转利润分配 | 经 6901 科目 | 上年损益错账 |

**红冲 command**（❌需新建 `@register_command("finance.red_reversal")`）：入参=原 voucher_id → 生成红字 Voucher 草稿（每行 debit/credit 取负、source_doc_type='VOUCHER'+source_doc_id=原 id、is_auto_generated=True、description="红冲 {原凭证号}"）→ 财务审核过账。

**财务自行判断选蓝冲/红冲**：UI 在凭证详情提供「冲销」按钮，按期间状态自动建议（OPEN→蓝冲、CLOSED→红冲），财务可强制选红冲。

### 4.6 业务红冲（决策 13·必广播）
- 触发：客户退货/退款/采购退货（SALES_RETURN/RMA/采购退货）单据过状态。
- 动作：顺 `source_doc` 回链找原凭证 → 自动生成**红字冲销凭证草稿** → 财务审核过账。
- ★**必须广播全部财务**：`Notification.recipient_role` ∈ {ACCOUNTANT, FINANCE_MANAGER, FINANCE_DIRECTOR}，站内通知「{业务单}已生成红冲凭证草稿待审核」。
- effect：`@register_transition_effect("finance.create_red_reversal_from_business")`（❌需新建，挂在退货/退款单据的相应 to_state）。

### 4.7 过账 effect 契约（❌需新建·所有交易模块共享）

### 4.8 校验规则 + 错误处理（绝不静默）

| 规则 | 触发点 | 失败提示 |

| 金额>0 且单边 | 提交审核 | 「第 {n} 行借贷不能同时为零/同时非零」 |

### 5.3 方向推导规则（禁止写死借贷）
```
最终借贷 = f(account.balance_direction, dr_cr_role 的增减语义)
- dr_cr_role=DR 且科目 balance_direction=DEBIT  → 记借方(增)
- dr_cr_role=CR 且科目 balance_direction=CREDIT → 记贷方(增)
- 减少类(冲回/红冲)→ 记反方向
红冲: reversal_handling=NEGATE → 同科目同方向金额取负(红字)
```
> 方向永远从 `Account.balance_direction` + 增减语义推出，effect 不得 hardcode debit/credit 列。

### 5.4 金额取数公式约定
- `LINE.amount * (1 + tax_rate)` = 价税合计（INCLUSIVE）
- `LINE.amount` = 不含税额（EXCLUSIVE）
- `LINE.amount * tax_rate` = 税额（TAX_ONLY），tax_rate 按 tax_rate_source 取
- 成本类：`SUM(InventoryTransaction.total_cost where reference=doc)`（移动加权出库成本）
- 暂估：入库 `qty * 估价`（无发票），冲回 = 暂估金额取负 + 实际进项
- 外币：`amount * exchange_rate`（折本位币入账）
