## ① 模块定位（一句话）

把**人力成本**（工资计提/发放）与**期间费用**（员工报销）从源头单据原子生成凭证并过账写 `AccountBalance`——**没有这两块，利润表的「销售费用/管理费用」是空的，净利润算不出来**；附带**部门辅助核算**让费用可按部门归集，支撑后续管理看板与利润表拆分。

引擎现状一句话：薪酬/报销/员工模型**全部 ❌不存在**（仅 `frontend/src/pages/FlowEditor.jsx:895` 有一句占位 placeholder 文案「员工报销流程」，无任何模型/状态机/effect）。本模块是**纯新建**，但**复用**基石的 Voucher/VoucherEntry/AccountBalance/VOUCHER 状态机 + `finance.post_voucher` 过账层 + `AccountMappingRule` 映射框架 + `Notification` 广播 + cosign 会签件。

## ② 财务对象清单（统一对象范式 8 列）

本模块 4 个对象：**B-工资计提单 PAYROLL_RUN**、**B-费用报销单 EXPENSE_CLAIM**、**A-员工 Employee**（含部门辅助核算锚点）、子表 **PayrollLine / ExpenseClaimLine**（随头走，无独立状态机）。

### 对象 1 · 工资计提单 PAYROLL_RUN（B 交易凭证单据）❌新建

| 列 | 内容 |
|---|---|
| ① 对象 | `PayrollRun`（doc_type `PAYROLL_RUN`，`__doc_types__`）+ 子表 `PayrollLine`（`__queryable__`）。B 类 |
| ② 类型 | B 交易凭证单据（一月一公司一张主单，按员工/部门铺行） |
| ③ 数据 | 见 §③ 字段级数据字典（头：期间/计提月/本位币/合计；行：员工/部门/应发/各扣项/实发/单位社保公积金） |

| ⑤ 逆操作 | 未结账：蓝冲反过账（POSTED→AUDITED，`finance.unpost_voucher` 冲回 AccountBalance）/反审核（AUDITED→DRAFT）→改→重过账。已结账：红冲（`finance.red_reversal` 对计提/发放凭证生成红字+蓝字重做）。作废仅限 DRAFT |
| ⑥ 效果 | 计提过账 effect `payroll.create_accrual_voucher`→`finance.post_voucher`：借 6601/6602/6603（按部门归口费用）+ 借 6602 社保单位部分，贷 2211 应付职工薪酬；发放过账 effect `payroll.create_payment_voucher`→`finance.post_voucher`：借 2211，贷 1002 银行存款 + 贷 2221 应交税费(代扣个税) + 贷 2241 其他应付款(代扣社保个人部分)。详见 §④ |
| ⑦ 回链 | Voucher.source_doc_type=`PAYROLL_RUN` + source_doc_id=本单 id；本单 accrual_voucher_id / payment_voucher_id 反挂；command_log_id 全程留痕；行级 dept 写入 VoucherEntry 辅助核算 |
| ⑧ 准则/账簿 | CAS《职工薪酬》准则：受益对象计提进当期损益（销售/管理/财务费用），代扣代缴经应交税费/其他应付款过渡。HKFRS：Staff costs 进 Administrative/Selling expenses，无强制社保科目（按公司 region 套用模板）。company_id 行级隔离，绝不合账 |
| +拓展 | **计提与发放解耦**（决策范式）：计提=权责发生（费用入账），发放=收付实现（清应付+代扣）；**单位社保公积金计入费用**、**个人部分从应发扣回经其他应付款代缴**；个税科目 222103 应交税费-应交个人所得税（CAS 模板新增明细，标财务确认） |

## ④ ★科目映射取数公式（业务单×动作→借/贷科目代码+金额取数，全部走 AccountMappingRule，标【财务确认】）

> 框架：业务 effect → `map_to_voucher_entries(db, company_id, source_doc_type, trigger_action, doc)` → 建 Voucher 草稿（业财同源同事务）→ 过账跑 `finance.post_voucher`。方向永远 `f(account.balance_direction, dr_cr_role)` 推导，**禁止 effect 写死借贷**。科目码取自基石 digest。

### 4.1 PAYROLL_RUN · 计提（trigger_action=ACCRUE，ACCRUAL_REVIEW→ACCRUAL_POSTED）

| 行 | dr_cr | 科目 | account_source | amount_formula | 说明 |
|---|---|---|---|---|---|
| 1..n | DR | 行 `expense_account_code`(6601/6602/6603) | LINE | Σ同费用科目行(gross_pay + employer_si_hf) | 受益对象费用=应发+单位社保公积金；按 dept 写辅助核算 |
| L | CR | 2211 应付职工薪酬 | FIXED | Σ全行(gross_pay) | 应付员工的应发部分 |
| L+1 | CR | 2211 应付职工薪酬 / 2241 其他应付款 | FIXED | Σ全行(employer_si_hf) | 单位社保公积金挂应付薪酬-社保（CAS 明细，标【财务确认】是否单列 2211 子目还是 2241） |

借贷平衡校验：Σ借(费用)=Σ应发+Σ单位社保 = Σ贷(2211应发 + 单位社保) ✓

### 4.2 PAYROLL_RUN · 发放（trigger_action=PAY，ACCRUAL_POSTED→PAID）

| 行 | dr_cr | 科目 | amount_formula |
|---|---|---|---|
| 1 | DR | 2211 应付职工薪酬 | Σ全行(gross_pay) |
| 2 | CR | 1002 银行存款 | Σ全行(net_pay) |
| 3 | CR | 222103 应交税费-应交个人所得税 | Σ全行(personal_iit) 【财务确认 222103 明细】 |
| 4 | CR | 2241 其他应付款-代扣社保公积金 | Σ全行(personal_si_hf + other_deduction) |

平衡：DR 应发 = CR(实发 + 代扣个税 + 代扣社保个人) ✓（net_pay=gross−personal_si_hf−personal_iit−other_deduction）

### 4.3 EXPENSE_CLAIM · 过账（trigger_action=POST，FIN_REVIEW→POSTED）

| 行 | dr_cr | 科目 | account_source | amount_formula | tax_handling |
|---|---|---|---|---|---|
| 1..n | DR | 行 `expense_account_code` | LINE | Σ同科目行(deductible ? amount_excl : amount_incl) | EXCLUSIVE(可抵)/INCLUSIVE(不可抵) |
| T | DR | 222101 应交税费-进项税额 | FIXED | Σ行(deductible ? tax_amount : 0) | TAX_ONLY，仅专票可抵扣 |
| L | CR | 2241 其他应付款（REIMBURSE 垫付）/ 1002 银行存款（即付） | CONFIG by pay_method | Σ行(amount_incl) | NONE |

### 4.4 EXPENSE_CLAIM · 付款（trigger_action=PAY，POSTED→PAID）

| 行 | dr_cr | 科目 | amount_formula |
|---|---|---|---|
| 1 | DR | 2241 其他应付款 | Σ行(amount_incl) |
| 2 | CR | 1002 银行存款 | Σ行(amount_incl) |

### 4.5 红冲（已结账期）

按基石 `finance.red_reversal`：入参原 voucher_id→生成红字 Voucher（每行 debit/credit 取负、source_doc_type=`VOUCHER`+source_doc_id=原 id、is_auto_generated、description=「红冲 {原凭证号}」）→财务审核过账+蓝字重做。`reversal_handling=NEGATE`（同科目同方向取负），发生额净额=0、余额不变。

> 所有上表每条映射须落地为 `AccountMappingRule` 行并经**财务确认**；本位币外币行按 `amount×exchange_rate` 折本位币（海外公司报销外币时）。

## ⑥ 校验规则 + 边界 + 错误处理（失败精确提示，绝不静默）

| 规则 | 触发点 | 精确提示（失败） |
|---|---|---|
| 借贷平衡 Σ借=Σ贷 | 过账前（hard_rule，全局不变量1） | 「借方合计 {x} ≠ 贷方合计 {y}，差额 {d}，禁止过账」 |
| 工资勾稽 net=gross−个人社保−个税−其他 | PAYROLL 提交审核（逐行） | 「第 {n} 行实发 {net} ≠ 应发−扣项 {calc}，请核对」 |
| 报销勾稽 incl=excl+tax；tax=deductible?excl×rate:0 | EXPENSE 提交（逐行） | 「第 {n} 行含税额 {incl} 与不含税+税额 {calc} 不符」 |
| 期间锁 period.status=OPEN | 所有过账/付款边 | 「会计期间 {period} 已结账，禁止过账；如需冲销请用红冲」 |
| 职责分离（开关） | 审核/过账边 | 「职责分离已开启：过账人不能等于制单人 {name}」 |
| 必输 | 提交边 | 「{字段} 必填」（行/科目/金额/期间/报销事由/票据张数） |
| 票据张数>0 | EXPENSE 复核过账 | 「报销单无附票，请补录票据张数后再过账」 |
| 进项可抵扣一致性 | EXPENSE 过账 | deductible=True 但无专票→「专票未认证，进项不可抵扣，请取消勾选或补认证」 |
| 蓝冲前置：未结账 | POSTED→AUDITED/反过账 | 已结账期触发→「期间已结账，蓝冲不可用，请走红冲」 |
| 红冲必广播 | 业务红冲生成草稿后 | 必发 Notification（见 §⑦），否则视为失败 |
| 作废限 DRAFT | CANCELLED 边 | 「已审核/过账单据不可作废，请走反审核/反过账或红冲」 |
| 公司隔离 | 全查询 | 走 `_company_filter(user)`，跨公司单据不可见/不可操作 |
| 幂等 | create_*_voucher effect | 已存在 source_doc_type+source_doc_id 对应 Voucher→no-op 返回既有（防退回重触发重复建凭证，参照 `confirm_payment_request_settlement` 幂等守卫） |

**边界**：①跨期报销（票据月≠当前期）→按 period_id 选定期间入账，跨已结账期用调整凭证；②负数报销（退费）→走业务红冲/红字行；③外币报销→amount×exchange_rate 折本位币，差额不进汇兑（费用类按入账即时汇率）；④薪资全额扣完为 0 实发（如离职补缴）→允许 net=0，但 gross>0；⑤一人多公司→薪资按 active_company 各自建单，绝不合账。

## ⑩ 验收用例（Given-When-Then，含边界）

1. **正常计提**：Given 某内地公司 8 月在职 3 人（应发合计 30000，单位社保 6000）→ When ACCOUNTANT 制单提交、FINANCE_MANAGER 审核过账 → Then 生成凭证「借 6602 36000 / 贷 2211 30000 + 6000」，借贷平衡，AccountBalance.period_debit(6602)+=36000、period_credit(2211)+=36000。
2. **正常发放**：Given 上单计提已过账（应发 30000、个税 1500、个人社保 3000、实发 25500）→ When CASHIER 发放过账 → Then 「借 2211 30000 / 贷 1002 25500 + 222103 1500 + 2241 3000」平衡，2211 余额归零。
3. **报销即报即付（垫付后付）**：Given 员工差旅含税 1130（不含税 1000、可抵专票税 130）→ When 复核过账→出纳付款 → Then 过账「借 6601 1000 + 222101 130 / 贷 2241 1130」，付款「借 2241 1130 / 贷 1002 1130」。
4. **不可抵扣发票（普票）**：Given 招待费含税 1060，deductible=False → Then 「借 6602 1060（含税全进费用）/ 贷 2241 1060」，无进项税行。
5. **边界·借贷不平**：Given 工资行实发录错致 net≠gross−扣项 → When 提交审核 → Then 校验拦截「第 2 行实发 ≠ 应发−扣项」，禁止进审核。
6. **边界·期间已结账蓝冲**：Given 工资计提凭证已过账、所在期已结账 → When 点反过账 → Then 拦截「期间已结账，蓝冲不可用，请走红冲」；改走 red_reversal 生成红字+蓝字，发生额净额=0。
7. **边界·跨期报销**：Given 7 月票据 8 月报销、7 月已结账 → When 选 period=8 月 → Then 正常入 8 月；若强选 7 月 → 拦截期间锁。
8. **边界·职责分离**：Given segregation_of_duties=ON，制单人=过账人(同一 FINANCE 账号) → When 过账 → Then 拦截「过账人不能等于制单人」；OFF（小公司）→ 放行。
9. **边界·非本人报销不可见**：Given 员工 A 登录 → When 查报销台账 → Then 仅见 A 本人单，B 的单不可见（行级 owner 过滤）。
10. **边界·薪资隐私**：Given 非财务角色（SALES）→ When 访问工资单 → Then 整单不可见/拒绝（字段防火墙）。
11. **红冲广播**：Given FINANCE_MANAGER 对已过账报销红冲 → Then `Notification` 广播 recipient_role ∈ {ACCOUNTANT,FINANCE_MANAGER,FINANCE_DIRECTOR}「报销单 EX-2608-003 已生成红冲凭证待审核」。
12. **幂等**：Given 报销 FIN_REVIEW→POSTED 后退回 DRAFT 又再过账 → Then create_claim_voucher 发现既有 source_doc 凭证→no-op 不重复建凭证。
13. **海外公司外币报销**：Given 香港公司 USD 报销、本位币 USD → Then 直接入账无折算；若员工垫付 HKD → amount×exchange_rate 折 USD 入账。

6. **部门主数据**：本期 dept 用受控字符串字典（对齐 UserAccount.department=String(50)），是否需独立部门表（DepartmentMaster）做层级辅助核算？需确认（影响利润表按部门拆分粒度）。
7. **HR 角色**：员工主数据/薪资制单是否独立 HR 角色，还是并入 ACCOUNTANT/FINANCE_MANAGER？本期并入财务，需确认。
8. **薪资明细可见性**：CASHIER 发放需见实发，是否可见个税/社保明细？隐私边界需财务/老板定。
9. **借款/备用金核销**：员工借款冲报销（ADVANCE_OFFSET）本期占位，是否本期上线？需确认（涉及其他应收款 1221 科目，基石未列）。
