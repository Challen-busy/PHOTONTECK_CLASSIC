## ① 模块定位

**一句话**：把 Layer1 所有已过账凭证「收口」成可结账的当期账——外币按期末汇率调汇、损益结平至本年利润、跑前置校验清单后锁期间，让本期 closing 余额干净结转为下期 opening；并提供启用日的期初建账一刀切与金蝶并行对账验平。

期末是 ERP 财务最不可逆、最高风险的批处理，**必须向导式**（符合 UX 律 14：低频不可逆 = 向导，结账是向导唯一例外）。

## ② 本模块财务对象清单（对象范式 8 列）

本模块涉及 **4 个对象**：(A) 期末调汇批 / (B) 结转损益批 / (C) 期末结账（=AccountingPeriod 状态翻转 + 校验清单）/ (D) 期初建账批。前三个共用 VOUCHER 状态机的 E 类节点（FX_ADJUSTED/PL_TRANSFERRED/CLOSED）+ 各自一个 effect；期初建账是独立一次性 command + 专用导入屏。

### 对象 A · 期末调汇批（FX Revaluation）

| 列 | 内容 |
|----|------|
| ① 对象 | 无独立 model；**effect `finance.fx_revaluation` 生成的调汇 Voucher**（`source_doc_type='FX_REVALUATION'`、`is_auto_generated=True`）；过程批处理挂 VOUCHER 状态机 `POSTED→FX_ADJUSTED` 边 |
| ② 类型 | **E 期末过程批处理**（无专用单据，向导内一步） |
| ③ 数据 | 调汇 Voucher 头（voucher_date=期末日、period_id=当期、voucher_type='转'、description="{期间}期末调汇"）+ 行（每个外币货币性科目一对借贷调差行，见 §④）；辅助：取 `ExchangeRate` 期末汇率 |
| ④ 动作×角色 | 「过账直接调汇」`POSTED→FX_ADJUSTED`：`ACCOUNTANT` 编制（结账向导内触发）；调汇 Voucher 本身再走 审核→过账（`FINANCE_MANAGER`）。时机=当期凭证全过账后、结转损益前。前置：本位币已设、期末汇率已录 |
| ⑤ 逆操作 | 期间 OPEN → 反过账调汇 Voucher（蓝冲，`finance.unpost_voucher`）后删，改汇率重调；期间 CLOSED → 须先反结账。**调汇凭证可直接整张作废重生成（幂等：同期间同科目重跑覆盖）** |
| ⑥ 效果 | `finance.fx_revaluation`（❌新建）：对所有**外币货币性科目**按 (期末汇率−账面汇率)×外币余额 算调差 → 差额借/贷计 **660301 财务费用-汇兑损益**；生成调汇 Voucher 草稿。过账时跑 `finance.post_voucher` 写 AccountBalance |
| ⑦ 回链 | `Voucher.source_doc_type='FX_REVALUATION'`+`source_doc_id`=period_id；`command_log_id`；调汇 Voucher → AccountBalance |
| ⑧ 准则/账簿 | CAS《企业会计准则第19号-外币折算》/ HKFRS=HKAS 21；规则一致（货币性项目按资产负债表日即期汇率折算，差额进当期损益）。`company_id` 账套，仅多币种公司触发 |
| +拓 | **仅货币性项目调汇**：现金/银行/应收账款/应付账款/应收应付票据/其他应收应付=货币性→调；**预收/预付/存货/合同资产负债/固资=非货币性→不调**（财务确认科目调汇标记） |

### 对象 B · 结转损益批（P&L Transfer）

| 列 | 内容 |
|----|------|
| ① 对象 | 无独立 model；**effect `finance.pl_transfer` 生成的结转 Voucher**（`source_doc_type='PL_TRANSFER'`、`is_auto_generated=True`）；挂 VOUCHER 状态机 `FX_ADJUSTED→PL_TRANSFERRED` 边 |
| ② 类型 | **E 期末过程批处理** |
| ③ 数据 | 结转 Voucher 头（description="{期间}结转损益")+ 行：每个有余额的收入类(REVENUE)/费用成本类(COGS/EXPENSE)科目反方向冲平，对方科目 **4103 本年利润** |
| ④ 动作×角色 | 「结转损益」`FX_ADJUSTED→PL_TRANSFERRED`：`ACCOUNTANT` 编制（向导内）；结转 Voucher 再 审核→过账（`FINANCE_MANAGER`）。时机=调汇后。前置：当期收入/费用类凭证全过账 |
| ⑤ 逆操作 | 同对象 A：OPEN 蓝冲反过账后重结，CLOSED 先反结账。幂等：同期重跑覆盖 |
| ⑥ 效果 | `finance.pl_transfer`（❌新建）：把每个收入/费用/成本科目本期 closing 余额反方向结平（收入贷余→借方冲、费用借余→贷方冲），净额计 **4103 本年利润**。结转后这些损益科目本期 closing=0。过账跑 `finance.post_voucher` |
| ⑦ 回链 | `source_doc_type='PL_TRANSFER'`+`source_doc_id`=period_id；→ 4103 余额 |
| ⑧ 准则/账簿 | CAS/HKFRS 一致；**月度均结转至 4103 本年利润**；4103→4104 利润分配为年末单独动作（年末结账时结转，见 §⑪ 边界），不在月结内 |
| +拓 | **结转目标默认 4103（非 4104）**——基石 §1.5 定义 4103=本年利润、4104=利润分配；seed RULE 旧文本误写「结转到 4104」需在 seed 修正为 4103（财务确认）。营业外收支(6301/6711)、税金及附加(6403)、资产减值(6701)等均纳入结转范围 |

| ② 类型 | **E 期末过程批处理**（专用屏=结账向导） |
| ③ 数据 | `AccountingPeriod`(status/closed_by_id/closed_at) + 派生「结账前校验清单」（无独立表，向导内实时算）；本期所有 `AccountBalance` |
| ④ 动作×角色 | 「期末结账」`PL_TRANSFERRED→CLOSED`：`FINANCE_MANAGER`+`BOSS`（会签）。时机=调汇+结转完成。前置=校验清单全绿（见 §⑥） |

## ④ ★科目映射取数公式（业务批 → 凭证分录·标「财务确认」）

> 方向永远从 `Account.balance_direction` 推（基石 §5.3），下表「记借/记贷」是结果示意，effect 不写死。

### 期末调汇 `finance.fx_revaluation`（仅外币货币性科目，逐科目）

```
调差额 = 外币余额 × (期末汇率 − 账面账面汇率)   # 折本位币的增减
账面本位币余额 = AccountBalance.closing（本位币）
重估本位币余额 = 外币余额 × 期末汇率
调差 = 重估 − 账面
```

| 场景 | line | DR/CR | 科目 | 金额公式 | 说明 |
|------|------|-------|------|----------|------|
| 资产类货币性升值(如外币银行/应收，期末汇率↑) | 1 | DR | 该外币科目(1002/1122/1121/1221…) | abs(调差) | 资产折本位币增→借 |
| 同上 | 2 | CR | **660301 财务费用-汇兑损益** | abs(调差) | 收益→贷(冲减费用) |
| 资产类货币性贬值 | 1 | DR | 660301 财务费用-汇兑损益 | abs(调差) | 损失→借 |
| 同上 | 2 | CR | 该外币科目 | abs(调差) | 资产减→贷 |
| 负债类货币性(应付2202/2201、其他应付2241)升值 | 1 | DR | 660301 汇兑损益 | abs(调差) | 负债折本位币增=损失→借 |
| 同上 | 2 | CR | 该外币负债科目 | abs(调差) | 负债增→贷 |
| 负债类货币性贬值 | 1 | DR | 该外币负债科目 | abs(调差) | 负债减→借 |
| 同上 | 2 | CR | 660301 汇兑损益 | abs(调差) | 收益→贷 |

> **调汇范围（财务确认）**：货币性=1001/1002/1012/1121/1122/1221/2201/2202/2241 等；**非货币性不调**=1123预付/2203预收/1405存货/1231合同资产/2205合同负债/1601固资。调汇后更新该科目账面汇率为期末汇率。

### 结转损益 `finance.pl_transfer`（逐损益科目，结平至 4103）

| 场景 | line | DR/CR | 科目 | 金额公式 |
|------|------|-------|------|----------|
| 收入类(6001/6051/6111/6301)结平 | 1 | DR | 该收入科目 | 该科目本期 closing 贷余净额 |
| 费用成本类(6401/6402/6403/6601/6602/6603/6701/6711)结平 | 2 | CR | 该费用科目 | 该科目本期 closing 借余净额 |
| 差额转本年利润 | 末 | DR或CR | **4103 本年利润** | Σ收入 − Σ费用（盈→贷4103，亏→借4103） |

> 结转后所有损益科目本期 closing=0。**年末额外**：FiscalYear 结账时把 4103 本年利润全额结转 4104 利润分配（年末专跑，财务确认）。

### 期初建账 `finance.import_opening_balance`

| 写入 | 目标 | 公式 |
|------|------|------|
| 科目期初 | `AccountBalance.opening_debit/credit`（启用期） | 直接取清单1，借贷各自累加 |
| 未结往来 | AR/AP 台账行 | 取清单2，status=PENDING、batch='OPENING' |
| 存货 | InventoryValuation + 期初 InventoryTransaction(type=OPENING) | 清单3，total_value=qty×unit_cost |
| 平衡校验 | — | **Σ清单1 opening_debit = Σ opening_credit**（不平禁导入） |

> 期初不生成凭证（直接写 opening 列，区别于日常过账走 period 列）；这是「建账期专属写入路径」，与全局不变量5（财务唯一写入路径=凭证过账）的唯一例外，**仅启用期一次，须 FINANCE_DIRECTOR 确认**。

## ⑥ 校验规则 + 边界 + 错误处理（绝不静默）

### 结账前校验清单（hard_rule，向导内逐条实时算，全绿才放行 close_period）

| # | 规则 | 校验点 | 失败提示 |
|---|------|--------|----------|
| 1 | **无未过账凭证** | 当期无 status∈{DRAFT,AUDITED} 的 Voucher | 「当期有 {n} 张凭证未过账，请先过账或作废：{凭证号列表}」 |
| 2 | **借贷平衡(试算)** | Σ本期 closing_debit = Σ closing_credit（误差<0.01） | 「试算不平：借 {x} ≠ 贷 {y}，差额 {d}，禁结账」 |
| 3 | **调汇已完成** | 多币种公司须已生成并过账当期 FX_REVALUATION 凭证（无外币则跳过） | 「存在外币货币性科目余额未调汇，请先期末调汇」 |
| 4 | **损益已结转** | 所有损益科目本期 closing=0（已结平至4103） | 「{科目} 损益未结转(余额{x})，请先结转损益」 |
| 5 | **成本已核** | 当期 InventoryTransaction 暂估出库已转实际/差异已冲回（依赖存货成本模块） | 「{n} 笔出库仍为暂估成本，请先期末成本核算」 |
| 6 | **银行已对** | 当期银行流水已对账（依赖出纳资金模块标记，软校验/可跳过+留痕） | 「银行未对账，确认仍要结账？(留痕)」 |
| 7 | **往来已核** | 应收应付已核对（软校验/可跳过+留痕） | 「往来未全部核对，确认仍要结账？(留痕)」 |
| 8 | **前序期已结** | 上一期 status=CLOSED（顺序结账，不可跳期结） | 「上一期 {y}-{p} 未结账，须按顺序逐期结账」 |

### 调汇/结转过程校验

| 规则 | 触发 | 提示 |
|------|------|------|
| 期末汇率已录 | 调汇 | 「{币种}→{本位币} 缺期末({期末日})汇率，请先在汇率表录入」 |

| 规则 | 提示 |
|------|------|

## ⑪ 待财务确认 gap

1. **结转损益目标科目**：月结结转至 **4103 本年利润**（基石 §1.5）；seed 旧 RULE 文本写「4104」需修正为 4103。**确认月结目标=4103、4103→4104 利润分配仅年末结转**。
