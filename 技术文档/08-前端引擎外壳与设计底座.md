# 08 · 前端引擎外壳与设计底座

## 8.1 模块定位

一个业务无关的 React SPA 外壳，在通用后端引擎之上渲染元数据驱动的管理界面。它提供：(1) 薄 axios 传输层（`/api` 代理 + Cookie 会话 + 401→登录拦截）；(2) 会话鉴权上下文 + 路由守卫；(3) 常驻应用框架（固定侧栏、粘性顶栏、用户菜单、待办徽标轮询、Outlet）；(4) 四个可复用的 **schema/元数据驱动视图壳**——数据浏览器、流程图查看/编辑器、通用单据编辑器、命令日志查看器；(5) 带 SSE 思维流与"确认才提交"ChangeCard 的 Agent 对话面。所有视觉身份集中在一套 ElevenLabs 风格 token 体系。

> 关键文件：`frontend/src/api.js`、`auth.jsx`、`App.jsx`、`main.jsx`、`index.css`、`components/{Layout,DocEditor,ChangeCard,WorkflowEdge}.jsx`、`pages/{DataExplorer,WorkflowActions,FlowEditor,NodeView,AgentChat,CommandCenter}.jsx`、`utils/layout.js`

## 8.2 传输与鉴权壳
- `api.js:3`：`axios.create({baseURL:'/api', withCredentials:true})` + 单一 401 响应拦截器（非 /login 时跳登录）。通用 RPC 薄封装：`query/aggregate/transition/previewTransition/commitTransition/getWorkflows/getHistory/agentChat` + 命令中心簇。
- `auth.jsx`：`AuthProvider` 挂载即 `GET /me` 水合 `user`，暴露 `login/logout/user/loading`；`Protected`（`App.jsx:21`）加载中转圈、无 user 跳登录。`main.jsx`：`BrowserRouter → ConfigProvider(zhCN, theme) → App`。

## 8.3 四个元数据驱动视图壳

### 数据浏览器（`DataExplorer.jsx`）— 靠 introspection
`query(table,{search,limit:200})` 取行；`allCols`(:318) **useMemo over `Object.keys(data[0])`**——纯从首行派生列定义：`status` 自动转过滤器、采样判数值/字符串排序、数字右对齐，render 把 `status` 渲成 `StatusPill`、ISO 时间格式化、`amount/price/cost/quantity/total` 千分位、对象 `JSON.stringify`；有 `status` 列时追加"历史"动作列跳 `/history/:docType/:id`。列显隐 + 密度 + 客户端 BOM CSV 导出。`DataLanding` 按 `user.allowed_tables` 门控表卡 + `aggregate(t,'id','COUNT')` 实时计数。

### 流程图查看/编辑器（`WorkflowActions.jsx` 运行态 + `FlowEditor.jsx` 管理态）
两者都消费 `workflow.states`(节点数组) + `workflow.node_positions`。states→nodes、`s.next`(过滤已知码)→edges；`decorateWorkflowEdges` 算同对 sibling 索引做标签扇出；无坐标则 `layoutGraph(nodes,edges,'TB')` 跑 dagre 自动布局。`FlowEditor` 全 CRUD JSONB states：addNode/saveNode/deleteNode `PATCH /admin/workflows/:id/states {states,force:dangerMode}`、`POST /admin/save-positions`、生命周期 create/fork/publish/disable/enable/delete、"发布即锁内容 + danger 覆盖 + 审计弹窗"。右栏编辑每条边 `label/target/roles/editable_fields/hard_rules/hooks`。

### 通用单据编辑器（`DocEditor.jsx`）— 最深的通用壳
`load`(:401) 扇出：`GET /schema/:table`(字段描述) + `query(table,{filters:{id}})` + `getHistory` + `GET /related/:table/:id` + `GET /workflows` + 为每个 `field.fk` 预取选项。后端 `/schema` 把列自省成 `{name,label,type,nullable,primary_key,fk,has_default}` 并自动派生子表(名含 `_line/_entry` 的反向 FK)。`classifyField` 按名分桶(hero_id/hero_party/hero_amount/status/system/hidden/detail)。动作区按 `actions[]` 渲 tab；选中动作的 `editable_fields` 决定哪些字段变可编辑输入。`FieldInput` 按 type→控件(fk→可搜 Select、boolean/date/datetime/number/json/text/string)。`SubTableEditor` `GET /schema/:subtable` 装子行，`_new/_modified/_delete` 脏模型、自动 line_number、quantity*unit_price→total_price，`getPendingUpdates()` 回 diff。提交：`requestAction`→刷子表→`previewTransition`→`pendingCard`→ChangeCard→`commitTransition`。`NodeView` 若状态有 `custom_html` 则 `innerHTML` 注入（原生 HTML 逃生舱）。

### Agent 对话（`AgentChat.jsx`）— SSE
`POST /agent/chat/stream`(`credentials:'include'`)，`getReader()` 流式解析 `data: {json}` 成 thinking/tool_call/tool_result/done；`done` 带 `cards` 则填 `pendingCards`，输入锁定直到全部处理。`ChangeCard.jsx` 是共享确认件：渲 `changes[]` from→to、`checks`、`reason`、建议口吻，Approve→`commitTransition` / Reject。

### 命令日志查看器（`CommandCenter.jsx`）
多字段过滤 + 失败汇总 + 详情抽屉(请求/结果 payload + 能力标志) + 可重试失败项的 retry。

## 8.4 设计底座（`main.jsx` + `index.css` + `DESIGN.MD`）
单个 antd `ConfigProvider.theme` 集中 token：`colorPrimary #000`、近白底、暖石选中 `#f5f2ef`、Inter 字体、圆角 4/8/12/16、药丸按钮 `borderRadius 9999`、多层 sub-0.1 阴影；`index.css` 镜像为 `--pt-*` CSS 变量。组件复用 `CARD_SHADOW` 常量做卡片高度。`DESIGN.MD` 是 ElevenLabs 风格权威规范（Inter / 近白画布 / 暖石 / 药丸 / 多层低透阴影）。

## 8.5 引擎数据契约（前端消费的形状）
```
WorkflowDefinition: {states:[{code,name,node_type?,is_initial?,is_terminal?,allowed_roles?,
  description?,hard_rules?[],hooks?[],custom_html?, next:[{to,label,roles?[],editable_fields?[],hard_rules?[],hooks?[]}]}],
  node_positions:{code:{x,y}}, ...}
/schema 响应: {table, table_label, fields:[{name,label,type,nullable,primary_key,fk,has_default}], sub_tables:[{table,table_label,parent_fk}]}
/related 响应: {forward:[{field,target_table_label,row,labels}], reverse:[{table,fk_field,count,samples[],labels}]}
ChangeCard / commit 响应 / CommandLog 行 / User(/me) / SSE 事件 —— 见各后端模块分册
```

## 8.6 功能需求

- **FR-8.1** 单 axios 传输：`/api` 基址 + 凭据(Cookie) + 全局 401→登录拦截。
- **FR-8.2** 会话鉴权上下文：挂载从 `/me` 水合用户，暴露 login/logout，路由守卫先 loading 再对未鉴权跳登录。
- **FR-8.3** 常驻应用框架：声明式菜单（admin-only 标志按用户能力过滤）的可折叠侧栏、含身份+角色徽标+登出的粘性顶栏、定时轮询的通知徽标、路由内容出口。
- **FR-8.4** 通用数据浏览器：给表名经 `/query` 取行，**按首行 introspection 自动生成列**（类型感知排序、status 自动过滤、数字/时间格式化、JSON 预览）、列显隐、密度、键值详情抽屉、客户端 CSV 导出；着陆页按每用户允许集列表 + `/aggregate` 实时计数。
- **FR-8.5** 流程图渲染器：消费定义的 states(码/名/类型/初/终/出边) + 保存坐标，构 React-Flow 图（缺坐标 dagre TB 自动布局），按类型/状态码着色，多边标签扇出。
- **FR-8.6** 管理流程编辑器：create/fork/publish/disable/enable/delete，整 states 数组 PATCH 增删节点/边，拖拽重定位显式保存，"发布即锁内容 + 受审计的 danger 覆盖"，变更审计查看器。
- **FR-8.7** 通用单据编辑器：给 doc-type/记录 id 取 `/schema`(类型字段 + 自动子表) + 记录 + 正反向关联 + 历史；渲只读详情 + 按动作 tab（可编辑字段由流程边 editable_fields 驱动）；自动挂脏跟踪子表编辑器；经 预览→确认卡→提交 提交，回显规则失败。
- **FR-8.8** 可复用变更确认卡：渲字段 diff(from→to)、检查、建议、批准/拒绝；Agent 对话与单据编辑器共享。
- **FR-8.9** Agent 对话面：SSE 流式、实时"思维"步（工具调用与结果配对 + 耗时）、markdown 答复；响应含待确认卡时锁输入直到逐卡批准/拒绝（含全批/全拒）。
- **FR-8.10** 每节点原生 HTML 逃生舱：流程状态可供 `custom_html` 就地渲染替代通用编辑器。
- **FR-8.11** 通用操作日志查看器：多字段过滤、失败汇总、详情抽屉(请求/结果 payload + 能力标志)、可重试失败项 retry。
- **FR-8.12** 视觉身份集中在一个 theme token 对象 + 镜像 CSS 变量（近白画布、暖石强调、单主色、Inter、药丸控件、慷慨卡片圆角、规范多层低透卡片阴影），一处即可整体换肤。

## 8.7 边界：本模块明确排除的业务

- `api.js` 的 WMS 簇（`getWms*/reserveWmsInventory/...`）与 `getOrderChains`。
- 业务页面：`BusinessPortal/OrderChainMonitor/WmsWorkbench/Dashboard/MyTodos/ReportDrawer/DocHistory/Login/Admin`。
- 业务字典：`DataExplorer.TABLE_META/CATEGORIES/STATUS_STYLE/DOC_TYPE_MAP`、`DocEditor.VALUE_LABELS/TABLE_MAP/ID_FIELDS/HERO_FK_FIELDS`、`FlowEditor.stateColors/ROLE_OPTIONS/REPORT_NODE_MAP`、`Layout` 业务菜单项/`roleColors`、`CommandCenter` 库存流水列、`NodeView` 的 doc_type→table 映射/REPORT_CONFIG。

## 8.8 现状与差距

- **D-08a** 无 `ARCHITECTURE.md` 管前端；`DESIGN.MD` 是设计规范非架构文档，与主题/CSS 实现一致。
- **D-08b** `DESIGN.MD` 指定显示字体 Waldenburg 300，但主题只挂 Inter、无 Waldenburg @font-face——显示标题实际是 Inter 300，签名字体**未接线**。
- **D-08c** `DESIGN.MD` 强调暖色阴影 `rgba(78,50,23,0.04)` 与 inset 半像素边，`index.css` 定义了 `--pt-shadow-warm/inset` 但组件普遍用中性 `CARD_SHADOW`——暖色/inset 阴影层**基本未用**。
- **D-08d** `custom_html` 经 `innerHTML` 注入是原生逃生舱——若内容来源不可信存在 XSS 面，需在治理上约束 custom_html 只由可信管理员写。
