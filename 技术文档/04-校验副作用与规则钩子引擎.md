# 04 · 校验 · 副作用 · 规则 · 钩子引擎

> 这是"业务 = 状态机 + **字段变更** + **副作用**"里"副作用/校验"那一极的接入面。它由两套东西组成：
> （A）**Validator / Effect 注册器**——业务把领域校验与领域副作用挂到流程推进上；
> （B）两套**受限 DSL**——`hard_rules`（只读断言）与 `hooks`（多语句脚本），声明在流程节点 JSONB 里、由状态机引擎评估。
> 注册器在 **Workflow** 侧（`services/workflow_extensions.py`），不是命令注册器。

> 关键文件：`services/workflow_extensions.py`、`services/rules.py`、`services/hooks.py`、`services/workflow.py`（接线 ~315-396）、`core/registry.py`

## 4.A Validator / Effect 注册器

### 4.A.1 注册（扩展点）

```python
@register_transition_validator("<name>", doc_type=?, to_state=?, auto=True)
async def v(db, doc_type, doc, to_state, user) -> list[str]:
    return [...]   # 非空=阻断，元素是失败原因串

@register_transition_effect("<name>", doc_type=?, to_state=?, auto=True)
async def e(db, doc_type, doc, to_state, user, command_log_id) -> list[str]:
    return [...]   # 返回可读日志行
```

- `_TransitionExtension.matches(doc_type,to_state)`：`None` 当通配；`(doc_type,to_state)` 都匹配才命中。
- 扩展模块在 `workflow_extensions._EXTENSION_MODULES` 列出，首次跑校验/副作用时惰性 import（`load_workflow_extensions` 幂等）。

### 4.A.2 调度语义（引擎契约）

- **Validator**：仅在**推进**时，`run_transition_validators` 跑所有 `auto` 且匹配的校验器，拼接失败串；非空→回滚 + `{error:'领域校验未通过', rule_failures}`。
- **Effect**：仅在**推进**时，`run_transition_effects` 分两轮：
  1. **EXPLICIT**——节点/边 JSONB `effects:[...]` 里显式列名，逐个按名查注册表（未注册→`ValueError` 中止整笔推进），跑并标记已执行；
  2. **AUTO**——再跑所有 `auto` 且匹配 `(doc_type,to_state)` 且未执行过的 effect。
- 两种触发对照：
  - `auto=True`：命中 `(doc_type,to_state)` 自动执行（如"出库扣库存""入库结成本"这类与态强绑定的）。
  - `auto=False`：仅当节点/边显式声明 `effects:["..."]` 才执行（如"询价→报价""订单→采购通知"这类跨单据派生）。
- Effect 拿得到 `command_log_id`（来自外层命令），可把派生写挂到同一命令的审计链上。

## 4.B 规则 DSL（`services/rules.py`）— 只读断言

- 每条规则是单个表达式，`ast` `mode="eval"` 解析。`validate_rule` 用**节点白名单** `ALLOWED_NODES`：放行布尔/算术/比较/下标/属性读/推导式/常量；**Assign/Import/FunctionDef/ClassDef/Lambda 一律拒绝**。
- 函数调用白名单 `ALLOWED_FUNCS = {sum,len,min,max,all,any,abs, lookup,query,count,sum_field, str,int,float,bool}`；**对象方法调用（`doc.delete()` 之类）无条件拒绝**。
- 求值沙箱：`__builtins__` 限定为固定 `SAFE_BUILTINS`，上下文 = 跨表 helper + 子表 + `{"doc": doc}`。规则若抛错或返回 falsy 即"失败"，收集为 `"✗ <rule>"`。
- 跨表 helper（`lookup/query/count/sum_field`）经 `core.registry.table_map()` 解析表名跑同步 SELECT，可跨表断言；子表上下文自动加载名含 `_line/_entry` 且指向 doc 的子行（别名 `entries/lines`）。
- 公开异步入口 `evaluate_rules(db, doc, rules)` 经 `db.run_sync` 跑，返回 `(全过?, 失败信息列表)`。

## 4.C 钩子 DSL（`services/hooks.py`）— 多语句脚本

- 与规则对称但 `mode="exec"`：白名单是规则的超集，**额外放行 Assign/AugAssign/For/If/IfExp/Pass**；仍拒绝 Import/def/class/lambda/while/try/with/return/yield/raise。
- 额外守卫：Assign/AugAssign 目标只能是 `ast.Name`（`obj.attr=x`、`d[k]=v` 拒绝）；对 Attribute 的调用拒绝。
- **写能力特性开关**：`ALLOW_WRITE_HOOKS = os.environ["ALLOW_LEGACY_WORKFLOW_WRITE_HOOKS"]=="1"`；仅当置位时 `insert/update` 才进 `ALLOWED_FUNCS`。
- 上下文：`doc`、子表、读 helper（`lookup/query/count/sum_field`）、日期 helper（`today/now`）；启用写时 `insert(table,fields)`（registry 解析 + 字段过滤 + flush + 记 `insert table#id`）、`update(table,where,fields)`（**要求非空 where**，防全表更新，记 `update table xN`），返回 `op_log`。
- `execute_hooks_sync` 先逐条 `validate_hook` 再 `exec`（依赖 AST 白名单已挡住恶意形态）；钩子异常向上传播触发回滚。

## 4.D 声明与接线（由状态机引擎消费）

- `hard_rules`、`hooks` 都活在 `WorkflowDefinition.states` JSONB 里（节点级 + 边级）。
- `execute_transition`：`rules = current_state.hard_rules + edge.hard_rules` 经 `evaluate_rules`，失败回滚 `{error:'自动校验未通过'}`；`hooks = current + edge + 进入态` 经 `execute_hooks_sync`，在规则过且切态后、提交前跑，失败回滚 `'钩子执行失败'`。预览路径不写库地重跑规则。

## 4.E 现状取向：effects 优先于 hooks

`hooks` 的写能力默认禁用，`ALLOW_LEGACY_WORKFLOW_WRITE_HOOKS=1` 只为迁移期残留老流程定义保留。**新增/维护的业务流必须用已注册的 effects 或 commands**，不允许在流程定义里写内联写入钩子（呼应 INV-1：副作用归 Effect 注册器）。

## 4.F 功能需求

- **FR-4.1** 业务须能用装饰器按 (doc_type?, to_state?) + auto 标志注册领域 Validator（返回失败串列表即阻断推进）与领域 Effect（返回可读日志行）。
- **FR-4.2** 扩展模块须惰性、幂等自动发现，首次评估前完成注册。
- **FR-4.3** Effect 须支持 AUTO（匹配即跑）与 EXPLICIT（节点/边 effects 数组列名才跑）两种派发；命名未注册 effect 须中止整笔推进。
- **FR-4.4** Effect 须能拿到外层命令 id，使派生写归并到同一命令审计链。
- **FR-4.5** 须提供只读断言 DSL：单表达式，AST 节点白名单 + 函数白名单静态校验；assignment/import/def/class/lambda/对象方法调用须在校验期拒绝。
- **FR-4.6** 规则求值须在受限 builtins 沙箱内，上下文含目标单据、自动加载的子集合、跨表查询 helper；真值且不抛才算过，失败回传可读信息。
- **FR-4.7** 须提供多语句脚本 DSL（hooks），是规则文法的严格超集（加局部名赋值/for/if/pass），仍禁 import/def/class/lambda/while/try/with/return/yield/raise 与属性/下标赋值、对象方法调用。
- **FR-4.8** 钩子写操作（insert/update）默认禁用，仅经显式环境迁移开关启用；启用时 update 须要求非空 where，且每次写须向返回给调用方的 op_log 追加可读条目。
- **FR-4.9** rules/hooks 须以数据形式声明在流程节点定义（节点级 + 边级数组）；推进器须先评估并集规则、后跑并集钩子，任一失败整体回滚。
- **FR-4.10** DSL 查询面须经中央注册器解析表/类型名；子表按 FK + 命名约定自动发现。
- **FR-4.11** 引擎须以"effects 优先、写钩子默认禁用"为既定取向，写钩子仅由迁移开关临时开启。

## 4.G 边界：本模块明确排除的业务

- 具体 Validator/Effect（`wms.validate_shipment`、`apply_shipment_costs`、库存成本结转、凭证生成…）与注册它们的模块（`phase1_effects.py`、`wms_workflow_extensions.py`）。
- 整条业务派生链（询价→报价→订单→采购→入库→出库→发票→应收/应付）。
- 任何具体规则/钩子表达式内容（信用额度、FIFO 之类业务规则文本）。

## 4.H 现状与差距

- **D-04a** `ARCHITECTURE.md` 把 Validator/Effect 注册写得像挨着命令注册器；实际在 Workflow 侧。归属已在本篇厘清。
- **D-04b** `models.py:1287` 称 hard_rules 未实现执行器——**假**，本篇 4.D 即其接线。
- **D-04c** 钩子 `exec` 用的是完整 builtins，安全完全依赖 AST 白名单在前置已挡住恶意形态（代码注释自陈）。这是有意的设计取舍，但意味着白名单是唯一防线，新增放行节点须极谨慎。
